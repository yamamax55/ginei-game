# 縦スライス実装ロードマップ（最小の1ループを通す）

> 本書は「**何から実装するか**」の整理。新システムの設計ではなく、**すでに設計済みの大量のシステムを“着地”させる順番**を示す。
> 上位の計画は [`roadmap.md`](./roadmap.md)、各システムの設計は [`README.md`](./README.md) の各設計書を参照。

---

## 0. 現状認識（なぜこの文書が要るか）
- **設計が実装に対して極端に先行している。** 政治・経済・群像・宗教・外交など Phase D の設計 Issue が約100件ある一方、**戦略コア（Phase C）も Phase A の RTS操作も未着手**。
- **実装済み**：戦術コア（Battle シーン＝個艦戦闘・陣形・提督能力・士気・ブラックホール・後退・設定画面…）＋**多勢力基盤**（`FactionData`/`FactionRelations`）＋`RankSystem`（未使用）。
- **未着手**：Phase A の RTS操作（#82/#85 ほか）、Phase C 戦略レイヤー（銀河グラフ #34 ほか）、Phase D 全部（#106〜#194 は**設計のみ**）。
- **リスク**：縦に1本通っていないため、深い meta システムが「**実際に何に効くか**」を実機で確認できない。横展開を続けるほど**統合不能リスク**が増える。

## 1. 目標：最小の縦スライス（end-to-end 1ループ）
次の1ループが**実機で通る**ことをゴールにする：

> **1惑星から開始 → ハイパーレーンで隣星系へ艦隊移動 → 接触で既存の会戦（Battle シーン）→ 勝者が惑星を占領 → 占領惑星の POP が税/資源を産む → 支持が動く → 1つの政治的帰結が画面に出る**

これが回れば、外交・世論・宗教・経済・群像の深いシステムは「**この回路のどこに刺すか**」で**増分実装**できる。逆にこれが無いまま meta を足し続けると、どれも宙に浮く。

## 2. 横断基盤（最優先・他のすべての前提）
縦スライスに着手する前に、**最小サブセットだけ**入れておく（後の統合で暴走・破損を防ぐ）：
- **#106 係数（修正子）パイプライン** … すべての meta 効果の唯一の合流点。最初に薄く作る。
- **#108 SaveData スキーマ規約** … 戦略状態（マップ・所有・POP）の保存。`schemaVersion`＋migration の器だけ。
- **#116 イベントエンジン** … 占領・政治的帰結の発火（データ駆動）。最小の発火→効果だけ。
- （**#107 入力マッピング** … 戦略/戦術のキー整理。スライスで操作が増えてから。）

> いずれも**スライスに必要な最小機能のみ**。フル実装は後段。

## 3. 縦スライスのビルド順（各システムは「最小サブセット」）
既存 Issue の**最小版**だけを順に積む。括弧内が対応 Issue。

| ステップ | 内容（最小版） | 後回しにするもの |
|---|---|---|
| **S1 戦略マップ最小**（#34 C-1） | 3〜5星系のグラフ＋**ハイパーレーンで接続**（#177）＋艦隊コマの移動 | ハイパーレーン整備レベル#177・時間制ワープの精緻化・手続き生成#118 |
| **S2 戦術への受け渡し**（#36 C-3） | 2艦隊が同星系で接触 → **既存 Battle シーンを起動** → 結果を戦略へ返す | シームレスズーム#35（最初はシーン遷移で可）・有界戦場の作り込み |
| **S3 占領**（#131/#132） | 勝者が**無防備な惑星を占領**（所有が移る） | 侵略値#133・軌道防衛#134・航行不能領域#135 |
| **S4 惑星の最小内政**（#109/#110/#93） | 惑星に **1つの POP 値＋産出**。占領で所有移転 | 階級モデル#110 の深掘り・安定度#111・政策#112 |
| **S5 税と支持**（#162/#113） | POP から**税→国庫**、**税率レバー1つ**で支持が動く | 社会保障#162・市場経済#179・財政PB#161 |
| **S6 政治的帰結1つ**（#14 or #117） | 支持が閾値を割ると **1イベント**（例：提案可否が変わる／不満が出る） | 政党#159・世論#193・政体創発#117 の全体 |

→ **S6 まで通れば「戦術 ↔ 戦略 ↔ meta」が1本** ＝ 縦スライス完成。

## 4. スライス後の増分（深いシステムを“どこに刺すか”）
スライスが通った後、設計済み EPIC を回路の各点に接続していく：
- **会戦の前後** → 外交 #189（宣戦/講和・casus belli）、自動解決 #41。
- **占領・内政** → 階級 #110／宗教 #172／民族 #194／生活水準 #181。
- **支持** → 世論 #193／政党 #159／政体創発 #117／三権 #171。
- **経済** → 市場 #179／財政・税の高度化 #161/#163。
- **群像** → 人物ライフサイクル #150／政府役職 #141／艦隊編制 #146/#147（「誰が統治・指揮するか」）。

各 EPIC は「**スライスのどの点に刺さるか**」が明確なので、**独立に・任意の順で**増分できる。

## 5. もう一つの近道：戦術スライス（家でテスト容易）
meta の縦スライス（§1）とは独立に、**いま最もテストしやすい**のは戦術コアの肉付け：
- **#82 ドラッグ矩形選択 → #85 標準命令（アタックムーブ/停止/保持）→ #69 配下艦挙動** で「RTS会戦として面白い」を先に確立。
- これは Battle シーン単体で完結し、戦略レイヤーを待たずに進められる（`roadmap.md` の S1/S2 と同じ）。

> **2トラック並走が現実的**：Track A＝戦術を面白くする（家でテスト）／Track B＝§3 の meta 縦スライス（統合の証明）。

## 6. 推奨スプリント（スライス優先）
- **Sprint 0（基盤）**：#106 / #108 / #116 の最小サブセット。
- **Sprint 1（戦略マップ）**：S1。
- **Sprint 2（戦術受け渡し）**：S2。
- **Sprint 3（占領＋内政＋税＋支持＋帰結）**：S3〜S6 → **縦スライス完成**。
- **以後**：§4 に従い EPIC を増分接続。

## 7. 方針との整合（崩さない前提）
- **タイクン化回避**：各最小版は「**1レバー＋1帰結**」。指標を増やさない。
- **後方互換**：戦略レイヤーは既存の**単体シナリオ会戦と併存**（`GameSettings.gameMode`、創発 #117）。スライスは別モードで、**既存 Battle を壊さない**。
- **既存窓口を流用**：戦闘在庫 `FleetRegistry` と編制 `OrderOfBattle`(#146系) を混同しない／敵対判定は `FactionRelations` の唯一窓口（外交 #189 が駆動）／meta 効果は #106 に集約／シーン間状態は `GameSettings`。
