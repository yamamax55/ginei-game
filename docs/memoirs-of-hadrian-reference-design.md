# ハドリアヌス帝の回想 参考設計（EPIC #HDRN）

> 参照元：マルグリット・ユルスナール『ハドリアヌス帝の回想』。死を間近にした老帝が、後継者に宛てて自らの生涯・統治・死生観を一人称で語る回想録という形式の小説。
> 戦役でなく**元首の主観（一人称内面）と後継指名（引き際の美学）**で「権力の終わり方」を描く。
> 本ドキュメントは、当プロジェクト（Ginei＝銀英伝風の星間国家戦略＋既に巨大な人物・継承・統治の純ロジック層）にとって**役に立つ視点**だけを抽出し、EPIC `#HDRN` として issue 化する提案。
> 著作権注意：固有名・文章・キャラクター・固有設定（ハドリアヌス／アントニヌス／個別エピソード等）は一切流用せず、**「老いた元首が後継を選び引き継ぐ」という統治の構造パターンのみ**を参考にする。

## 0. なぜ「ハドリアヌス帝の回想」が本システムに役立つか

当プロジェクトは継承・人物・統治の**マクロ純ロジックを大量に保有**している（CLAUDE.md 参照）：

| 既存（カバー範囲） | 何をしているか |
|---|---|
| `SuccessionLawRules`/`SuccessionLaw`（PDX-1 #646） | 長子/分割/指名/選挙の**法形式**・相続シェア・継承危機リスク |
| `SuccessionRules`/`Organization`（#812） | 英雄死後の**組織存続**＝制度化分は残り個人カリスマ分は後継者の正統性×カリスマで引き継ぐ |
| `VacancyRules`（LIFE-2 #152） | 死亡/捕虜で空いた**役職席の後任補充**（有資格の最高tier・同点は若い） |
| `LifecycleRules`/`Calendar`（LIFE-1/2） | 年齢・**死亡率カーブ**・没年・`IsDeceased`/`IsAvailable` |
| `RetirementRules`/`ServiceStatus`（#530-536） | 階級別停年・**アップオアアウト**・元帥終身・戦時召集・恩給 |
| `Person`/`PersonRules`（正名 #866） | 軍人/文民の**適材適所**・役割一致の実効値 |
| `DynastyRules`/`Regime`（孔子 #867） | 天命・腐敗・徳・正統性・易姓革命 |
| `DisclosureRules`/`DisclosureLedger`（FND-4） | 条件→開示の**物語の背骨**（秘史/真相/エンディング） |
| `ProtagonistRules`（GON-6） | 主人公提督＝AI非制御フラグ |

**しかし、これらはすべて「制度・法・盤面」から見た第三者視点の継承**であり、本作が固有に描く以下が**欠けている**：

| 本作が固有に持つ視点 | 当プロジェクトでの欠落 |
|---|---|
| **元首の一人称内面（死の自覚・統治への倦怠・遺したいもの）** | 統治者は能力値の束。**元首が「自分の終わり」を主観的に意識し意思決定が変わる**回路が無い |
| **後継者の「資質」を選ぶ＝法的順位でなく中身で選ぶ** | `SuccessionLawRules` は**法形式**（誰が継ぐ権利を持つか）。**「誰がふさわしいか」を能力・徳・適合で吟味する指名**が無い |
| **指名養子（後継を血統外から選び“息子”にする）** | 継承は血統/法順位/役職tier。**血統外の人物を後継に引き上げる養子的指名**の経路が無い |
| **生前継承・摂政・引き際（自発的撤退の美学）** | `RetirementRules` は停年＝**強制リタイア**。**元首が自ら時期を選んで譲位し、過渡期を摂政で橋渡しする**仕組みが無い |
| **回想＝統治の自己評価が正統性／lore になる** | `DisclosureLedger` は外的トリガの開示。**治世を回顧して評価が後継の正統性・国家の物語へ残る**回路が無い |
| **後継の質が次代の統治に効く（良い継承は安定、悪い継承は危機）** | 継承は所有権の移動。**指名の良し悪しが次代の安定度・正統性へ創発的に波及**しない |

**結論**：本作は当プロジェクトの継承システムに**「元首の主観」という人間スケールの顔**と、**①元首内面（死の自覚と遺産志向）②資質本位の後継指名 ③生前継承・摂政（引き際）④回顧＝統治評価が物語と正統性に残る**という4つの欠落軸を与える。法形式（#646）と組織存続（#812）の**間に空いていた「元首個人が後継を選び、引き継ぎ、去る」過程**を埋める。タイクン化を避け、すべて**高位の決断（誰を後継に指名するか・いつ譲るか）→エンジン→次代への創発帰結**に留める。

## 1. 役に立つ視点（要約）

本作の世界観・構造を、**本システムに効く形**で1行ずつ（既存モジュールへの共鳴を明記）：

1. **権力者にも内面がある＝死を自覚した元首は意思決定が変わる**。残された時間が短いほど「遺産」を優先する。→ `DynastyRules`/`Person`/`LifecycleRules` に**元首の主観状態**を足す欠落軸。
2. **後継は“継ぐ権利”でなく“ふさわしさ”で選ぶべきもの**。法順位の一位が無能なら国は傾く。→ `SuccessionLawRules`（#646）の法形式に**資質本位の指名スコア**を直交させる。
3. **血統外からの登用＝指名養子は、王朝の停滞を破る制度的バイパス**。→ 五賢帝型の養子継承＝`SuccessionRules`（#812 制度化）×`PersonRules`（適材適所 #866）の人事に接続。
4. **引き際こそ統治の最後の技術＝生前継承と摂政による滑らかな移譲**。→ `RetirementRules`（強制停年 #530）に**自発的・計画的な譲位**を additive で足す。
5. **回顧は自己評価であり、評価は国家に残る＝治世の総括が後継の正統性と物語になる**。→ `DisclosureLedger`（FND-4）×`DynastyRules`（正統性）への**lore＋正統性データ入力**。CCX-6（世界観codex退避）方針に一貫。
6. **継承の良し悪しは次代に効く＝良い指名は安定、悪い指名は継承危機**。→ `GovernanceRules`/`FactionState`（安定度・正統性）への**創発帰結**。`SuccessionLawRules.SuccessionCrisisRisk` を資質で変調。

## 2. 取り入れるべきメカニクス（優先度つき・既存への接続）

> 大原則：**継承法#646／組織存続#812／後任補充#152／停年#530／人物#866 を作り直さない**。HDRN はそれらに**「元首の主観」と「資質本位の指名・引き際」という欠落軸を足し、接続する**だけ（additive）。法形式は #646 のまま、HDRN は「誰を・いつ」の質を扱う。

### ★★★ 最優先（真の欠落・本作の signature）

#### HDRN 元首の主観状態（死の自覚・遺産志向＝Ruler Mindset）
- 元首（`Person`/`ICharacter`）に**主観状態**を持たせる：余命感（年齢・死亡率カーブ由来）／統治への倦怠／**遺産志向**（残り時間が短いほど高まる）。
- 余命感が高いほど意思決定の重みが「目先の拡張」から「**継承の準備・制度化投資**」へシフト＝`SuccessionRules.InvestInstitution`（#812）の動機づけ。
- 接続：新 `RulerMindsetRules`/`RulerMindset`（純ロジック・test-first）。入力は `LifecycleRules.AnnualMortality`（#152）＋`DynastyRules`（正統性/腐敗）。**基準能力は非破壊**（実効値パターン＝意思決定の重みだけを変える）。

#### HDRN 資質本位の後継指名（Designation by Merit／指名養子）
- **法順位（#646）とは別軸**で、候補者を**資質スコア**（`PersonRules` の軍才/文才・徳・元首との相性・年齢余命）で評価し、**最適な後継を指名**する。
- **指名養子**＝血統/役職tierの外から後継を引き上げる経路（五賢帝型）。`SuccessionLawRules.SuccessionLaw.指名` の中身を埋める。
- 接続：新 `SuccessorDesignationRules`（純ロジック・test-first）。`SuccessionLawRules`（法形式・#646）×`PersonRules`（適材適所・#866）×`SuccessionRules`（カリスマ継承・#812）。**指名そのものは作らず、指名の“質”を点数化**して既存継承へ渡す。

### ★★ 高（引き際・次代への帰結）

#### HDRN 生前継承・摂政・引き際（Voluntary Abdication & Regency）
- 元首が**自ら時期を選んで譲位**し、後継が未熟なら**摂政**で過渡期を橋渡しする。`RetirementRules`（強制停年 #530）に対する**自発的・計画的退場**。
- 早すぎる譲位＝経験不足、遅すぎる＝老衰・継承危機。**最適な引き際**を主観状態（HDRN-1）が駆動。
- 接続：`RetirementRules`/`ServiceStatus`（#530）に additive な「自発譲位」状態＋摂政の暫定権限（`GovernmentRegistry`/`OfficeRules` で空席を埋める既存経路を流用、新レジストリ非新設）。

#### HDRN 継承の質→次代の安定（Succession Quality Outcome）
- 指名スコア（HDRN-2）と引き際（HDRN-3）の良し悪しが**次代の正統性・安定度・継承危機リスク**へ創発的に波及。良い継承＝安定、悪い継承＝`SuccessionLawRules.SuccessionCrisisRisk` 上昇。
- 接続：`GovernanceRules`/`FactionState`（安定度・正統性）×`DynastyRules`（正統性）×`SuccessionLawRules`（危機リスク変調）。**新パラメータは作らず既存指標へ係数で効かせる**（タイクン回避）。

### ★ 中（回顧・世界観lore）

#### HDRN 治世の回顧と総括（Reign Retrospective＝評価が残る）
- 元首の治世を**回顧して総括スコア**（拡張/安定/制度化/正統性の達成度）を出し、それが**後継の初期正統性**と**国家の物語（lore）**に残る。
- 接続：`DynastyRules`（正統性）＋`DisclosureLedger`（FND-4）への**評価＋lore データ入力**（コード新設は集計のみ・物語は data 入力）。

#### HDRN（lore）世界観の開示データ
- 「権力には終わりがあり、引き際こそ統治の技術」「最良の後継は血でなく資質で選ばれる」「治世は死後に評価される」。
- 接続：**コード新設せず** `DisclosureLedger`（FND-4）への**lore データ入力**。CCX-6（世界観codex退避）方針に一貫。

### ❌ 不採用（重複・既存で十分）

| 不採用 | 理由 |
|---|---|
| 継承法そのもの（長子/分割/指名/選挙） | **`SuccessionLawRules`（PDX-1 #646）が既にカバー**。HDRN は法形式を作り直さず「指名の質」を直交させる |
| 死亡・年齢・没年カーブ | **`LifecycleRules`/`Calendar`（LIFE-1/2）がカバー**。HDRN-1 は入力として読むだけ |
| 役職の後任補充（空席埋め） | **`VacancyRules`（LIFE-2 #152）がカバー**。摂政は既存 `GovernmentRegistry`/`OfficeRules` を流用 |
| 英雄死後の組織存続・制度化投資 | **`SuccessionRules`/`Organization`（#812）がカバー**。HDRN は指名スコアを渡して接続するだけ |
| 政略結婚・請求権継承 | **`MarriageRules`（PDX-2 #647）がカバー**。HDRN は血統外指名（養子）に特化 |
| 強制停年・アップオアアウト | **`RetirementRules`（#530）がカバー**。HDRN-3 は「自発的譲位」だけを additive に足す |
| 一人称ナラティブ演出/テキスト生成 | ゲームメカニクスでなく演出。lore は `DisclosureLedger` のデータ入力に留める（新コードなし） |

### 推奨着手順
`HDRN-1`（元首の主観＝最も固有で欠落の大きい signature・他issueの駆動源）→ `HDRN-2`（資質本位の指名＝主観が選ぶ対象）→ `HDRN-3`（引き際＝主観が決める“いつ”）→ `HDRN-4`（次代への創発帰結＝盤面接続の本丸）→ `HDRN-5`（回顧＝正統性＋lore）→ `HDRN-6`（lore データ入力）。

> いずれも継承法#646・組織存続#812・停年#530 を**後退させず接続**する additive 設計。法形式（誰が継ぐ権利を持つか）と組織存続（カリスマの日常化）の**間に空いていた「元首個人が後継を選び、引き継ぎ、去る」過程**を埋める。
