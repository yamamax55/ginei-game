# 性別と性的指向 — 設計メモ

> 人物（`Person`）と POP（`Population`）の**性別**を定義・実装する。**性的指向**（ストレート/LGBTQ+）は
> 隠しパラメータ案として記録するが、**現状は検討項目＝未実装**（実装可否は作者判断で確定する）。

## 1. 性別（実装済み）

### データ
- **`Sex`(Core enum)**：`{ 男性, 女性 }`。
- **`Person.sex`**：個別の性別（既定=男性＝後方互換）。
- **`Population.femaleShare`**：POP のマクロな男女比（女性割合 0..1・既定0.5＝均衡）＋`Females`/`Males`。
- **`SexRules`(Core)**：`BalancedFemaleShare`(0.5)／`BalanceFactor(femaleShare)`（偏り→出生係数：均衡で1.0・完全偏りで`1-MaxSkewPenalty`＝極端に偏ると番が組みにくく出生が鈍るマクロ近似）／`ShareOf(sex, femaleShare)`。

### 配線
- `GalaxyView` が新任人材へ**所有勢力の政策男女比**で性別を割り当てる（デモ：帝国＝家父長的で女性少`ImperialFemaleShare`0.08／同盟＝平等で多め`AllianceFemaleShare`0.35＝銀英伝風の体制差）。
- `SystemView` が惑星の男女比を表示。
- `SexRules.BalanceFactor` は出生（`PopulationDynamicsRules`）へ掛けられる Core 関数（デモは均衡ゆえ実質無効＝将来の偏り発生時に効く）。

### 方針
- マクロ背景として効かせる（**タイクン化回避**）＝個別の交配・家系図を作らない。性別による能力差は付けない（差別的設計を避ける）。
- 体制差は「政策としての登用比率」で表現（能力ではなく機会の差）。

## 2. 性的指向（検討項目・未実装）

> **隠しパラメータ**として `Orientation`（ストレート/LGBTQ+ …）を持たせるか、を検討する。**現状は実装しない。**

### 想定する効き先（もし実装するなら）
- **関係性・社交**：社交場（#1145）・親愛/反目の関係グラフ・恋愛/伴侶。
- **政略結婚（#647 `MarriageRules`）**：同盟の縁組・請求権継承の相手選び。
- **継承（#646）**：実子の有無 → 後継問題（養子#812/HDRN との接続）。
- いずれも**戦闘・能力には一切影響させない**（隠しパラメータ＝物語の襞のみ）。

### 検討すべき論点（要・作者判断）
- **目的**：何のための変数か（物語の多様性・関係イベントの厚み）。ゲーム機構に必須ではない＝「あってもなくても戦略は回る」。
- **表現の配慮**：ステレオタイプ化・機構的ペナルティ（不利益化）を**設けない**。多様性を「設定の襞」として尊重し、揶揄・優劣化しない（世界観方針 #305/#15 に準じる）。
- **隠し方**：完全な隠しパラメータ（UI 非表示）か、関係イベントでのみ示唆されるか。
- **スコープ**：人物（ネームド）のみか、POP（集計の指向分布）まで持つか。POP まで持つとタイクン/PERF リスク＝持つなら集計スカラ1つに留める。
- **接続の最小性**：`MarriageRules`/社交#1145/継承#646 への**接続のみ**で新機構を増やさない（既存窓口を使う）。

### 現状の結論
- **検討項目（保留）**。性別（§1）は実装済み。性的指向は**作者が目的と表現方針を確定してから**、上記スコープ最小（ネームド人物の隠しパラメータ＋関係/結婚イベントへの接続のみ・戦闘無影響）で additive に入れる想定。それまでは未実装。
