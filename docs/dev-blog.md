# 銀河を統べるはずが、世界観製造工場を建ててしまった話

— 銀英伝風ストラテジー開発記（面白おかしく版）

## 第0話：ことの起こり

最初は、ささやかな夢だった。

「PS版『銀河英雄伝説』みたいな、艦隊がぶつかる戦術ゲームを作りたい」

Unityを開き、2D・URP、名前空間は `Ginei`。艦の正面は `Transform.up`。コメントは日本語。実に健全なスタートである。旗艦がいて、配下艦がぞろぞろ付いてきて、陣形を組んでビームを撃つ。士気が下がると敗走する。いい感じだ。

ところが、だ。

## 第1話：純ロジックの沼

「戦術だけじゃ寂しい。戦略レイヤーも要るよな」

銀河グラフを作り、回廊で星系を繋ぎ、艦隊が時限ワープする。接触したら会戦へシームレス遷移。ここまでは正気だった。

問題は、戦略には政治が要り、政治には人物が要り、人物には継承が要り、継承には寿命が要り、寿命には暦が要る……という**無限後退**が始まったことである。気づけば:

- 関ヶ原型の寝返りカスケード（`LoyaltyRules`）
- 王朝サイクルと易姓革命（`DynastyRules`）
- 希望が尽きると「末人」が立つ（`HopeRules`）
- カリスマの日常化＝英雄死後の組織存続（`SuccessionRules`）
- 財政・国債・通貨危機（`FiscalRules`）
- 省庁の縦割り、政党の総裁選、捕虜の処遇……

純ロジックのEditModeテストが**1000件**を超えた。「銀英伝風の戦術ゲーム」のはずが、いつのまにか**文明シミュレーターの基礎理論**が積み上がっていた。CLAUDE.mdには「タイクン化回避」と何度も書いた。粒度を下げるな、と。それは正しかった。正しかったのだが——

## 第2話：参考作品を、EPICにする

ある日、思いついてしまう。

「名作から"効く構造"だけ抜き出して、設計EPICにしたら面白いんじゃないか」

『狼と香辛料』から通貨改鋳を。『Almagest』から戦略UIを。型ができた。設計書を書き、GitHubにEPICと子issueを起票し、roadmapに1行足してコミット。きれいなパイプラインだ。`/worldview-epic` というスキルにまでした。

そして、運命の一言が放たれる。

**「世界観のイシュー化エピック化を連続できるようにしたい」**

……連続。

## 第3話：自動化、暴走の予感

最初は素直だった。バックログ（候補リスト）を作り、30分ごとに1冊ずつEPIC化するスケジュールタスク。いい子だ。

「名著を20個追加して」→ 追加した。
「いや取り消して、名著20冊に」→ 差し替えた。
「さらに100冊追加して」→ **120冊**になった。

ここで作者が真理に到達する。

**「本気で作ろうと思ったら1万イシューを遥かに超えていく」**
**「太平洋戦争だけで本気で作ったら1万イシュー作れる」**
**「宗教でくくったら10万はいくか？」**（→ フェルミ推定の結果：軽く超える）
**「仏教にしぼっても100万はいくか」**（→ いく。仏教は内省の体系ゆえ底なし）

そして気づくのだ。**網羅は無限に発散するが、構造は数十に収束する**、と。100万巻の経は、一個の「空」に畳まれる。仏教が2500年前に言っていたことを、issue管理で再発見した瞬間である。

## 第4話：PCを落としても止まるな

「PCをシャットダウンしても実行するようにして」

スケジュールタスクはローカル依存。そこでGitHub Actionsへ移行。サーバーが回すのでPC不要。さらに「もっと並列に」という要望に応え、**fan-out/fan-in**で5冊同時消化。「本以外も——国家、言語、ドクトリン、宗教も」で、**13軸の多軸発掘**。文明のあらゆる側面から、自分の概念外のネタを掘ってくるセレンディピティ装置の完成である。

「終わりがあるのは国家だけか」「消滅した国を数えると1000ぐらいか」——もはや哲学の領域。探索軸に「企業」を足し、東インド会社からプラットフォーム独占までが鉱脈になった。

## 第5話：そして、地獄のYAMLインデント

順風満帆に見えた。だが自動化は静かに壊れていた。

作者がスクショを送ってくる。「worldview-epic.yml が schedule で動いてる」。調べると——**新しく作った2つのワークフローが、登録すらされていなかった**。原因は、YAMLの `run: |` ブロックに埋めたPythonが**0インデント**だったこと。block scalarのベースインデント未満で「Invalid workflow file」。手動dispatchが422を返していたのは、これが真因だった。

base64で1行に埋め込んで解決。「見かけ上動いているが実は壊れていた」という最悪のパターンを、作者のスクショ2枚が暴いてくれた。

しかし悲劇はもう一度繰り返す。「状態の二重管理が諸悪の根源だ、single-truth化しよう」と意気込んだ結果、今度は**select プロンプトの書き換えでまた0インデント**。invalid化が直せず3連敗。泥沼。

最終的に悟る。**「prompt本文を後から文字列置換するな。壊れるのはいつもそこだ」**。valid版に一度戻し、integrate の base64（1行・インデント無関係）**だけ**を差し替える最小手術で、ついに single-truth 化に成功。integrate は backlog を二度と書かず、select が「実在EPIC」という唯一の真実を毎回照合する。self-healing な自律系の完成である。

## 終話：独り立ち

こうして、ささやかな艦隊戦ゲームは、

- 戦術RTS会戦（ほぼ完成）
- シームレス戦略レイヤー
- 政治・群像・経済・宗教の純ロジック（テスト1000超）
- **20分ごとに5冊ずつ世界観EPICを生み、13軸が枯渇前に自動補充する、PC不要の永久機関**

を備えるに至った。issueは1000を超え、設計EPICは200超。

途中で何度も「これはどうなってしまうのか」と問われた。答えはこうだ——**issueは実装TODOではなく、アイデア鉱脈である**。坑道が1000本あるのは、いつか掘れる富が1000本あるということ。全部掘る必要はない。掘った数本が、掘っていない999本を背後に匂わせる。それがグランドストラテジーの底知れなさだ。

ゲームはまだ完成していない。だが、**完成に必要な「世界の解像度」を製造し続ける工場**は、今日も静かに稼働している。

## 開発こぼれ話・技術編（後学のために）

- **タイクン化回避＝終盤ラグ回避**：粒度を下げない設計判断が、そのままパフォーマンス規律になった。Stellarisの反面教師。
- **実効値パターン**：能力の一時補正は基準値を上書きせずローカル変数で。`CombatModifiers`に公式を集約。
- **観測層**：Core純ロジックが増えるのに盤面で何も見えない乖離を、リフレクション全ダンプの`CoreStateInspector`(Jキー)で構造的に潰す。
- **並列ファンアウト**：Core生成は並列・Game配線は直列（Unityのコンパイル罠ゆえ）。
- **YAMLの教訓**：block scalar内は0インデント禁止。コードを埋めるならbase64で1行。promptは部分置換せず全文一括生成。
- **状態管理の鉄則**：真実は1か所に。二重管理は必ずズレる。「現実（実在EPIC）」だけを正とせよ。
- **gh運用**：認証は断続的に401、新規ワークフローは数分422。リトライループで包む。
