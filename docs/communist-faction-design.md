# 共産勢力の非対称設計（Issue #17）

> 共産主義圏を**他勢力と非対称**に設計する：独自の能力体系・艦・思想、そして穏健派／強硬派の内部対立。
> 前提：#16（FactionData：色/思想/階級ranks/敵対関係）、#15（世界観バイブル §2-3）、RankSystem。
> **設計原則：戦闘エンジンをフォークしない。** 非対称性は「データ＋実効値パターンの小さなフック」で表現し、
> 他勢力の挙動は既定値で一切変えない（back-compat）。創作の核なので **【要・作者判断】** を明示する。

## 0. 確認したい論点（方向性・推奨デフォルトで本書は進める）
1. **非対称の主軸**：推奨＝「**物量（人海/計画経済）× 政治将校の士気底上げ × 不退転**」。質より量・崩れにくいが硬直。…採用してよい？
2. **内部対立の表現粒度**：推奨＝**1勢力内の“派閥スタンス”**（穏健/強硬）を提督・状況で持たせる軽量版。将来 #14 政治で「強硬派の分離独立（別 FactionData）」へ拡張可能にする。…軽量版から始めてよい？
3. **「独自能力体系」の意味**：推奨＝**6能力(統率/攻撃/防御/機動/運営/情報)は共有のまま、共産だけ“効き方”を変える**（運営=計画経済が量・士気に効く等）。新ステータスは足さない。…この解釈でよい？

> 上記すべて「推奨で進める」前提で本書を書く。違えば指摘で差し替える。

## 1. 思想と党構造
- **思想**：党による計画と集団主義。個より全体。解放区の住民を組織化し、**量**で戦線を支える（兵站 L-4 と強く結びつく）。
- **階級**：#16 で党称号（准司令官/司令官/上級司令官/軍司令官/総司令官）を tier 整合で設定済み。RankSystem でそのまま同列比較・昇進が効く。
- **派閥**：**穏健派**（消耗を避け戦線維持・交渉余地）と **強硬派**（不退転・粛清的規律）。平時は一枚岩、戦況で対立が表面化（§5）。

## 2. 非対称の3本柱（データ＋最小フックで実装）
### 2-1. 物量ドクトリン（人海／計画経済）
- **多数・低質**：1部隊あたりの配下艦数を増やし（`Squadron.escortCount` を勢力倍率で上げる）、個艦の質（提督攻撃/防御）は控えめ。総兵力で押す。
- **再生産が速い**：兵站 L-4 の徴募コストが低く回復が速い（量を維持できる）。前線損耗を後方人口で埋める。
- **トレードオフ**：個々の機動・精度が低い＝**側背面に弱い／陣形変更が遅い**（硬直）。

### 2-2. 政治将校（コミッサール）＝士気底上げ
- **崩れにくい**：士気に**下限（floor）**を設け、敗走（rout）しにくい。`FleetMorale.GetMoraleFactor()`/`IsRouted` が下限を尊重する（実効値パターン：基準は変えず下限でクランプ）。
- **代償（非対称コスト）**：いざ下限を割って崩れると**雪崩（カタストロフ）**＝復帰が遅い／周囲も崩れやすい。あるいは「政治将校が健在な間だけ floor が効く」とし、旗艦撃破で floor 消失。
- **【要・作者判断】**：floor の値（例 0.6 倍で下げ止まり）、崩壊時ペナルティの重さ。

### 2-3. 不退転（強硬派ドクトリン）
- **退却しない**：強硬派の艦隊は `FleetAI` の撤退閾値を実質 0 にし、**全滅まで戦う**（`FleetStrength.BeginRetreat` を選ばない＝兵力0で消滅）。崩しにくいが、引き際が無く損耗も大きい。
- **穏健派は通常**：撤退・補給線維持を選ぶ（既存 AI のまま）。

## 3. 能力体系の非対称（6能力の“効き方”を変える）
> 新ステータスは足さず、共産勢力でのみ既存6能力の効果配分を変える（読み手＝実効値の解釈を勢力で分岐）。
- **運営（operation）が主役化**：他勢力では将来用だが、共産では **operation が「計画経済ボーナス」**として量（配下艦数）・徴募速度・士気floorに寄与。→ 運営型の提督が強い。
- **統率（leadership）＝集団指揮**：大部隊運用の上限（兵力）に強く効く（既存 maxStrength 補正を勢力係数で増幅）。
- **機動・情報は控えめ**：硬直ドクトリンの表現（個艦の小回り・索敵が弱い）。
- 提督ロスター（#18 の同志ヴォルコフ/ライサ）は **運営・統率高め／機動・情報低め** に作る（既に近い配分）。

## 4. 艦・命名・ビジュアル
- **色**：橙赤（#16 で設定済み）。艦体は赤系、量産型の没個性なシルエット（ビジュアルのみ・スプライト差し替え/色味）。
- **旗艦名**：理念・労農・赤系（《赤い暁》《プロレタリ》《十月》…バイブル §3）。
- **マーカー/ハロー**：既存 `FlagshipMarker`（金ダイヤ）＋陣営色ハロー（橙赤）で他勢力と同じ識別系に乗る（特別扱いしない＝実装簡素）。

## 5. 内部対立（穏健派／強硬派）のゲーム表現
- **軽量版（まず実装）**：提督 or 艦隊に **スタンス（穏健/強硬）** を持たせ、上記 2-2/2-3 の効き方を切替（強硬＝不退転・規律強、穏健＝通常・柔軟）。シナリオで混在可。
- **政治連動（将来 #14）**：戦況・merit で党内バランスが動き、強硬派優勢だと「**不退転が全軍に強制**」、穏健派優勢だと「補給優先・撤退許容」。極端化すると **強硬派が分離独立**（新 FactionData を `nonHostileFactions` 操作で敵対化）＝内戦フック。
- **【要・作者判断】**：スタンスをどこに持たせるか（`AdmiralData` のフラグ／シナリオ指定／党内メータ）。

## 6. 実装フック（最小・back-compat）
> いずれも「未設定なら従来どおり」。他勢力は既定値で無影響。実効値パターン（基準は非破壊）を厳守。
| 追加/接続 | 内容 |
|---|---|
| `FactionData` に**ドクトリン項目**（任意） | `squadronSizeMultiplier`(物量)／`moraleFloor`(政治将校)／`noRetreat`(不退転)／`operationDoctrineWeight`(計画経済の効き)。既定は中立値＝従来動作。 |
| `Squadron` 生成（`BattleSetup`） | 勢力の `squadronSizeMultiplier` で `escortCount` を補正（root スケールは変えない規約を厳守）。 |
| `FleetMorale.GetMoraleFactor()`/`IsRouted` | 勢力の `moraleFloor` で下限クランプ（floor 未設定＝0＝従来）。政治将校（旗艦健在）条件を付けるなら旗艦生存を見る。 |
| `FleetAI`（撤退判断） | 勢力/スタンスが `noRetreat` なら撤退状態に入らない。 |
| 能力の効き（`FleetStrength.ApplyAdmiralData` 等） | 共産は `operation`/`leadership` を量・士気・徴募へ寄与させる係数を勢力から読む。 |
| 兵站 L-1/L-4 | 徴募コスト/速度・人口産出に物量ドクトリンを反映（`docs/wartime-logistics-design.md` と接続）。 |

> ※ これらは **FactionData の任意フィールド追加**が中心で、コードは「読み手が勢力係数を掛ける」数行ずつ。エンジン分岐や新クラスの大量追加はしない。

## 7. 段階手順・リスク
1. **データ先行**：`FactionData` にドクトリン項目を追加（既定＝中立）。Communist アセットに値を設定。**他勢力は無影響**をまず確認。
2. **物量**：`BattleSetup` で `squadronSizeMultiplier` を反映（量で押す体感）。
3. **政治将校**：`FleetMorale` の floor。崩壊時ペナルティ。
4. **不退転**：`FleetAI` の撤退抑止（強硬スタンス）。
5. **能力の効き**：operation/leadership の共産係数。
6. **内部対立**：スタンス切替（軽量版）→（将来 #14）党内メータ・分離独立。

- **リスク**：非対称が「ただ強い/弱い」に偏らないようバランス必須（量↔硬直、崩れにくい↔崩れると雪崩、で相殺）。
- **リスク**：フックが増えると勢力分岐が散る → **FactionData のドクトリン項目を唯一の出所**にし、読み手は係数を掛けるだけに統一（敵対判定を `FactionRelations` に集約したのと同じ思想）。
- **【要・作者判断】**：各係数の値、政治将校の条件、スタンスの持たせ方、分離独立の発火条件。
