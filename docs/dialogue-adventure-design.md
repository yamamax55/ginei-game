# 社交場の対話アドベンチャー 実装設計（EPIC #DLG・SLN の上に乗る）

> 親系譜：社交場 深堀りEPIC #1145（[`social-venue-design.md`](./social-venue-design.md)）。SLN/TALE で「対話アドベンチャー化」は**タイクン化を理由に不採用**としていたが、**方針転換＝社交場ではプレイヤー向けの対話シーンを実装する**（作者決定 2026-06-11）。
> タイクン化回避との両立は**「観測解像度とシミュ解像度の分離」（PERF #1117 原則5）**で解決する：**プレイヤーが出席した社交場だけ ADV シーン化**し、非出席・AI同士の社交は従来どおり高位状態から自動解決（SLN）。＝対話は「全社交の逐次マイクロ管理」ではなく「観たい場面だけ降りて演じる」二層遷移 #586 の社交版。
> 著作権注意：固有名・文章・キャラクターは流用せず、構造パターンのみ参考。

## 0. 位置づけ — なぜ対話アドベンチャーか・なぜ今なら成立するか

**SLN/TALE 時点の不採用理由**は「会話の逐次選択肢＝タイクン化/別ジャンル化」。これは「**全ての**社交を手動で回す」前提での判断だった。本EPICは前提を変える：

| 懸念（旧不採用理由） | 本設計での解決 |
|---|---|
| 全社交の逐次マイクロ管理＝タイクン化 | **プレイヤー出席時のみ** ADV 化（DLG-4）。非出席は SLN の自動解決＝会戦の「潜行/自動解決」二層 #586 と同型 |
| 別ジャンル化（ADVがゲームの主役になる） | 対話は**既存メカニクスの操作面**＝選択肢の効果は全て既存窓口（関係値/調略#819/縁談#647/噂SAW-3/旗幟#817）へ流す。新しい勝利条件や独立進行は作らない |
| 挿話は高位状態から創発（TALE-2） | **共存**＝挿話生成（自動・全人物対象）はそのまま。ADV は「プレイヤーがその場に居るときの一人称体験」を足すだけ |

**得られるもの**：銀英伝の「夜会の毒杯」「敵将との対面」「酒場の密談」を**プレイヤーが演じられる**。SLN が用意した舞台（場の類型・参加資格・関係グラフ）の上に、操作可能な一人称の社交が乗る＝**戦闘RTS（会戦）と対になる「社交ADV（夜会）」**。

## 1. 設計方針（大原則）

- **SLN-1〜3 が前提**（関係グラフ・場の類型・参加資格）。SLN を置換せず、その上に乗る。
- **会話の骨格は決定論・データ駆動**：ノードグラフ＋条件＋効果（`EventEngine`#116 の多段会話版）。純ロジックは Core で test-first。
- **効果は既存窓口へ**：選択肢の帰結は `RelationshipGraph`（SLN-1）/`LoyaltyRules.ApplyIntrigue`（#819）/`MarriageRules`（#647）/噂（SLN-5）/`Allegiance`（#817）に**接続のみ**。基準値非破壊（実効値パターン）。
- **観測とシミュの分離（PERF原則5）**：ADV はプレイヤー出席時のみ。AI同士・非出席の社交は SLN の集約解決。対話シーン中は `Time.timeScale=0`（`FleetDetailPanel` 同型）。
- **UI は UITK**（`GineiUITK`・コード生成・見切れ防止の段階移行方針に従う）。
- **LLM は任意の肉付け層**：台詞の文面はテンプレート（決定論）が正、ローカルLLM（#909〜915）があれば言い換えで豊かに（LLM-2 #911 の入出力契約に相乗り）。**LLM 無しでも完全動作**（フォールバック＝テンプレート）。

## 2. メカニクス（優先度つき・既存への接続）

### ★★★ 核（縦に1本通す）

#### DLG 会話データモデル（ノードグラフ）
- `DialogueScene`（1会話）＝`DialogueNode` の有向グラフ。ノード＝発話（話者/テキスト or テンプレートID）＋選択肢（`DialogueChoice`=ラベル/条件/効果/次ノード）。分岐・合流・終了。
- 解決は新 `DialogueRules`（static・Core・test-first）：`Advance`(現在ノード＋選択→次)/`AvailableChoices`(条件評価)/`ApplyEffects`。**決定論**（roll 引数）。
- 接続：`EventEngine`#116 を**置換しない**＝単発の条件発火イベントは従来どおり、多段の会話だけ DLG。`EventContext` を共有。

#### DLG 文脈条件と選択肢ゲート
- 選択肢の可視/可否を**文脈**で決める：相手との関係値（SLN-1）・自他の階級#14（目上に言えない台詞）・思想・場の類型（夜会では言えるが軍議では言えない）・噂の保有（SLN-5＝知っていれば選べる）。
- **無礼な選択肢**は隠さず「選べるが代償がある」＝面子の事件（SLN-4）として関係値/評判に跳ねる。
- 接続：`RelationshipGraph`・`RankSystem`・SLN-2 類型・SLN-5 噂。

#### DLG 効果の接続（選択の帰結は既存メカニクスへ）
- 選択→ ①関係値の増減（SLN-1）②調略の試行/抵抗（`ApplyIntrigue`#819・社交接触前提 SLN-6 と一体）③縁談の打診（#647）④噂の入手/流布（SLN-5）⑤旗幟の揺さぶり（#817）⑥情報開示（`DisclosureLedger`）。
- **新しい数値系を作らない**：全て既存窓口の呼び出し。会話は「既存メカニクスの操作面」。

#### DLG 出席と機会生成（タイクン化回避の核）
- 社交場の開催（SLN-2 契機）→ プレイヤー勢力の主人公/元首が**出席を選んだ場合のみ**、参加者（SLN-3）から会話相手と `DialogueScene` を決定論生成して ADV へ。
- **非出席・AI同士は従来の自動解決**（SLN の関係値更新のみ）。出席はコスト/機会（出れば調略の機会、出れば調略される危険）＝高位の決断。
- 接続：SLN-2/3・主人公#735（`ProtagonistRules`）・二層遷移#586 の思想。

### ★★ 体験（UI・声）

#### DLG ADV シーン UI（UITK）
- ポートレート（無ければ陣営色シルエット）＋名前/階級＋テキスト＋選択肢ボタン。**テキスト送り/スキップ/オート/バックログ**（ADV標準QoL）。表示中 `Time.timeScale=0`、Esc で退出（`PauseManager` 譲り＝既存Escチェーンに従う）。
- 接続：`GineiUITK`（`OrderOfBattlePanel` パイロットの型）・`FontProvider`/JapaneseFont_TMP。

#### DLG voice×台詞テンプレート
- 台詞は**テンプレートID＋文脈差し込み**（相手名/階級名/戦況）。人物の voice 属性（TALE-6 #1088：高雅/粗野/雄弁/皮肉）で語尾・言い回しを決定論に変える＝同じ意味でも人物で台詞が違う。
- 接続：TALE-6・`AdmiralData`（#523 呼称/異名）・`RankSystem.ResolveRankName`（#14）。

### ★ 拡張（任意）

#### DLG ローカルLLM 肉付け（任意・フォールバック必須）
- テンプレート台詞を**ローカルLLM が人物文脈で言い換え**（LLM-2 #911 の構造化状態→プロンプト契約に相乗り）。選択肢・効果・分岐は**ルールが決める**（LLM は文面のみ＝ゲーム状態を触らない）。LLM 無し環境ではテンプレート素通し。
- 接続：#909〜915（ルール＝土台/LLM＝思考層の方針そのまま）。

#### DLG サンプル会話＋lore 蓄積
- サンプル `DialogueScene` を3本（夜会の密談/捕虜交換の敵将対面/士官クラブの派閥誘い）＝エディタメニューでワンクリック生成（`SampleScenarioCreator` の型）。重要会話の結末は列伝#784/`DisclosureLedger` へ記録。
- 接続：TALE-2/7・殿堂#785。

### ❌ 不採用（本EPICでもやらないこと）

| 不採用 | 理由 |
|---|---|
| 全社交の手動会話（出席の強制） | タイクン化。出席は任意・非出席は自動解決（DLG-4 が核） |
| 好感度ギフト・周回デート・恋愛シミュ化 | 別ジャンル。関係値は社交・事件・恩義から動く（SLN 方針踏襲） |
| LLM にゲーム状態を変えさせる | ルール＝土台/LLM＝文面のみ（#915 フォールバック方針） |
| ボイス・立ち絵の大量アセット前提 | コード生成・シルエットで成立させる（アセットは後から差し替え可能な設計） |
| `EventEngine` の置換 | 単発イベントは #116 のまま。DLG は多段会話だけ |
