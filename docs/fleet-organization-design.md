# 艦隊編制基盤 設計（艦隊番号・提督配属・永久欠番／軍団梯団・直轄部隊・任務戦術）

> 対応 Issue：**EPIC #148**（艦隊編制基盤）＝ **#146 フリートナンバー** ＋ **#147 軍団システム基盤**。
> 関連：政治/階級 #14（`RankSystem`）・政府役職 #141／文民統制 #145・係数 #106・SaveData #108・イベント #116・創発 #117・戦略マップ #33/#34・提督スキル #137系・世界観バイブル #15。
> 本書は「なぜ・何を・どう既存に接続するか」を示す設計メモ。数値・裁定は各 Issue の【要・作者判断】に従う。

---

## 0. 狙いと一言まとめ
- 現状は「**提督 ＝ 艦隊**」が一体（`AdmiralData` が艦隊の名前・兵力の出所）。これを **「艦隊（番号付き常設の器）」と「提督（配属される人）」に分離**する。
- その上に **梯団（軍団 → 軍集団）** を載せ、**作戦ごとに配下を着脱するモジュール型編制**＋**直轄部隊の集中投資**＋**任務戦術（目的だけ与え手段は現場裁量）**を表す。
- 顕著な功績を上げた艦隊番号は **永久欠番**（メジャーリーグの背番号引退方式）。

```
軍集団 ⊃ 軍団 ⊃ 艦隊(FleetUnitData) ⊃ 旗艦＋配下艦（既存ランタイム）
            └ 直轄部隊(directAssets)            └ FleetStrength / EscortShip
```

---

## 1. ★最重要：名前の衝突を避ける（FleetRegistry とは別物）
- 既存 **`FleetRegistry`** は「**会戦中の艦(`IShipTarget`)在庫**」＝ランタイムの揮発リスト（出現で Register／破棄・退却で Unregister）。
- 本書の編制台帳は「**永続的なオーダー・オブ・バトル（編制ツリー）**」＝勢力の常設組織。**まったく別物**。
- → 新規レジストリは **`OrderOfBattle`**（仮称、`FleetRoster` を内包/拡張）とし、**`FleetRegistry` を流用・拡張・改名しない**。両者の責務を混ぜない。

| | `FleetRegistry`（既存） | `OrderOfBattle`（本書・新規） |
|---|---|---|
| 性質 | ランタイム・揮発 | 永続・セーブ対象 |
| 中身 | 会戦中の個艦（旗艦＋配下艦） | 番号付き艦隊・梯団・配属・欠番 |
| 寿命 | 会戦ごとにクリア | キャンペーン/セーブを跨ぐ |
| 窓口 | 索敵・勝敗カウント | 編成・任免・梯団・永久欠番 |

## 2. #146 フリートナンバー（艦隊番号）

### 2-1. データ
- **`FleetUnitData`（ScriptableObject・永続）**：1個の艦隊ユニット。
  - `fleetNumber`（int・勢力内で一意。例 13）／`fleetName`（固有名。無ければ「第N艦隊」を自動表示）
  - `faction`（`FactionData`。**番号体系は勢力ごとに独立**＝帝国の第1艦隊と同盟の第1艦隊は別系列）
  - `assignedAdmiral`（`AdmiralData`・配属中の指揮官。null＝空席）
  - `baseStrength`/`formation` 等の編制既定（**段階移行**：当面は提督側のまま番号だけ付与でも可。最終的にユニット側へ寄せる）
  - `meritScore`（功績の累積。永久欠番判定に使用。#14 の merit と連動）
  - `status`（`enum FleetStatus { 現役, 解隊, 永久欠番 }`）
- **`FleetRoster`（勢力ごとの艦隊台帳）**：`OrderOfBattle` の一部。
  - `GetFleet(faction, number)` / `NextAvailableNumber(faction)` / `Retire(faction, number)` / `AllFleets(faction)`。

### 2-2. 提督の配属（FN-2）
- `AssignAdmiral(unit, admiral)` / `Unassign(unit)` / `ReassignAdmiral(...)`。戦死・更迭で空席化。
- **階級ゲート（#14 `RankSystem`）**：規模に応じた最低 tier を要求（例：正規艦隊＝中将級、分艦隊＝准将級）。判定は **既存 `RankSystem` 窓口に委譲**（重複実装しない）。
- **任免権限は #141/#145**：艦隊司令の任免は軍務の人事。「誰が任免できるか」は **#145 文民統制型**で決まる（民主派＝軍務大臣承認、軍閥＝軍が自前、王党派＝君主）。**配属の実体は Roster、権限判定は #141/#145**。

### 2-3. 永久欠番（FN-3）
- `meritScore` しきい値到達／英雄的戦死／規定戦果で `status=永久欠番` にし、**以後その番号を払い出さない**。
- **発火は #116 イベント**（「第N艦隊、永久欠番」を記録イベント化）。功績判定は #14 merit／戦果、係数は #106。
- **二系統**：(a) キャンペーンで創発的に欠番化、(b) **世界観バイブル #15 が史実的に最初から欠番/固有名**指定（プリセット）。
- 歴代司令・顕彰の **履歴を記録**（結果画面・年表で表示）。

## 3. #147 軍団システム（梯団編制）

### 3-1. 階層
- **軍集団 ＞ 軍団 ＞ 艦隊**（将来 `軍 Armee` を挟める拡張余地）。実史対応＝艦隊≈師団（基本戦闘単位）、軍団＝Korps、軍集団＝Heeresgruppe。
- **ファンアウト目安：1軍団＝2〜5個艦隊**（実史の軍団＝2〜5個師団に倣う既定の目安。固定強制ではない）。

### 3-2. ナチス軍団制から採る「3つの核心」（構造・運用のみ。政治的含意は持ち込まない）
1. **① 司令部は固定・中身は流動（モジュール型）**
   - 軍団司令部は特定艦隊を恒久お抱えにしない。作戦目的・戦況で配下艦隊を**短期で入れ替え**（突破時は機動艦隊を集中、防衛時は重防御艦隊へ差し替え）。
   - データ：番号付き常設の器＋柔軟な隷属替え（**attach/detach**、`attachedTemporarily` で恒常編制と区別）。
2. **② 軍団直轄部隊（Korpstruppen）＝共有アセットの集中投資**
   - 司令部が**自前の強力な直轄戦力**を持つ（実史：重砲兵・架橋工兵・通信・独立重戦車）。本作翻案＝**独立重砲艦隊**／**工兵・架橋艦**（回廊啓開・要塞攻略支援＝#131系）／**通信艦**（索敵・指揮範囲拡張）／**親衛・特務艦隊**（精鋭打撃）。
   - 運用＝**決勝点への集中投資**：軍団長が「いま一番がんばっている配下艦隊」へ直轄戦力を一時配属（attach の特殊形）。効果は **#106 係数**で対象艦隊に支援補正として乗る。
   - **タイクン化回避**：直轄は**固定アセットの再配分**であって建設・増産対象にしない。
3. **③ 任務戦術（Auftragstaktik）＝目的を与え、手段は現場裁量**
   - 命令はミクロ手順でなく**任務 intent**（突破／包囲／防衛／遅滞／待機）。実行は**配下 `FleetAI` の裁量**に委ねる（既存の接近/交戦/撤退ロジックを intent で方向づけ）。
   - 委譲の巧拙は**司令官能力**（`operation`/`intelligence` 実効値）＋**提督スキル #137系**で差別化（優秀なほど放任でも意図どおり動く）。係数は #106。

### 3-3. データ
- **`MilitaryFormationData`（ScriptableObject・永続）**：1梯団。
  - `echelon`（`enum FormationEchelon { 艦隊, 軍団, 軍集団 }`・将来 `軍` 拡張）
  - `formationNumber`（勢力内一意）／`formationName`（固有名）／`faction`
  - `commander`（`AdmiralData`・梯団司令官。null＝空席）
  - `children`（**下位梯団 or `FleetUnitData` のリスト**・再帰。最下位の子＝艦隊ユニット）
  - `attachedTemporarily`（作戦中だけ隷属する子。①）
  - `directAssets`（軍団直轄部隊。②）
  - `currentMission`（任務 intent。③）
- **`OrderOfBattle`（勢力別の編制ツリー窓口）**：`FleetRoster` を木構造へ拡張。
  - `GetFormation(faction, echelon, number)` / `Attach(child, parent)` / `Detach(child)` / `CommanderChain(unit)`（艦隊→軍団→軍集団を辿る）/ `Subtree(formation)`（配下の全艦隊列挙）
  - `ConcentrateAsset(asset, targetChildFleet)`（②集中投資＝一時 attach＋支援係数）
  - `IssueMission(formation, intent)`（③任務下達＝配下 `FleetAI` へ intent 配布）

### 3-4. 指揮系統・階級
- 梯団別の最低 tier：艦隊司令＝中将級／軍団司令＝大将級／軍集団司令＝元帥級（数値は要・作者判断。帝国の上級大将 tier9 の扱いも）。判定は `RankSystem`。
- 上位司令官の能力は配下へ**指揮補正**として波及しうる（#106 係数・実効値パターン・基準値非破壊）。**過度な数値積み上げは避ける**。

## 4. 既存システムへの接続
- **#14 `RankSystem`**：配属/任命の tier ゲート（艦隊/軍団/軍集団）。merit は永久欠番判定に流用。
- **#141 政府役職 / #145 文民統制**：司令官（艦隊/軍団/軍集団）の**任免権限**を政体で切替。編制の実体は `OrderOfBattle`、権限判定は #141/#145。文民統制が弱い/揺らぐと軍の独走→#116/#117 へ。
- **#106 係数パイプライン**：指揮補正・②集中投資・③委譲の巧拙をすべて係数で集約（直書き禁止）。
- **#137系 提督スキル**：③裁量委譲・②集中投資の効果を司令官の個性で差別化。
- **#33/#34 戦略マップ**：軍団を1編隊として移動・展開。命令は**任務の下達＋裁量委譲**でカスケード。会戦解決は配下艦隊の集合で。
- **`ScenarioData`**：`FleetEntry` に `fleetNumber`／梯団参照を**任意**追加。`BattleSetup` が解決し「第N艦隊（提督名）」で生成。**未指定なら従来どおり提督名のみで動作（後方互換）**。
- **HUD/結果（`FleetHUDManager`/`ResultManager`）**：「第3軍団（○○元帥）／隷下：第13艦隊…／直轄：重砲艦隊／任務：包囲」のツリー表示。永久欠番は顕彰表示。
- **#108 SaveData**：番号・配属・恒常/臨時隷属・直轄・任務・永久欠番・履歴を schemaVersion 付きで永続化。
- **#117 創発キャンペーン**：キャンペーンでは編制が動的（新設・改編・解隊・欠番）。シナリオモードはプリセット。**両モード併存**（`GameSettings.gameMode`）。
- **#15 世界観バイブル**：勢力別の番号体系・著名艦隊（第13艦隊＝ヤン）・既定の永久欠番の**正典の出所**。サンプル生成（`RosterCreator` 等）が参照。

## 5. 段階実装（推奨ビルド順）
1. **FN-1**：`FleetUnitData` ＋ `FleetRoster`（番号体系・勢力別・払い出し）。HUD に「第N艦隊」表示。**`FleetRegistry` と別物**として最小実装。番号未設定の既存シナリオは従来動作で回帰確認。
2. **FN-2**：提督配属（#14 tier ゲート）。`ScenarioData` で番号付き編成 → 会戦で表示。
3. **CORPS（#147 ①）**：`MilitaryFormationData` ＋ `OrderOfBattle` 木構造。attach/detach。`CommanderChain`/`Subtree`。
4. **CORPS ②③**：`directAssets`＋`ConcentrateAsset`（集中投資）／`currentMission`＋`IssueMission`（任務戦術→`FleetAI`）。
5. **FN-3 永久欠番**：merit しきい値→#116 イベント→`Retire`。#15 のプリセット欠番読込。
6. **永続化 #108**：編制ツリーをセーブ。**創発 #117** で動的改編に接続。

> 各段でコンパイル＆既存単体会戦の無事を確認してから次へ（CLAUDE.md 運用方針）。番号/梯団は常に**任意**で、未設定なら現行どおり動くこと（後方互換）を回帰の基準にする。

## 6. 【要・作者判断】（数値・裁定）
- ユニット移管の刻み：`baseStrength`/`formation` を `AdmiralData`→`FleetUnitData` へ寄せる範囲と時期。
- 永久欠番の判定基準（merit しきい値／英雄的戦死／作者指定の重みづけ）。
- 梯団の段数（3段で十分か `軍` を挟むか・勢力で段数が違ってよいか）。1軍団のファンアウト（2〜5）。
- 梯団別の必要 tier（中将/大将/上級大将/元帥の境界。帝国 tier9 の扱い）。
- **②直轄部隊**のラインナップ（重砲/工兵架橋/通信/特務打撃…どれを実装するか）と効果強度。直轄も艦隊ユニット(#146)として番号管理するか別カテゴリか。
- **③任務 intent** の種別セット（突破/包囲/防衛/遅滞…の粒度）と `FleetAI` へ渡す裁量度（完全自律〜半自動）。
- 上位司令官の指揮補正の有無・強度（タイクン化回避と両立する範囲）。
- 艦隊司令の任免を #141 の役職(Office)として扱うか、軍編制の別軸とするか（#145 文民統制と整合）。
