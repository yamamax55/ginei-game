# 戦時兵站レイヤー 設計（L-1 / L-2 / L-3 / L-4）

> EPIC #92。Railway Empire 的な物流を「平時のタイクン経営」ではなく **戦時兵站** として導入する＝**C-6（補給）の拡張**。
> 回廊網を輸送路とし、戦闘が輸送路を奪い合う。子Issue：L-1 #93 / L-2 #94 / L-3 #95 / L-4 #96（L-4 のみ Phase D）。
> 前提：戦術コア（RTS群）＋ Phase C 戦略最小プロト（C-1〜C-3）が通った後。基盤は `docs/phase-c-core-design.md`。

## 0. スコープと「意図的に作らないもの」
- **作る**：資源の産出（星系）・輸送（船団＋補給線）・消費（艦隊/人口）・通商破壊（補給線攻撃）。**戦時の兵站＝兵糧攻め/補給線の攻防**を成立させる。
- **作らない（タイクン化回避）**：路線・駅の建設管理／通貨・利益経済／多段生産ツリー／旅客需要最適化。
  → 産出は**星系類型で自動的に決まる**（建設マイクロ無し）。経済は**現物（資源）収支のみ**で通貨は持たない。
- 既存規約（CLAUDE.md）と Phase C コア設計の語彙（`StarSystemData`/`CorridorData`/`GalaxyManager`/`StrategicFleet`/`CorridorBattleHost`）に乗る。

## 1. 全体像：C-6（消費）と L（生産・輸送）の接続
```
[星系=工場 L-1] --産出--> [勢力備蓄] --補給線 L-2--> [前線艦隊の現地補給] --消費 C-6--> 継戦
        ^                                   │
        │ 人口/食料 L-4                     × 敵ZOC/通商破壊 L-3 で断たれると前線が枯れる
```
- **C-6＝消費側**（継戦で艦隊が補給を減らす／尽きると戦線維持不可）。
- **L＝生産・輸送側**（星系が現物を産み、補給線で前線へ届ける）。本書は L の供給網と、それを断つ攻防（L-3）を定義する。

## 2. 資源モデル（少数に絞る）
- **物資（supplies）**：汎用＋食料兼用（L-4 の人口給養に使う）。
- **弾薬（ammo）**：戦闘継続に必要（C-6 の消費対象）。
- **燃料（fuel）**：ワープ/移動に必要（C-6 の消費対象）。
- **人口（population）**：L-4 で導入（徴募基盤・要給養）。
- 表現：`enum ResourceType { 物資, 弾薬, 燃料 }`（人口は別枠で星系が保持）。備蓄は**勢力ごとの現物プール** `Dictionary<ResourceType,int>`。

## 3. L-1：資源と生産（星系＝工場） … #93
- **星系類型** `enum SystemType { 工業, 農業, 鉱業, 居住 }` を `StarSystemData.systemType` に付与（Phase C コアで予約済み）。
- **産出（時間で自動）**：支配下の星系が毎 tick、類型に応じて勢力備蓄へ加算（建設マイクロ無し＝類型で決まる。将来の強化はフックのみ）。
  - 工業→弾薬＋物資、農業→物資（食料）、鉱業→燃料、居住→人口（L-4）。レートは `public` 調整値。
- **支配**：`GalaxyManager` が星系オーナー（`FactionData`）を保持。オーナーが居る星系のみ産出。
- **データ**：`GalaxyManager` に `Dictionary<FactionData, Dictionary<ResourceType,int>> stockpiles`。`GameSettings`（または GalaxyManager）に保持。
- **完了条件**：支配星系から資源が時間で貯まり、種類ごとに見える（戦略HUDに勢力備蓄を表示）。

## 4. ZOC（支配領域 / Zone of Control）定義 … L-2/L-3 の前提
> L-2/L-3 が参照する「敵ZOC」を本書で定義する（専用Issue化は任意）。
- **定義**：ある回廊（エッジ）が **敵ZOC下** とは、次のいずれか：
  1. その回廊上に敵対 `StrategicFleet`（または敵要塞 C-7）が居る／交戦中。
  2. 両端いずれかの星系が敵支配。
- **用途**：補給線（L-2）と輸送船団（L-3）は **敵ZOC下の回廊を通れない**。通そうとすると遮断＝前線が枯れる、または迎撃に晒される。
- **実装**：`GalaxyManager.IsCorridorUnderHostileZOC(corridor, viewerFaction)` を新設。敵対判定は既存 `FactionRelations.IsHostile` を流用（重複実装しない）。

## 5. L-2：輸送船団と補給線 … #94
- **補給線**：生産星系（デポ）→前線艦隊までの、**自勢力支配かつ敵ZOCを通らない回廊の連なり**（グラフ経路）。`GalaxyManager` が BFS/Dijkstra で「前線艦隊が備蓄に接続しているか」を判定。
- **現地補給**：各 `StrategicFleet` は **現地補給ストック**（物資/弾薬/燃料）を持つ。補給線が**繋がっている間だけ**毎 tick リフィル（勢力備蓄から引く・ワープ＝時間を流用して到達遅延あり）。
- **C-6 接続**：前線艦隊の C-6 消費は**現地補給ストックから**引く。補給線が断たれるとリフィルが止まり、消費だけが進む＝**補給切れ→戦線維持不可**（C-6 の撤退/再補給トリガに接続）。
- **輸送船団**：補給線上を実際に資源を運ぶユニット `Convoy`（`StrategicFleet` の軽量派生／別クラス）。ワープ＝時間で進む（既存設計流用）。船団が運ぶ量が前線リフィル量の実体。
- **完了条件**：後方から前線へ資源が流れ、線（または船団）を断つと前線が補給切れになる。

## 6. L-3：通商破壊（輸送船団の迎撃） … #95
- **迎撃**：敵対艦隊が回廊で敵 `Convoy` を捕捉→**小戦闘**を起動。処理は2系統：
  - プレイヤーが関与する戦線＝ズームインの戦術戦闘（`CorridorBattleHost` を流用、護衛艦隊 vs 襲撃艦隊＋輸送船団）。
  - 非指揮戦線＝**C-8 自動解決**（戦力・護衛厚で簡易判定）。
- **撃破効果**：船団撃破＝その分の資源が前線へ届かない＝敵前線が枯渇（兵糧攻め）。
- **護衛**：`StrategicFleet` を船団に随伴させ防御。`FleetAI` に「護衛（随伴・迎撃排除）」「襲撃（船団優先攻撃）」の役割を追加（戦術側は既存 AI を流用、戦略側は船団を標的に取れるよう拡張）。
- **完了条件**：補給線を叩いて敵を干上がらせる／護衛で守る、の攻防が成立する。

## 7. L-4：人口・徴募・消費（人を運ぶ） … #96（Phase D）
- **人口**：`StarSystem` に `population` と `provisionState`（給養充足度）を持たせる。`居住`類型が人口を産み、解放（支配）した住民は**徴募基盤**になる。
- **徴募**：給養が足りる人口から、時間＋資源で**新 `StrategicFleet`（再生産）**を生む。前線の損耗を後方の人口×補給で補う、という戦略圧。
- **飢餓（滅びの時計）**：食料（物資）補給が届かない星系は人口が減少。補給途絶＝じわじわ滅ぶ。
- **住民退避**：危険星系（ハザード/敵接近）からの住民退避も輸送対象（船団の積荷に人口を含める）。
- **収支**：消費（艦隊維持＋要塞＋人口給養）と生産の差し引きが戦略の主たる圧。`GameSettings`/`SaveData`（#19）に状態保持。
- **完了条件**：補給で人口を養い徴募で艦隊を再生産でき、補給途絶で人口が減る。

## 8. 既存／Phase C コードとの接続
| 対象 | 兵站での使い方 |
|---|---|
| `StarSystemData.systemType`（Phase Cコアで予約） | L-1 の産出類型。 |
| `CorridorData` / グラフ（`GalaxyManager`） | 補給線の経路探索・ZOC 判定の土台。 |
| `StrategicFleet` | 現地補給ストックを追加。`Convoy` は軽量派生。 |
| C-6（補給消費） | L-2 の現地補給ストックから引く＝消費側。本書は供給側を定義。 |
| `FactionRelations.IsHostile` | ZOC・迎撃・支配の敵味方判定の唯一の窓口。 |
| `CorridorBattleHost`（C-3） | L-3 の通商破壊戦闘の起動に流用。 |
| C-8（自動解決） | 非指揮戦線の船団迎撃の判定。 |
| `GameSettings` / `SaveData`（#19） | 備蓄・人口・補給状態の保持/保存。 |
| `BlackHole`(A-5) | L-4 の危険星系（退避動機）の一例。 |

## 9. 段階ビルド手順（各段でコンパイル＆既存動作維持を確認）
1. **L-1**：`SystemType` 産出＋勢力備蓄（時間で貯まる・HUD表示）。戦闘に影響させず可視化まで。
2. **ZOC**：`IsCorridorUnderHostileZOC` ＋ 戦略ビューでの可視化（敵ZOC回廊を着色）。
3. **L-2a**：補給線の経路判定＋現地補給リフィル（C-6 消費に接続）。線を切ると枯れる。
4. **L-2b**：`Convoy` ユニット（実際に運ぶ・ワープ時間）。
5. **L-3**：船団迎撃（ズーム戦術＝`CorridorBattleHost` 流用 ／ 非指揮＝C-8）＋護衛 AI。
6. **L-4（Phase D）**：人口・徴募・飢餓・退避輸送。

## 10. リスク・未決事項
- **ZOC の粒度**：回廊単位で足りるか（星系内ZOC・部分支配は当面持たない）。まず回廊単位で最小実装。
- **補給線の計算コスト**：経路探索を毎 tick 全艦隊分やると重い。間引き（`congestion` と同様の更新間隔）＋差分更新。
- **C-6 とのインターフェイス**：C-6 の「現地補給ストック」を L-2 がリフィルする、という責務境界を最初に固定（二重実装を避ける）。
- **船団の表現**：`StrategicFleet` 派生か別クラスか。標的（L-3）・護衛・積荷（L-4 人口）を扱える最小形にする。
- **【要・作者判断】**：資源の産出/消費レート、補給線の到達遅延、徴募コスト、飢餓速度（滅びの時計のテンポ）、人口の初期分布。
- 着手順：**L-1 → ZOC → L-2 → L-3**（ここまで Phase C）→ **L-4（Phase D）**。C-4（並行）/C-8（自動解決）/C-7（要塞）と相互に噛む。
