## 1. 目標：最小の縦スライス（end-to-end 1ループ）
次の1ループが**実機で通る**ことをゴールにする：

> **1惑星から開始 → ハイパーレーンで隣星系へ艦隊移動 → 接触で既存の会戦（Battle シーン）→ 勝者が惑星を占領 → 占領惑星の POP が税/資源を産む → 支持が動く → 1つの政治的帰結が画面に出る**

これが回れば、外交・世論・宗教・経済・群像の深いシステムは「**この回路のどこに刺すか**」で**増分実装**できる。

### S5/S6 の具体タスク（次スプリント）
1. **国庫**：`ResourceStockpile` or `FiscalState` を `CampaignState` の勢力状態に持たせ、`GovernanceRules.OutputFactor`×税率で時間徴収（`CampaignRules.Tick` に挿す）。
2. **税率レバー**：戦略マップに勢力メニュー（最小=スライダー1本）。高税→`TaxBurdenPenalty` で支持（`FactionState` の合意/希望）が下がる。
3. **支持閾値→イベント**：`EventEngine` に「支持低下」イベントを登録し、発火を**モーダル提示**（`EventManager` 新設・UITK推奨＝`GineiUITK`）。選択肢の効果は `FactionState` へ。
4. 受け入れ基準：**増税→支持低下→イベント発火→選択→帰結が画面で見える**こと（1レバー＋1帰結・タイクン化回避）。

## 4. スライス後の増分（深いシステムを“どこに刺すか”）
スライスが通った後、**「配線待ち」（純ロジック済）を優先**して回路の各点に接続する：
- **会戦の前後** → 外交 DIP-2/3 #191/#192（宣戦/講和）・自動解決の等価値調整 #41。
- **占領・内政** → Pop階級 #110／統治政策 #112／占領/解放 #114／宗教 #172／民族 #194。
- **支持** → 内部勢力 #113／世論 #193／政体創発 #117。
- **経済** → 造船UI（BUILD-5/6）／市場 #179／財政の画面化。
- **群像** → 指揮班HUD（CMD-5/6 #896/#897）／昇進ループ #79／列伝・殿堂 #784/#785。

各 EPIC は「**スライスのどの点に刺さるか**」が明確なので、**独立に・任意の順で**増分できる。

## 7. 方針との整合（崩さない前提・変更なし）
- **タイクン化回避**：各最小版は「**1レバー＋1帰結**」。指標を増やさない。
- **後方互換**：戦略レイヤーは既存の**単体シナリオ会戦と併存**。スライスは別モードで、**既存 Battle を壊さない**。
- **既存窓口を流用**：戦闘在庫 `FleetRegistry` と編制 `FleetRoster`/`OrderOfBattle` を混同しない／敵対判定は `FactionRelations`（外交 #189 が駆動）／シーン間状態は `GameSettings`・戦役状態は `StrategySession`/`CampaignState`／SO解決は `ContentDatabase`。
