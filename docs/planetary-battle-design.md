# 惑星の戦い 設計（EPIC #131）

> 敵勢力圏の回廊を突破→**惑星領域**へ進み、**侵略値**で惑星を占領する戦い。惑星は**軌道防衛（アルテミスの首飾り）**で守られ、**航行不能領域**で機動を制限する。戦略の「征服」ループの中核。
> 関連：PB-1 #132 / PB-2 #133 / PB-3 #134 / PB-4 #135 ／ C-3 #36（有界戦場）／ 要塞 #76-78 ／ 占領/解放 #114 ／ 内政 #109 ／ ZOC #100 ／ 入植 #129。
> 基盤：`docs/phase-c-core-design.md`（回廊戦闘・`CorridorBattleHost`・`BattlefieldBounds`）。

## 0. スコープ
- 「**惑星を実際に奪う**」征服メカニクスを定義する（回廊で勝つ＝通り道、惑星を獲る＝侵攻占領）。
- 既存の戦術エンジン・要塞モデル・有界戦場を**流用**（新エンジンを作らない）。戦術側（軌道防衛・航行不能）は Battle シーンで先行プロト可能。
- タイクン回避：占領は「侵略値を貯める意思決定（艦を留める/守る）」で、建設パズルにはしない。

## 1. フロー（回廊→惑星→占領）
```
[回廊戦闘 C-3 #36] ──突破/通過(ZOC #100)──▶ [惑星領域アリーナ]
                                              ├ 軌道防衛(アルテミス PB-3) を制圧
                                              ├ 航行不能領域(PB-4) を避けて接近
                                              └ 占領圏に留まり 侵略値(PB-2) 蓄積 ──▶ 占領 ──▶ 占領/解放 #114・内政 #109
```
- **回廊突破/通過（PB-1 #132）**：敵ZOC下の回廊は突破（C-3で勝つ）しないと先へ進めない。抜けると星系内の惑星領域へ。
- **惑星領域**：惑星を中心とした有界アリーナ（`CorridorBattleHost` 相当を惑星用に流用、または回廊戦の延長）。

## 2. 惑星領域アリーナ
- C-3 の有界戦場（壁＝侵入不可コライダー＋`BattlefieldBounds`、両端＝進入口）を**惑星周辺**に展開。中心に惑星（航行不能領域 PB-4）。
- 攻撃側は進入口からスポーン、惑星へ接近。防御側＝駐留艦隊＋軌道防衛。
- 回廊戦と同一戦闘の延長にするか別フェーズにするかは【要・作者判断】（推奨：回廊で決着→勝者が惑星領域へ移る別フェーズ。実装簡素）。

## 3. 軌道防衛＝アルテミスの首飾り（PB-3 #134）
- **要塞モデル（#76-78）の惑星リング版**：惑星周回に固定配置の防衛衛星群（`IShipTarget`/`FleetWeapon`・不動・段階制圧・側背面無効）を流用。
- 各衛星が射界で射撃、個別破壊可能 → **破壊した方角が死角＝攻め口**（要塞と同じ攻略リズム）。
- **アルテミスの首飾り**＝高耐久・全周防御（バリア/反射衛星のイメージ）。規模・種類は惑星/内政(#109)/研究(#123)で変わり、内政で建造。
- 回廊の固定拠点（要塞）と惑星の軌道防衛（アルテミス）を**使い分け**（同じ Fortress 系コンポーネントの配置違い）。

## 4. 侵攻・占領＝侵略値（PB-2 #133）
- 惑星に**侵略ゲージ**。攻撃側の艦が**占領圏**（惑星近傍）に留まり、かつ**軌道防衛が抑えられている**間、侵略値が時間で増加（艦数/兵力で速度）。
- 防御側が占領圏の攻撃艦を排除すると停滞/減少。**閾値で占領成立**。
- 占領後は **占領/解放 #114・内政 #109** へ（敵惑星＝侵攻占領、未領有＝入植 #129 と区別）。
- 実効値パターン・public 調整値（蓄積速度/閾値/占領圏半径）。

## 5. 航行不能領域（PB-4 #135）
- 惑星本体・重力圏・デブリ帯・宙域の壁＝**進入不可**（C-3 #36 の壁＋`BattlefieldBounds`、`BlackHole` 回避方式を流用）。
- AI は航行不能を避けて移動（`SteerAroundBlackHoles` 方式＝**移動本体は変えずAI側で目標補正**）。
- 「無制限の航行・素通り・無限旋回」を防ぐ＝惑星を盾に・死角を突く戦術が生まれる。

## 6. 既存システムとの接続・洗練
| 対象 | 惑星の戦いでの扱い |
|---|---|
| C-3 #36（有界戦場・壁） | 惑星領域アリーナ・航行不能（PB-4）の枠組みを共有・拡張。 |
| 要塞 #76-78 | 軌道防衛（アルテミス PB-3）に流用（惑星リング配置）。回廊要塞と使い分け。 |
| 占領/解放 #114・内政 #109 | 占領の「実行」は侵略値（PB-2）、その後の統治はそちら。 |
| ZOC #100 | 回廊突破の条件（PB-1）。 |
| 入植 #129 | 未領有＝入植／敵惑星＝侵攻占領、の区別。 |
| `BlackHole`(A-5) | 航行不能領域の回避方式の前例。 |
| `FactionRelations` | 攻防の敵味方判定の唯一の窓口。 |

## 7. 段階ビルド手順（戦術側を先行プロト可）
1. **PB-4 航行不能領域**：惑星本体＝進入不可＋AI回避（Battle シーンで単体検証）。
2. **PB-3 軌道防衛**：要塞 #76 を惑星リング配置で流用（衛星群・段階制圧）。
3. **PB-2 侵略値**：占領圏＋ゲージ＋占領→（仮の）支配反転。
4. **PB-1 到達**：Phase C の回廊戦（C-3）から惑星領域へ繋ぐ（ZOC突破条件）。
5. 占領後を **#114/#109** に接続。

## 8. リスク・未決事項
- 回廊戦と惑星戦の**繋ぎ方**（同一戦闘 or 別フェーズ）。`FleetRegistry` 単一性（Phase C既知課題）→ 1戦場ずつ（`phase-c-late-design.md` の方針）。
- 侵略値のバランス（守り切れる/攻め切れるの均衡）。軌道防衛の規模インフレ。
- 航行不能領域の配置が機動を殺しすぎないこと。
- **【要・作者判断】**：侵略値の条件/速度/閾値、軌道防衛の再建可否、惑星領域の広さ、回廊戦との接続方式。
