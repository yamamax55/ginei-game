# イノベーションのジレンマ 参考設計（EPIC #INNO）

> 参照元：クレイトン・クリステンセン『イノベーションのジレンマ』（経営学・破壊的イノベーション論）。
> 中核命題＝**「優良企業（合理的に正しく経営された組織）ほど、破壊的技術の前に敗れる」**。顧客の声を聞き、高収益市場へ資源を最適配分する“正しい経営”が、性能は劣るが安く・新市場で立ち上がる**破壊的技術**への対応を組織的に不可能にする。
> 本ドキュメントは、当プロジェクト（Ginei＝銀英伝風の星間国家戦略＋既に巨大な社会・政治・軍事の純ロジック層）にとって**役に立つ視点**だけを抽出し、EPIC `#INNO` として issue 化する提案。
> **著作権注意**：書名・固有名・文章・図表・著者固有の事例（ディスクドライブ業界等）は流用しない。**「破壊的イノベーション」という構造パターン／メカニクスのみ**を参考にする。固有名詞は当プロジェクトの語彙（研究/軍事ドクトリン/組織）へ翻訳して実装する。

## 0. なぜ「イノベーションのジレンマ」が本システムに役立つか

当プロジェクトは技術・組織・統治の**マクロ純ロジックを大量に保有**している（CLAUDE.md「テスト基盤」節 参照）：

| 既存（カバー範囲） | 何を解いているか |
|---|---|
| `ResearchRules`/`ResearchProject`（#123-127） | 研究力×生産力→研究ポイント線形蓄積→完成で性能ボーナス。政体で分野が偏る（`IdeologyBias`） |
| `Organization`/`SuccessionRules`（#812） | 結束・制度化・個人カリスマ。英雄死後に組織が続くか／属人組織は滅ぶ |
| `GovernanceRules`/`Province`（#109） | 安定度の目標値収束・占領統合・産出係数 |
| `CampaignState`/`FactionStateRules`（統合層） | 勢力レベルの社会・政治シミュの合成・崩壊判定 |
| `AutonomyRules`/`CommandDoctrine`（#544-550） | 集団依存↔自律分散の指揮ドクトリン・創発シナジー |
| `MeritRankRules`（#900-905） | 戦功→爵位・制度の畏怖・法家の罠（短期最強→長期崩壊） |
| `DisclosureRules`/`DisclosureLedger`（FND-4 #495） | 世界観loreの条件付き連鎖開示 |

**しかし、これらは「技術＝線形に蓄積する単調な強化」かつ「組織＝結束/制度化の盛衰」**であり、クリステンセンが固有に描く以下が**欠けている**：

| 『イノベーションのジレンマ』が固有に持つ視点 | 当プロジェクトでの欠落 |
|---|---|
| **持続的技術 vs 破壊的技術の二系統** | `ResearchProject` は単一の線形進捗。**既存路線を強化する研究**と**性能は劣るが軌道を変える研究**の質的区別が無い |
| **性能のS字曲線と“性能の供給過剰（overshoot）”** | 完成で `strengthYield` を足すだけ。**性能が顧客（戦線）の要求を超えて飽和し、過剰性能が無価値化する**現象が無い |
| **破壊的技術は新市場（劣る指標）で立ち上がり、改善して本流を飲む** | 研究は単調に強くなるのみ。**最初は弱い技術が下から本流を奪取する**ダイナミクスが無い |
| **“正しい経営”ゆえの失敗＝優良組織が合理的に破壊的技術を拒む** | `Organization` は結束/制度化の量。**高制度化・高実績ほど新技術を採れない**という負の連動（資源配分の重力）が無い |
| **価値ネットワーク／資源配分プロセスが既存顧客へ重力的に向かう** | 研究投資先は外生。**既存戦力（高収益部門＝主力艦隊/主流ドクトリン）が予算を吸い、破壊的研究を兵糧攻めにする**回路が無い |
| **両利きの経営＝破壊的事業を本体から隔離した独立部門で育てる** | 組織は単一。**本体の重力から切り離した自律部門でのみ破壊的技術が育つ**という処方が無い |
| **軍事ドクトリンの保守性（勝者の硬直）** | `AutonomyRules` はドクトリンの型。**過去の勝利が組織を硬直させ、次の破壊に最も弱くなる（勝者の呪い）**が無い |

**結論**：『イノベーションのジレンマ』は、当プロジェクトの「単調強化される技術」に**①持続/破壊の二系統 ②性能S字と供給過剰 ③下からの本流奪取**という**技術軌道の動学**を、そして「結束/制度化の量」でしかなかった組織に**④“正しい経営”ゆえの硬直（高実績ほど破壊に弱い）⑤資源配分の重力 ⑥両利き（隔離部門）という処方**を与える。**`ResearchRules` と `Organization` の交差点**――軍の保守性×研究軌道――という、既存層が触れていない欠落軸を埋める。

## 1. 役に立つ視点（要約）

『イノベーションのジレンマ』の構造を、**本システムに効く形**で1行ずつ：

1. **技術には二系統ある＝持続的（既存路線を磨く）と破壊的（軌道を変える）**。研究は一本道でなく分岐する。→ `ResearchRules`/`ResearchProject` に**研究の種別**を導入する核。
2. **性能はS字に伸び、やがて戦線の要求を超えて飽和する（供給過剰）**。過剰性能は価値を失い、安く・別軸で十分な破壊的技術に逆転される。→ `MeritRankRules`「法家の罠（短期最強→長期崩壊）」と同じ“盛者必衰”の技術版。
3. **破壊的技術は最初は劣り、辺境（新市場＝低要求の戦線）で立ち上がり、改善して本流を飲む**。→ #160 フェザーン／辺境星系＝破壊的技術の苗床として戦略マップに接続。
4. **“正しい経営”が失敗を生む＝高実績・高制度化の組織ほど合理的に破壊的技術を拒む**。→ `Organization.institutionalization`/`SuccessionRules` に**負の連動（硬直度）**を入れる。CLAUDE.md「タイクン化回避＝高位の決断→創発帰結」と完全に整合（プレイヤーは投資配分という高位決断だけ、敗北は創発）。
5. **資源配分プロセスが既存顧客（主力部門）へ重力的に向かう＝破壊的研究は兵糧攻めにされる**。→ `ResearchRules` の投資配分に**重力バイアス**を足す。`CampaignState`/`FiscalRules` の予算と接続。
6. **処方は“両利き”＝破壊的事業を本体から隔離した自律部門で育てる**。→ `AutonomyRules`/`CommandDoctrine`（自律分散）と直結。隔離した独立研究部門だけが破壊的技術を完成させられる。
7. **軍事への翻訳＝勝者の呪い**。過去の勝利でドクトリンが硬直した勢力ほど、次の破壊的戦術（新編制・新兵科）に最も弱い。→ `AutonomyRules`×`Organization`×会戦の係数（#106 `CombatModifiers`）。

## 2. 取り入れるべきメカニクス（優先度つき・既存への接続）

> 大原則：**`ResearchRules`#123/`Organization`#812/`AutonomyRules`#544/`MeritRankRules`#900 を作り直さない**。INNO はそれらに**欠落軸（技術軌道の動学・組織硬直）を足し、接続する**だけ（additive）。プレイヤー操作は「投資配分・隔離部門の設置」という**高位の決断のみ**（マイクロ操作を増やさない＝タイクン回避）。

### ★★★ 最優先（真の欠落・本作の signature）

#### INNO 技術軌道：持続的 vs 破壊的の二系統（研究の種別）
- **持続的技術**＝既存性能指標を磨く（確実・既存戦力を強化）。**破壊的技術**＝当初は主指標で劣るが別軸（コスト/量産/到達性）で優れ、**独自のS字で改善**する。`ResearchProject` に**種別（持続/破壊）＋性能トラジェクトリ**を持たせる。
- 接続：新 `TechTrajectoryRules`/`enum TechKind{持続,破壊}`（純ロジック・test-first）。`ResearchRules.ResearchOutput`/`Tick` を**置換せず**、その上に軌道（S字・別軸性能）を載せる。

#### INNO 性能S字と供給過剰（overshoot）→ 下からの本流奪取
- 性能はS字（逓増→逓減）で伸び、**戦線の要求水準（demand line）を超えると過剰**になり、追加性能が無価値化。破壊的技術の性能が要求水準に追いつくと、**主指標で劣ったまま本流を奪取**（安さ・別軸で十分になるため）。
- 接続：`TechTrajectoryRules.PerformanceAt`（S字）／`IsOvershoot`（要求超過）／`DisruptionCrossover`（破壊側が要求水準に到達＝逆転点）。`MeritRankRules`「法家の罠」の技術版＝**短期最強が長期に飲まれる**。

#### INNO “正しい経営”ゆえの硬直（高実績・高制度化ほど破壊を拒む）
- 組織の**実績・制度化が高いほど**、破壊的技術の採択確率が**下がる**（資源配分が既存顧客＝主力へ向かう重力）。合理的判断の積み重ねが敗北を生む＝**プレイヤーが“正しく”振る舞うほど創発的に負ける**。
- 接続：`Organization.institutionalization`/`leaderCharisma`＋実績指標から **`IncumbentRigidity`（硬直度）** を導出（純ロジック・基準値非破壊＝量を読むだけ）。`SuccessionRules` と同じ「制度化の光と影」を技術面に拡張。

### ★★ 高（資源配分の重力と処方）

#### INNO 資源配分の重力（破壊的研究の兵糧攻め）
- 限られた研究予算は**既存主力（高収益部門＝主流ドクトリン/主力艦隊）へ重力的に配分**され、破壊的研究は構造的に痩せる。プレイヤーの配分決断に**重力バイアス**がかかる（放置＝自動的に持続側へ流れる）。
- 接続：`TechTrajectoryRules.AllocationGravity`（硬直度×実績→破壊側配分の目減り係数）。`FiscalRules`/`CampaignState` の予算、`ResearchRules` の投資先選択に接続。

#### INNO 両利き＝隔離した自律部門でのみ破壊的技術が育つ（処方）
- 本体の重力下では破壊的研究は完成しない。**独立・自律（`CommandDoctrine.自律分散`）の隔離部門**を立てると、重力バイアスを免れて破壊的技術を育てられる＝唯一の処方。隔離はコスト（本体との結束低下リスク）を伴う。
- 接続：`AutonomyRules`/`CommandDoctrine`×`TechTrajectoryRules`。隔離部門の `DoctrineFactor`/`EmergentSynergy` が破壊的研究の重力を打ち消す。**高位の決断＝「隔離部門を設置するか」だけ**（マイクロ無し）。

#### INNO 勝者の呪い（軍事ドクトリンの保守性）
- **過去の勝利が組織を硬直させ**、次の破壊的戦術（新編制・新兵科）に最も弱くする。連勝した勢力ほど `IncumbentRigidity` が上がり破壊に脆い。
- 接続：`AutonomyRules`×`Organization`×会戦係数（#106 `CombatModifiers`／会戦勝敗 `BattleManager`）。破壊的戦術を採れた側が会戦で優位を得る（実効値パターン・基準非破壊）。

### ★ 中（盤面接続・世界観lore）

#### INNO 戦略マップへの軌道接続（辺境＝破壊の苗床／本流逆転）
- 各勢力に**技術軌道の状態**を持たせ、暦境界Tickで持続/破壊S字を進め、逆転点で本流転換イベント。辺境星系（低要求戦線）で破壊的技術が立ち上がる。観測層に追従。
- 接続：`CampaignState` に勢力ごとの技術軌道を載せ、`CalendarDispatcher`（暦境界Tick・PERF規律）で進める。観測層（`CoreStateInspector` 用語集＋必要なら `Register`）へ追従。

#### INNO（lore）破壊的イノベーションの世界観開示データ
- 「正しさが滅びを呼ぶ」「下からの逆転」「勝者の硬直」を**コード新設せず** `DisclosureLedger`（FND-4）への lore データ入力として供給。世界観EPIC（盛者必衰・易姓革命 `DynastyRules`）と共鳴。
- 接続：`DisclosureLedger`。新規コードなし。

### ❌ 不採用（重複・既存で十分・本作スコープ外）

| 不採用 | 理由 |
|---|---|
| 研究ポイント蓄積・分野別効率・政体偏り | **`ResearchRules`#123-127 が既にカバー**。INNO は作り直さず、その上に軌道層を載せて接続するだけ |
| 組織の結束/制度化/継承そのもの | **`Organization`/`SuccessionRules`#812 がカバー**。INNO は量を読んで硬直度を導出するのみ（read-only） |
| 指揮ドクトリンの型（集団依存/自律分散）そのもの | **`AutonomyRules`#544-550 がカバー**。INNO は隔離部門の処方として接続するだけ |
| 「短期最強→長期崩壊」の盛者必衰 | **`MeritRankRules` 法家の罠 #900・`DynastyRules` 易姓革命 がカバー**。INNO は技術版として共鳴させる（思想は重複させない） |
| 市場/顧客セグメント/利益率の経済モデル化 | **`MarketRules`#179・`FiscalRules`#161 がカバー**。INNO は「顧客＝戦線の要求水準」へ翻訳し、市場経済を新規に作らない（タイクン化回避） |
| 企業マネジメント・人事採用・製品ラインのマイクロ管理 | **タイクン化回避**。プレイヤー決断は投資配分・隔離部門設置という高位の2レバーに限定。マイクロ操作を新設しない |
