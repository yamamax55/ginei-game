# Almagest -Overture- 参考設計（EPIC #ALM）

> 参照元：『Almagest -Overture- “Third Edition”』ver.3.00（2006・Serendipity／むつきみかつ・フリーソフト）。
> 銀英伝風の星間国家 4X グランドストラテジー。**マウスのみ・ターン制（1年=52ターン）**。
> 本ドキュメントは、当プロジェクト（Ginei＝リアルタイム会戦＋シームレス戦略）に**取り入れるべき要素**と、特に優れた**UI構造**を整理し、EPIC `#ALM` として issue 化する提案。
> 著作権注意：Almagest はフリーソフトだが著作権は放棄されていない。**アセット（画像/文章/固有名/シナリオ）は流用せず、設計パターン・ゲームメカニクスのみを参考にする**。

## 0. なぜ Almagest が参考になるか

当プロジェクトは「純ロジックは大量に着地済み、ボトルネックは**盤面/UIへの配線**」という段階（[roadmap.md](./roadmap.md) §1）。
Almagest は **同じジャンル（銀英伝風 4X）を完成・出荷した実例**であり、当プロジェクトに足りない以下を「動く仕様」として提供する：

| 当プロジェクトの状態 | Almagest が示す完成形 |
|---|---|
| 戦略UIが**機能別パネルの寄せ集め**（B=編成・O=編制・I=星系情報…ホットキー分散） | **統一トップメニュー**（艦隊/外交/内政/士官/情報/記録/システム）＋共通リスト/ダイアログ部品 |
| `ResearchRules`/`ShipyardRules` は純ロジックのみ・**艦の中身は固定** | **艦艇再設計**（技術スロット装填）＋**FCS.Cap**（技術で配備数が増える）＝深い兵器カスタム |
| 外交 `DiplomacyState` はあるが**国際的評判の軸が無い** | **王道値/覇道値**の二軸＋**主義ドリフト**で外交成否とエンディングが決まる |
| `AdmiralSkillRules`/`Person` はあるが**特殊能力の中身が空** | **特殊能力10種**＋**作戦適性 S〜E**（地形別）の具体例 |
| 会戦は良質なRTS（完成形に近い） | **戦術4種の三すくみ**＋**旗艦隊/提督の波及**＝AI ドクトリンの参考 |

**結論**：Almagest からの最大の収穫は ①**戦略UIの統合シェル** ②**艦艇設計・技術・FCS.Cap** ③**王道/覇道メタ層**。この3つが当プロジェクトの「配線」フェーズを一気に前進させる。

## 1. UI構造（最重要・そのまま設計の下敷きにできる）

### 1-1. 戦略フェイズ基本画面の4領域

```
┌─────────────────────────────────────────────┬───────────────┐
│                                             │ ② コマンドエリア │
│                                             │ （固定・移動不可）│
│         ③ アルマゲスト星図                    │ ┌───────────┐ │
│         （HSS航路グラフ）                     │ │メニューセット名│ │
│         惑星/要塞クリック→情報               │ │星刻歴・ターン数│ │
│         コマンド中：左ク=決定/右ク=取消        │ ├───────────┤ │
│                                             │ │ 艦隊        │ │
│   ┌──────────────┐                          │ │ 外交        │ │
│   │① 国家・惑星データ │ ←移動可能な          │ │ 内政        │ │
│   │  元首/行政官 顔   │   フローティング       │ │ 士官        │ │
│   │  治安/経済/駐留   │                       │ │ 情報        │ │
│   └──────────────┘                          │ │ 記録 / システム│ │
│                                             │ │ 今期終了     │ │
│ ┌────────┐                                  │ ├───────────┤ │
│ │④勢力図   │←クリックで星図表示範囲移動        │ │👑王道 🛡覇道 │資金│ │
│ │(ミニマップ)│ 白枠=表示範囲/紫線=星系区分       │ └───────────┘ │
│ └────────┘                                  │               │
└─────────────────────────────────────────────┴───────────────┘
```

- **① 国家・惑星データウィンドウ**：移動可能なフローティング。元首/行政官の顔グラ＋治安/経済/駐留艦隊数/支配国経済規模。クリックした惑星の情報を出す常設インスペクタ。
- **② コマンドエリア**：固定。上部に「現在のメニューセット名＋星刻歴＋ターン数」、右下に「資金」、左下に「王道値（青・王冠）／覇道値（赤・兜）」。**メインウィンドウ上の右クリックでトップメニューに戻る**（万能エスケープ）。
- **③ 星図**：HSS航路グラフ。コマンド中は左ク=目標決定／右ク=キャンセル（当プロジェクトの `GalaxyView` と同型）。
- **④ 勢力図**：ミニマップ。クリックで表示範囲移動（当プロジェクトの #84 ミニマップと一致）。

➡ **当プロジェクトへの示唆**：現状ホットキー分散（B/O/I…）を、**固定コマンドエリア＋万能右クリック復帰**の IA に寄せる。`GalaxyView` の右側に常設コマンドカラムを置くだけで「迷わない」UXになる。

### 1-2. 共通部品（全コマンドで再利用＝統一感の源泉）

Almagest の操作の一貫性は、**少数の共通ウィンドウ部品**を全コマンドが使い回すことで生まれている。これは当プロジェクトの「UITK 段階移行・見切れ防止」方針と完全に一致する。

| 部品 | 仕様 | 当プロジェクトの対応 |
|---|---|---|
| **リストウィンドウ** | タイトル／**項目名クリックでソート**（昇降トグル）／セルクリックで選択／詳細エリア／ページ数＋送り／決定・撤回（**右ク=撤回**） | `GineiUITK` に `GineiList` を新設。`OrderOfBattlePanel`(UITKパイロット)の正統進化 |
| **数値入力ウィンドウ** | 入力ボックス（**初期値=最大値**）＋増減ボタン＋MAX/MIN＋決定/撤回。**Enter=決定**・範囲外は自動クランプ | 艦艇配備数・資金援助額・建造数で再利用 |
| **メッセージウィンドウ** | 確認/撤回（右ク=同義）／▼（右ク長押し=スキップ）／**受諾・拒否（左クのみ＝誤操作防止）**／返答選択（ラジオ＋決定） | イベント提示 `StrategyEventPanel`・外交提案で再利用 |
| **編成画面** | 司令官/副司令/参謀ボタン＋配備艦艇。**能力が上昇=水色・低下=赤・Cap超過=赤**で即時可視化 | `FleetOrganizationPanel` を Almagest 配色規約に合わせる |

➡ **設計原則の抽出**：「**重要操作（受諾/拒否・降伏）は左クリック限定、それ以外は右クリックで一律キャンセル**」。この**右クリック＝万能取消**の規約が学習コストを劇的に下げている。当プロジェクトの Esc 優先順位チェーンと併用する。

### 1-3. コマンドの自己説明（オンボーディング）

- **秘書官アシストレベル Lv1〜3**：実行条件の説明・決定/撤回確認の冗長さを段階調整（Lv3=丁寧＝初心者、Lv1=即実行＝上級者）。
- **実行不可コマンドは秘書官が条件を説明**（なぜ押せないかを言語化）。
- **報告書**：ターン開始時に「自国で起きた事件・新規士官候補・完成艦艇」を要約。

➡ 当プロジェクトの `NotificationFeed`（左下トースト）を補完する「**周期ダイジェスト（報告書）**」と「**設定可能な手取り足取り度**」は、複雑な meta システム群を抱える本作にこそ効く。

## 2. 取り入れるべきゲームメカニクス（優先度つき）

### ★★★ 最優先（真の機能ギャップを埋める）

#### ALM-C 艦艇設計・技術・FCS.Cap
当プロジェクトの `ResearchRules`/`ShipyardRules`（純ロジック・未配線）に、**深い兵器カスタムの遊び**を与える。

- **基礎技術ゲート**：`エネルギー/ソフトウェア/ハードウェア` の開発率が**新技術の出現条件**（ツリーが創発的に伸びる）。`艦隊制御システム`=FCS.Cap。
- **艦艇再設計**：ベース艦＋技術スロット（**拡張性**＝スロット予算、各技術に**搭載量**）。強化方式 `＋UP`/`％UP`/`特殊`。同種別は1つのみ（特殊は複数可）。**水色=向上/ピンク=低下**で即時フィードバック。名称変更可・100枠登録。
- **FCS.Cap**：`配備可能数 = 国家FCS.Cap ÷ 艦の必要Cap`。**階級制限と二重**で効く＝「技術が上がると同じ提督でも大艦隊を持てる」戦略性。当プロジェクトの「階級→最大編成数」に技術軸を直交させる。
- **建造 vs 購入**：建造=工期ターン・安い／購入=即時・高い。**保有可能数**は従星数スケール。
- **改装/復元**：再設計データへ既存艦を載せ替え（巡航艦は全処分が前提・防衛艦/衛星は差額のみ）。

> 設計判断：当プロジェクトは `ShipClass`(戦艦/巡航/駆逐)＋`Shipyard`/`BuildOrder` を既に持つ。再設計は **`ShipClassStats` の実効値パターンを「技術スタックで合成」する形**に落とすと既存と整合（基準非破壊）。

#### ALM-A 戦略UIの統合シェル
第1章の UI 構造を `GineiUITK` 上に部品化。**Almagest からの一番の収穫**。

#### ALM-D1/D2 王道値/覇道値 ＋ 主義ドリフト
- **二軸評判**：王道値＝国際的信頼（外交が成功しやすい・頻繁に上下）／覇道値＝不信（一度上がると減りにくい）。外交の成否とエンディング分岐を駆動。
- **主義 5種**（保守/改革/中道/孤立/中立）：印象値（opinion）は毎ターン自然減少、**同主義は減りにくい/上昇も**。具体閾値＝宣戦布告 ~120・同盟 ~260・同盟破棄リスク ~200。
- 当プロジェクトの `DiplomacyRules.DriftOpinion`/`FactionState` に**王道/覇道**フィールドを足すだけで配線できる。エンディングは `DisclosureLedger`（FND-4）の条件式に流す。

### ★★ 高（既存純ロジックに具体コンテンツを与える）

#### ALM-E 士官システムの拡充
- **特殊能力10種**：統率/戦略/研究/内政/参謀/軍事/独断/象徴/救急/制圧 → `AdmiralSkillRules` の実コンテンツに。
  - 特に **独断**（司令官だとプレイヤー命令を無視して勝手に動く）は当プロジェクトの `AutonomyRules`/委任 #592 と同型＝**そのまま配線可能**。
  - **参謀**（副司令/参謀なら司令より優れた対応能力に置換）・**象徴**（カリスマで士気＋）・**救急**（壊滅時の負傷を軽減）も既存システムに直結。
- **作戦適性 S〜E**：遭遇戦/拠点侵攻/拠点防衛の**地形別適性**。`AdmiralData` の6能力に直交する「状況特化」軸。
- **功績値→昇進→最大編成数**：少佐10→元帥50。当プロジェクトの「階級ゲート #14」に**功績の蓄積→昇進**を足す。
- **負傷/療養**：壊滅艦隊の士官は低確率で負傷（最長~26ターン行動不能）、救急で軽減。`CaptivityRules`/`LifecycleRules` の軽量版。
- **人材捜索/仕官要請**：在野プールから登用（思想が近いほど応じる・断られても確率上昇）。

#### ALM-G 惑星防衛の3層
当プロジェクトの惑星攻城（`PlanetSiegeRules` 制空権）を、Almagest の**防衛艦隊／防衛衛星／軌道部隊**の三層モデルで具体化：
- **防衛衛星**＝制空権（既存）。`衛星`技術でバージョンアップ・行政官の政治力で回復速度↑。
- **防衛艦隊**＝惑星専用艦（機動艦隊に編入不可・最大30隻・海賊被害を抑える）。
- **軌道部隊**＝衛星に戦闘機を載せてF攻撃/F防御を付与（強力）。
- **迎撃戦（惑星近傍）/防衛戦（軌道上）**の区別＝「衛星が落ちる前に近傍で迎撃せよ」の緊張感。

### ★ 中（会戦AI・演出の参考。RTSを置換しない）

#### ALM-F 戦術ドクトリン
当プロジェクトの会戦は良質なRTSなので**置換しない**が、AI と「艦隊姿勢」の参考に：
- **戦術4種**：応戦(100/100)・強行(150/30)・機動(100+機動差/70)・密集(60/150)。**三すくみ＝強行＞機動＞密集＞強行**（攻撃対象と被攻撃で相性が変わる非単純三すくみ）。→ `FleetAI` のドクトリン選択や、プレイヤーが部隊に与える「姿勢」コマンドに転用可。
- **旗艦隊/提督の波及**：最若番艦隊の司令官＝提督。その作戦適性が**全艦隊の攻防に波及**、旗艦隊が壊滅/退却すると**全体がE扱い**。→「指揮構造が崩れると全軍が弱る」緊張感。`OrderOfBattle` の旗艦隊概念に接続。
- **行動順**：知力＋適性＋機動＋運。**先手で後出し有利な戦術を選べる**。

### ◇ 設計上の相違（採用時に翻案が必要）

- **ターン制 vs リアルタイム統一クロック**：Almagest は 1年=52ターンの**ターン制**、当プロジェクトは `GameClock` の**連続時間**。`今期終了`・per-turn 制限・行動消費はそのまま移植**しない**。代わりに「**周期ダイジェスト＝報告書**」「日次/週次のアクションポイント」として翻案。
- **マイクロ内政の回避**：Almagest の per-惑星 `経済投資`/`治安強化` は当プロジェクトが避ける**タイクン化**そのもの。UIシェルは採るが、内政の中身は当プロジェクトの**三層集約（惑星⊃星系⊃勢力）**を維持し、Almagest の per-惑星操作は採らない。

## 3. 取捨選択（本プロジェクトのシステムに合うか／後退しないか）

> 方針：**当プロジェクトの設計思想に合致し、かつ既存機能の“後退”にならないものだけ**を採用する。
> 後退＝当プロジェクトが意図的に採った先進的な仕組み（連続時間・三層集約・良質RTS）を、Almagest の古い仕組みで置き換えてしまうこと。

### ✅ 採用（additive・既存純ロジックの配線 or 真のギャップ埋め）

| 採用要素 | 理由（fit）／接続先 |
|---|---|
| 戦略UI統合シェル（トップメニュー・共通リスト/ダイアログ・データHUD） | 真のギャップ。`GineiUITK`/`OrderOfBattlePanel`(UITKパイロット)の正統進化 |
| 艦艇再設計・技術ツリー・FCS.Cap | `ResearchRules`(未配線)に遊びを与える。階級制限に技術軸を**直交追加**（置換でない） |
| 王道値/覇道値＋主義ドリフト | `FactionState`/`DiplomacyRules` にフィールド追加のみ。評判の新軸＝additive |
| 外交コマンドシェル＋AI提案モーダル | `DiplomacyRules` 状態遷移はある→**操作UIが無い**ギャップを埋める |
| エンディング分岐 | `DisclosureLedger`(FND-4)の条件式に流すだけ |
| 特殊能力10種・作戦適性S〜E・功績→昇進 | `AdmiralSkillRules`/`MeritRankRules`(未配線)の空コンテンツを埋める。独断=`AutonomyRules` 配線 |
| 旗艦隊/提督の作戦適性が全軍へ波及 | `OrderOfBattle`(梯団)に乗る additive な指揮構造ルール |
| 惑星防衛3層（防衛艦隊/衛星/軌道部隊） | 攻城#131 の防御側を具体化（制空権=防衛衛星に対応） |

### ❌ 不採用（後退・タイクン化・重複）

| 不採用要素 | 理由 |
|---|---|
| **ターン制**（今期終了・per-turn制限・行動消費・行動順の手番） | **後退**。当プロジェクトは `GameClock` の連続時間＝潜行で止まらないシームレス時間。手番制への退行は不可 |
| **per-惑星マイクロ内政**（経済投資/治安強化を惑星ごと手動） | **タイクン化**。当プロジェクトは三層集約（惑星⊃星系⊃勢力）で微操作を排除済み。UIシェルだけ採り中身は集約を維持 |
| **戦術4種の抽象戦闘でRTSを置換**（応戦/強行/機動/密集の手番選択） | **後退**。当プロジェクトの会戦は良質なRTS。抽象手番戦闘への置換は不可（※部隊“姿勢”としての将来検討は余地ありだが今回 issue 化しない） |
| **負傷/療養** | 重複。`CaptivityRules`/`LifecycleRules`(捕虜化/死亡)が既に同領域をカバー |
| **報告書/秘書官アシスト** | 重複度高。`NotificationCenter`/`NotificationFeed` が情報集約を担う。今回見送り |
