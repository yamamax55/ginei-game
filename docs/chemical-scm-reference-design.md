# 化学サプライチェーン 参考設計（EPIC #CPL）

> 参照元：化学産業のサプライチェーン管理（SCM）という**実務ドメイン**。石油化学コンビナート型の連続プロセス産業に固有の構造を参考にする。
> 着眼点は一語で **「カップリング（連結）」**。当プロジェクト（Ginei＝銀英伝風の星間国家戦略＋巨大な経済・兵站純ロジック層）は資源・市場・物流を**単品の独立フロー**として持つが、化学SCMが固有に教えるのは**それらが勝手に連結して動く**ことである。
> 注意：固有の企業名・製品名・プロセス名は流用せず、**構造パターン（連結の力学）のみ**を抽象して参考にする。化学反応式・分子モデルの忠実再現はしない（ゲームに無関係）。

---

## 0. なぜ「化学SCM」が本システムに役立つか

当プロジェクトは経済・兵站の**マクロ純ロジックを大量に保有**している：

| 既存（マクロ・抽象） | カバー範囲 |
|---|---|
| `MarketRules`/`Good`/`Market`（#179-182） | **単品ごと**の需給均衡価格・生活水準→支持 |
| `ResourceProductionRules`/`ResourceStockpile`（#92 L-1） | 類型×安定度の資源産出（物資/弾薬/燃料を**独立に**） |
| `SupplyRules`（#94 L-2） | 補給源→所有回廊で到達・敵ZOC遮断・補給切れで枯渇 |
| `CommerceRaidingRules`（#95 L-3） | 護衛vs襲撃・撃破で補給0 |
| `LogisticsRules`（#844） | 版図の一体化度（最大連結成分） |
| `ShipyardRules`/SCM #982（SCM-1〜9） | **SAP/MRP型 BOM・多段生産ツリー**・造船兵站・会計接続 |
| `FiscalRules`/`FiscalState`（#161/162） | 国債/金利/為替・税 |

**しかしこれらは「単品・独立フロー」のマクロ均衡**であり、化学SCMが固有に描く以下が**欠けている**：

| 化学SCMが固有に持つ視点 | 当プロジェクトでの欠落 |
|---|---|
| **連産品（固定比の同時産出）** | 分解1工程で複数製品が**強制的に同時**に出る（エチレン＋プロピレン＋C4…）。資源は独立に生産＝**「欲しいものだけ作れない」連結が無い** |
| **スプレッド（出力−入力のマージン）** | 利益は単品価格でなく**入力市場と出力市場の差**。`MarketRules` は単品均衡で、**垂直の価格連動・原料高で採算が消える**動学が無い |
| **ブルウィップ（多段で需要変動が増幅）** | 前線の小さな需要変動が**上流へ増幅伝播**する。供給は独立フロー＝増幅が無い |
| **連鎖の脆さ（単一ノード遮断のカスケード）** | `CommerceRaidingRules` は「その補給0」止まり。**上流グルット／下流欠品の連鎖**が無い |
| **連続運転の硬直（止められない・稼働率）** | 生産は随意にON/OFF。**turndownコスト・committed稼働**（戦時に止められない）が無い |
| **中間体の貯蔵不能（バッファ無し）** | `Stockpile` は何でも貯まる。**貯蔵できずショックが増幅**する中間体が無い |

**結論**：化学SCMは当プロジェクトの「単品・独立フロー」経済に**カップリング（連結）**という欠落軸を与える——**縦（スプレッド）・横（連産品）・時間（ブルウィップ／脆さ）**。これは**新サイロを足すのでなく、既存物流（#92/#94/#95/#844/#982）を一つの脆い網へ連結する"接着剤"**＝かねて希少だった**統合に効く**。タイクン化（BOM手組み・在庫ミクロ）には**絶対に倒さない**＝係数・高位決断・創発でのみ表現する。

## 1. 役に立つ視点（要約）

化学SCMの構造を、**本システムに効く形**で1行ずつ：

1. **欲しいものだけ作れない（連産品）**。燃料が欲しくて装置を回すと、要らない副産物も同時に出て市場に溢れる＝**高位の決断に必ず副作用が付く**。タイクンの「作りたい物を作る」を否定する戦略的制約。
2. **儲けは価格でなく"差"（スプレッド）**。原料が高騰すれば製品価格据え置きでも採算が消える＝**垂直連動**。単品均衡に「上下の市場が連動する」遊びを足す。
3. **長い鎖は揺れを増幅する（ブルウィップ）**。前線の小さな需要変動・回廊1本の遮断が、上流で過剰と欠品のカスケードになる＝**既存の兵站#94/通商破壊#95/物流#844 を一つの脆い網にする**。
4. **連続プロセスは止められない**。稼働率を保つために走り続け、戦争で1ノードが落ちると鎖全体が立ち往生＝**「戦時経済は簡単に止められない」**コミット。
5. **貯められない中間体がある**。バッファが無いほどショックが直に伝わる＝脆さの増幅器。これは**実務ドメインの暗黙知**であり、本から蒸留するのでなくプレイヤー（実務者）の知見を直接モデル化できる数少ない軸。

## 2. 取り入れるべきメカニクス（優先度つき・既存への接続）

> 大原則：**市場#179/兵站#92/SCM#982/物流#844/財政#161 を作り直さない**。CPL はそれらに**カップリングを足して連結する**だけ（additive）。すべて純ロジック・test-first（EditMode/TestHarness）。

### ★★★ 最優先（カップリングの core）

#### CPL 連産品（固定比の同時産出）
- **連産レシピ**：1工程が複数 `ResourceType` を**固定比で同時産出**（新 `CoupledProductionRules`／レシピ定義 `JointRecipe`）。稼働量を上げると全製品が比率どおり増える＝**主産物が欲しくても副産物が必ず付く**。
- 接続：`ResourceProductionRules`（産出の連産版オーバーロード）/`ResourceStockpile`。**基準は非破壊**（既存の単品産出は残す）。
- 純ロジック・test-first。

#### CPL 副産物グルット→価格暴落（連産×市場）
- 連産で**強制的に出た副産物**が市場に溢れ、`MarketRules` の需給均衡価格を押し下げる（グルット）。**主産物のために装置を回すほど副産物市場が崩れる**＝意思決定のトレードオフ。
- 接続：`CoupledProductionRules`×`MarketRules`（供給増→`ClearingPrice` 低下）。高位の決断＋創発帰結。

#### CPL スプレッド（マージン＝出力−入力・原料高で採算消失）
- 採算を**単品価格でなく「出力市場価格 −（投入係数×入力市場価格）」のスプレッド**で評価（新 `SpreadRules`）。原料高騰でスプレッドが負に振れると**稼働するほど赤字**＝止めるか market share を守って耐えるかの決断。
- 接続：`MarketRules`（入力/出力の価格）×`FiscalRules`（採算→国庫・`FiscalHealthFactor`）。

#### CPL ブルウィップ（多段で需要変動が上流へ増幅）
- 多段の補給鎖で**前線（下流）の需要変動が上流へ行くほど増幅**する（新 `BullwhipRules`＝段ごとの増幅係数）。小さな前線スパイク／回廊遮断が上流で**過剰と欠品**を生む。
- 接続：`SupplyRules`（補給鎖の段）×`CommerceRaidingRules`（遮断ショックの注入源）。**既存兵站を連結する接着剤**＝統合に最も効く。

### ★★ 高（脆さと硬直＝③を噛ませる）

#### CPL 連鎖の脆さ（単一ノード遮断のカスケード）
- 通商破壊／回廊遮断を「その補給0」で終えず、**上流＝行き場を失いグルット、下流＝欠品**として**鎖に沿って伝播**させる（`CommerceRaidingRules`×`LogisticsRules` の連結判定を利用）。単一ノードが鎖全体を止める single-point fragility。
- 接続：`CommerceRaidingRules`/`SupplyRules`/`LogisticsRules`。`BullwhipRules` の topological 版。

#### CPL 連続運転の硬直（稼働率・turndownコスト）
- 連続プロセスは**随意に止められない**＝稼働率を保つ固定コスト、停止/再起動に turndown コスト。戦争で1ノードが落ちると**committed な鎖が立ち往生**＝「戦時経済は簡単に止められない」。
- 接続：`ResourceProductionRules`（稼働率係数）×`FiscalRules`（固定コスト/turndown は国庫ドレイン）。

### ★ 中（バッファ＝脆さの増幅器）

#### CPL 中間体の貯蔵不能（バッファ無し＝ショック増幅）
- 一部の中間体は**貯蔵不能**＝即時消費しないと損失（`ResourceStockpile` に「no-buffer」属性）。バッファが無いほど②③のショックが**減衰せず直に伝わる**。
- 接続：`ResourceStockpile`（貯蔵不能フラグ）×`BullwhipRules`（バッファ＝増幅の減衰項）。

### ❌ 不採用（重複・タイクン化）

| 不採用 | 理由 |
|---|---|
| BOM／MRP／多段生産ツリーの手組み・生産スケジューリング | **SCM#982（SCM-1〜9）が既にカバー**。CPL は作り直さず**連結だけ**足す |
| 在庫管理・配車・発注点のミクロ操作 | **タイクン化回避**（CLAUDE.md「タイクン化回避＝通貨/多段生産ツリー無し」＝`ResourceStockpile`/`ShipyardRules` の方針）。係数・創発でのみ表現 |
| 個別プラントの建設マイクロ | 造船#884／メガプロジェクト PIL#1081 が別途カバー。CPL は物理建設を持たない |
| 化学反応式・分子・収率の忠実再現 | ゲームに無関係。**連産比という抽象**だけ採る |
| 通貨／多段通貨経済 | タイクン回避。スプレッドは既存 `MarketRules`/`FiscalRules` の価格で表現 |

## 3. EPIC #CPL の子Issue（採用分のみ・着手順）

> 純ロジックは TestHarness/EditMode で先に固定（test-first）→ 盤面/UI へ配線。既存の兵站・市場・財政ロジックは**接続のみ・重複新設しない**。
> **盤面接続する state 型を新設する子は完了条件に「観測層への追従」（CLAUDE.md「観測層」節：用語集1行・独立新ルートなら `CoreStateInspector.Register`）も含める。**

> **EPIC = #1109**。GitHub issue 起票済み（#1110〜#1116）。

| # | issue | タイトル | 接続先 / 主眼 |
|---|---|---|---|
| **CPL-1** | #1110 | 連産品（固定比の同時産出 `CoupledProductionRules`/`JointRecipe`） | `ResourceProductionRules`/`ResourceStockpile`。欲しい物だけ作れない |
| **CPL-2** | #1113 | 副産物グルット→価格暴落（連産×市場） | `CoupledProductionRules`×`MarketRules`（供給増→`ClearingPrice`↓） |
| **CPL-3** | #1111 | スプレッド（マージン＝出力−入力・原料高で採算消失 `SpreadRules`） | `MarketRules`×`FiscalRules` |
| **CPL-4** | #1114 | ブルウィップ（多段で需要変動が上流へ増幅 `BullwhipRules`） | `SupplyRules`×`CommerceRaidingRules`。兵站連結の接着剤 |
| **CPL-5** | #1112 | 連鎖の脆さ（単一ノード遮断の上流グルット/下流欠品カスケード） | `CommerceRaidingRules`/`SupplyRules`/`LogisticsRules` |
| **CPL-6** | #1115 | 連続運転の硬直（稼働率・turndownコスト＝止められない戦時硬直） | `ResourceProductionRules`×`FiscalRules` |
| **CPL-7** | #1116 | 中間体の貯蔵不能（バッファ無し＝ショック増幅） | `ResourceStockpile`×`BullwhipRules` |

### 推奨着手順
`CPL-1 → CPL-2`（連産品＝化学SCM固有で最も欠落の大きい signature・横の連結）→ `CPL-3`（スプレッド＝縦の連結）→ `CPL-4 → CPL-5`（ブルウィップ→脆さ＝時間の連結・既存兵站の統合接着剤）→ `CPL-6`（連続運転の硬直）→ `CPL-7`（貯蔵不能＝脆さの増幅器）。

> いずれも既存マクロ経済・兵站を**後退させず接続**する additive 設計。**カップリング＝既存物流（#92/#94/#95/#844/#982）を一つの脆い網に統合する**のが核で、新サイロを足すのではない。
