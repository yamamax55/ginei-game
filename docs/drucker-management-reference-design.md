# ドラッカー『マネジメント』参考設計（EPIC #MGMT）

> 参照元：P.F.ドラッカー『マネジメント — 課題・責任・実践』（経営学の古典）。
> 組織を「成果を上げるための社会的器官」として捉え、**①組織の目的（顧客の創造・事業の定義） ②知識労働者の生産性 ③連邦分権（事業部の自律と本社の戦略）**を中核に置く経営理論。
> 本ドキュメントは、当プロジェクト（Ginei＝銀英伝風の星間国家戦略＋既に巨大な統治・人事・組織純ロジック層）にとって**役に立つ視点**だけを抽出し、EPIC `#MGMT` として issue 化する提案。
> 着眼点：**組織の目的・知識労働者・連邦分権**（省庁#158/HCM#992 の経営理論的裏打ち）。
> 著作権注意：固有名・文章・固有の図表・固有設定は流用せず、**経営メカニクス／組織論の構造パターンのみ**を参考にする。

## 0. なぜ「マネジメント」が本システムに役立つか

当プロジェクトは組織・人事・統治の**純ロジックを大量に保有**している（[CLAUDE.md]「テスト基盤」節 参照）：

| 既存（組織・人事・統治） | カバー範囲 |
|---|---|
| `MinistryRules`/`Ministry`（GOV-5 #158） | 省庁ツリー・配属・省益（縦割り強度）・臨時官庁 |
| `Organization`/`SuccessionRules`（#812） | 結束/制度化/個人カリスマ・英雄死後の組織存続・制度化投資 |
| `OfficeRules`/`Office`/`GovernmentRegistry`/`ICharacter`（GOV-1/3） | 役職・任命台帳・就任資格・提案権限 |
| `PersonRules`/`Person`（正名 #866） | 軍人/文民の役割×役職一致・適材適所・`BestFor`自動配属 |
| `CareerPipelineRules`/`CareerTrack`（LIFE-5/6/7） | 出自パイプライン（武/官/技）・学閥/専門閥の結束 |
| `GovernanceRules`/`Province`（#109） | 安定度収束・占領統合・産出係数（OutputFactor） |
| `CampaignState`/`FactionState`（統合層） | 王朝/統治体/組織/共同体の合成・実効安定度 |
| `MeritRankRules`（始皇帝 #900-905） | 戦功→爵位・インセンティブ士気・抽出型の長期崩壊 |
| `FiscalRules`/`FiscalState`（#161/162） | 歳入歳出・財政健全度係数 |
| `DisclosureRules`/`DisclosureLedger`（FND-4） | lore データ入力（条件→開示） |

**しかし、これらは「組織の構造（誰がどの席に座るか）と安定度のマクロ均衡」**であり、ドラッカーが固有に描く以下が**欠けている**：

| ドラッカーが固有に持つ視点 | 当プロジェクトでの欠落 |
|---|---|
| **事業の定義＝「われわれの事業は何か」**（顧客は誰か・何に価値を見るか） | 勢力に**目的・ミッション**が無い。`FactionState` は安定度の合成だが「何のための組織か」が空白＝目的喪失/再定義のドラマが無い |
| **目標管理（MBO）＝成果領域に目標を立て自己統制で達成** | 役職は配属されるが**目標を持たない**。`Office` は権限ツリーで、達成すべき成果（目標 vs 実績）の回路が無い |
| **知識労働者の生産性＝成果は産出量でなく「貢献」で測る** | `Person` は能力値はあるが、**知識労働者の生産性**（動機・自己管理・成果ベース）という別軸が無い。`OutputFactor` は土地（安定度）の係数で人の貢献ではない |
| **連邦分権＝事業部が自律的に成果責任を負い、本社は戦略に専念** | `MinistryRules` は縦割りツリー（権限の階層）。**事業部の自律 vs 本社統制のトレードオフ**（分権の度合い・本社肥大）が無い |
| **マネジメントの正統性＝成果が権限を正当化する** | 正統性は `Regime`/`Polity`（天命/合意）で供給されるが、**「成果を上げない管理者は正統性を失う」**という経営的正統性が無い |
| **イノベーションと体系的廃棄＝古い事業を捨てて新しい機会へ** | `ResearchRules` は技術ツリーだが、**「成果なき活動の計画的廃棄」**（資源を死んだ事業から引き上げる決断）が無い |
| **マネジメントの病＝官僚制の自己目的化・成果と無関係な肥大** | `Ministry.institutionalInterest`（省益）はあるが、**組織の肥大が成果を侵食する病理**（手段の目的化）の動学が薄い |

**結論**：ドラッカーは当プロジェクトの組織構造に**「成果のための器官」という目的論的な顔**と、**①事業の定義（ミッション） ②目標管理（成果 vs 実績） ③知識労働者の生産性 ④連邦分権（自律 vs 統制）**という4つの欠落軸を与える。そして**省庁#158（縦割りツリー）／HCM#992（人材）／`Organization`#812（制度化）**を**「成果で測られる組織」へ昇格**させ、タイクン化せずに**目的喪失・官僚制肥大・分権の創発帰結**を生む。

## 1. 役に立つ視点（要約）

ドラッカーの経営理論を、**本システムに効く形**で1行ずつ：

1. **「組織は成果のために存在する」＝構造でなく成果が目的**。誰が座るかでなく、何を成し遂げたかで組織を測る。→ `Office`/`Ministry`/`FactionState` を**目標と実績で評価する層**へ昇格（省庁#158/組織#812 への共鳴）。
2. **「われわれの事業は何か」＝目的の定義が戦略の起点**。目的が曖昧/陳腐化した組織は迷走する。→ 勢力に**ミッション（事業の定義）**を与え、その達成度が正統性・支持へ波及（`FactionState`×`Regime`正統性）。
3. **目標管理と自己統制（MBO）＝上から命じず、各部署が成果領域に目標を立て自ら測る**。→ `Office`/`Ministry` に**成果目標**を持たせ、達成/未達が人事・正統性を動かす（GOV-5 #158 への共鳴）。
4. **知識労働者は管理でなく動機で動く＝成果は貢献で測る**。→ `Person`/HCM#992 に**貢献ベースの生産性**軸を足す（適材適所#866 の動機版・MeritRank#900 の非抽出版）。
5. **連邦分権＝事業部は自律的に成果責任を負い、本社は戦略に専念する**。→ `MinistryRules` 縦割りに**分権度のトレードオフ**（自律で機動↑だが統制↓・本社肥大）を足す。
6. **マネジメントの正統性は成果が与える＝成果なき管理者は職を失う**。→ `RetirementRules`/`VacancyRules` の解任に**成果ベースの根拠**を接続（無能の制度的退場）。
7. **イノベーションと体系的廃棄＝成果なき活動を計画的に捨て資源を機会へ**。→ `ResearchRules`/予算配分に**廃棄の決断**を足す（官僚制の自己延命への対抗）。
8. **マネジメントの病＝官僚制は放置すると自己目的化し成果を侵食する**。→ `Ministry.institutionalInterest`（省益）を**肥大の病理動学**へ（手段の目的化＝タイクン回避の警鐘そのもの）。

## 2. 取り入れるべきメカニクス（優先度つき・既存への接続）

> 大原則：**省庁#158/組織#812/役職GOV-1/人事#866/HCM#992 を作り直さない**。MGMT はそれらに**「成果・目的・分権」の軸を足し、接続する**だけ（additive）。
> タイクン化回避：マイクロ管理を増やさない＝**高位の決断（ミッション設定・分権度・廃棄）→エンジン駆動→創発帰結（目的喪失・官僚肥大・正統性失墜）**。

### ★★★ 最優先（真の欠落・ドラッカーの signature）

#### MGMT 事業の定義（ミッション）と目的喪失（「われわれの事業は何か」）
- 各勢力／組織が**ミッション（事業の定義）**を持つ：顧客（誰のための組織か＝住民/軍/支持層）×価値（何を提供するか）。ミッションと**実態の乖離**が広がると目的喪失＝正統性低下・迷走。
- **目的の陳腐化**：成功は目的を風化させる（かつての目的が達成/陳腐化）→**再定義**を迫られる（しないと衰退）。
- 接続：新 `MissionRules`/`Mission`（純ロジック・test-first）。`FactionState`×`Regime`正統性に**目的整合度**を足す。**戦略の起点**＝以降の目標管理（MGMT-2）が乗る土台。

#### MGMT 目標管理（MBO）＝成果領域の目標 vs 実績
- 各役職/省庁（`Office`/`Ministry`）が**成果目標**（自軍維持/占領/財政均衡/技術等の数値目標）を持ち、**実績との差分**で評価される。
- **自己統制**：上から命じず各部署が目標を立て自ら測る＝達成は正統性↑、慢性未達は人事（解任/降格）と支持低下を呼ぶ。
- 接続：新 `ObjectiveRules`/`PerformanceTarget`（純ロジック・test-first）。`OfficeRules`/`GovernmentRegistry`×`MinistryRules` に**目標と実績の層**を重ねる。GOV-5 #158（省庁）に「成果で測る」回路を入れる核。

### ★★ 高（人と分権に成果軸を足す）

#### MGMT 知識労働者の生産性（貢献で測る・動機で動く）
- `Person` に**貢献ベースの生産性**を足す：成果は産出量でなく**貢献**（役割一致×動機×自律裁量）で決まる。管理を強めると下がり、目的整合と裁量で上がる。
- 接続：新 `KnowledgeWorkRules`（純ロジック・test-first）。`PersonRules`#866（適材適所）の動機版＋`AutonomyRules`（自律分散）×`MeritRankRules`#900（抽出 vs 動機）と接続。HCM#992 の生産性の裏打ち。基準値非破壊（実効値パターン）。

#### MGMT 連邦分権（事業部の自律 vs 本社統制・本社肥大）
- `Ministry`/方面組織に**分権度**を持たせる：高分権＝現場の機動・成果責任が明確だが全体統制↓・遠心力（分離#194 リスク）。低分権＝統制は効くが本社肥大・現場の硬直。
- **本社肥大**：中央が現場の仕事を抱え込むと官僚制が膨れ成果が落ちる。
- 接続：`MinistryRules`（省庁ツリー）×`FeudalRules`#168（封建＝分権の極）×`Culture`分離#194。**高位の決断（分権度）→創発（機動 vs 遠心力）**。

#### MGMT マネジメントの正統性＝成果が権限を正当化する
- 成果（MGMT-2 の目標達成）が管理者の**正統性**を生む。慢性未達の管理者は正統性を失い**制度的に退場**（解任/後任補充）。
- 接続：`RetirementRules`/`VacancyRules`（後任補充）に**成果ベースの解任根拠**を接続＋`Regime`/`Polity`正統性へ波及。`CivilianControlRules`#145（クーデター）の火種にも（成果なき統制弱）。

### ★ 中（廃棄・病理・lore）

#### MGMT イノベーションと体系的廃棄（成果なき活動を計画的に捨てる）
- **計画的廃棄**：成果を生まなくなった事業/部署/研究に資源を残さず、機会へ振り向ける高位の決断。廃棄しない組織は死んだ事業に資源を食われる。
- 接続：`ResearchRules`（研究の廃棄/再配分）×`MinistryRules.Dissolve`（臨時官庁の廃止＝既存窓口）×予算配分。新規ルールは最小（既存の廃止/再配分窓口に「成果トリガ」を足す）。

#### MGMT マネジメントの病＝官僚制の自己目的化（手段の目的化）
- `Ministry.institutionalInterest`（省益）を**肥大の病理動学**へ：放置/低分権/廃棄なしで省益が積み上がり、成果（MGMT-2）を侵食＝手段（組織維持）が目的化。
- 接続：`MinistryRules`（`SectionalismFriction`/`institutionalInterest`）×MGMT-2 目標。**新規ルール最小・既存の省益に動学を足す**。タイクン化回避の警鐘そのものを game 化。

#### MGMT（lore）経営思想の開示データ
- 「組織は成果のために存在する」「目的を問い続けよ」「知識労働者は資本でなくパートナー」「廃棄こそ革新の母」。
- 接続：**コード新設せず** `DisclosureLedger`（FND-4）への**lore データ入力**。世界観codex退避方針に一貫。

### ❌ 不採用（重複・既存で十分・タイクン化リスク）

| 不採用 | 理由 |
|---|---|
| 省庁ツリー/配属/省益そのもの | **`MinistryRules`#158 が既にカバー**。MGMT は作り直さず「成果・分権・病理」を接続 |
| 役職・任命・就任資格・提案権限 | **`OfficeRules`/`GovernmentRegistry`#GOV-1/3 がカバー** |
| 適材適所・軍人/文民の役割一致 | **`PersonRules`#866 がカバー**。MGMT は「貢献ベース生産性」のみ足す |
| 制度化投資・カリスマの継承 | **`Organization`/`SuccessionRules`#812 がカバー**。MGMT は目的論（成果）軸で接続 |
| 財政・予算編成の数値モデル | **`FiscalRules`#161/162 がカバー**。MGMT は廃棄/再配分の決断のみ |
| 人事評価を**個人マイクロ操作**で回す（査定UI/人事ミニゲーム） | **タイクン化＝不採用**。高位（ミッション/分権/廃棄）の決断→エンジン駆動に限定 |
| 顧客満足/マーケティングの市場シミュ | 当プロジェクトは戦略×軍事＝**市場#179/支持#113 に集約済み**。経営の「顧客」は住民/支持層へ写像 |
| 「全部入り」の経営シミュ化 | ドラッカーの**核（目的・成果・知識労働者・分権）のみ**取捨選択。残りは既存層が担う |

### 推奨着手順
`MGMT-1`（事業の定義＝最も固有で全ての土台）→ `MGMT-2`（目標管理＝成果 vs 実績の核）→ `MGMT-3`（知識労働者の生産性＝人の成果軸）→ `MGMT-4`（連邦分権＝自律 vs 統制）→ `MGMT-5`（成果ベース正統性＝退場）→ `MGMT-7`（官僚制の病＝省益肥大）→ `MGMT-6`（体系的廃棄）→ `MGMT-8`（lore）。
