# Phase C 後半 設計メモ（C-4 並行 / C-7 要塞 / C-8 自動解決）

> Phase C コア（C-1〜C-3＝単一戦線シームレス1周）の上に、**複数戦線・並行時間・固定拠点・非指揮戦線の処理**を載せる。
> 前提：`docs/phase-c-core-design.md`（`GalaxyManager`/`StrategicFleet`/`CorridorBattleHost`/`BattlefieldBounds`）。
> 対象 Issue：**C-4 #37（最難関）／C-5 #38（援軍）／C-6 #39（補給）／C-7 #40（要塞）／C-8 #41（自動解決）**。本書は C-4/C-7/C-8 を主に扱い、C-5/C-6 は接続点のみ示す。
> ★ 最難関。**C-1〜C-3 が安定してから着手**。Git ブランチ＋段階実装、各段でコンパイル＆単一戦線の無事を確認。

## 0. 確認したい論点（推奨デフォルトで記述）
1. **並行の最小形**：推奨＝**「ライブ戦術1戦線＋抽象N戦線」**。同時に**個艦シミュレーションするのは1戦線だけ**（プレイヤーがズームインしている戦線）。他戦線は **C-8 抽象解決**で進む。→ `FleetRegistry`（静的単一）の大改修を回避できる。…これで始めてよい？
2. **銀河時間と戦術時間**：推奨＝**銀河クロックは常時進行**（戦闘中も）。ライブ戦術は実時間で同時進行。`PauseManager` の Space=全停止／1-3=倍速は**両方に効く**。…OK？
3. **要塞の実体**：推奨＝**固定の超強艦**として既存 `FleetStrength`/`FleetWeapon` を流用（`FleetMovement`/`FleetAI` 無効・巨大兵力・長射程）。撃破/制圧で支配反転。…この流用でよい？

> 以降は上記「推奨」で記述。違えば差し替える。

## 1. C-4：リアルタイム並行（銀河時間は戦闘中も進行）＋複数戦線 — #37
**狙い**：銀河時間を止めずに進め、複数回廊で同時に戦闘が起こり得る。プレイヤーは1戦線を手動指揮、他は自動解決/抽象進行。

### 1-1. 時間モデル
- **銀河クロック `GalaxyManager.galaxyTime`**：常時 `+= Time.deltaTime`（timeScale 追従＝ポーズで停止・倍速で加速）。ワープ移動・産出（兵站L）・抽象戦線の解決はこの時間で進む。
- **ライブ戦術戦闘**：従来どおり実時間で進む（同一 `Time.deltaTime`）。銀河と戦術が**同じ時間軸で同時進行**＝「戦闘中も銀河が動く」。
- **ポーズ/倍速**：`PauseManager` をそのまま使い、`Time.timeScale` で銀河・戦術の双方を制御。

### 1-2. 「ライブ1戦線＋抽象N戦線」アーキテクチャ（最小プロト）
- 同時に**個艦を spawn して戦うのは1戦線**（=プレイヤーがズームインしている回廊）。`CorridorBattleHost` がその1戦場だけを生成・保持。
- 他の交戦回廊は **抽象戦線**（個艦を作らず、`StrategicFleet` の兵力を C-8 が時間で削り合う）。
- **戦線スイッチ**：別の交戦回廊へズームイン→ 現ライブ戦線を**スナップショットして抽象化**（生存兵力を `StrategicFleet` へ書き戻し、`FleetRegistry.Clear()`）、新戦線をライブ生成。→ `FleetRegistry` は**常に1戦場分**で済む（静的単一のまま）。
- これにより **C-4 の最難関＝複数同時個艦シミュレーションを回避**しつつ「並行」を体感させる。

### 1-3. 将来の本格並行（必要になったら）
- 真に複数戦場を同時個艦シミュレートするには **`FleetRegistry` の戦場スコープ化**（`Battlefield` がローカル在庫を持つ／ship に battlefieldId）と `ShipCombat` への文脈受け渡しが要る＝大改修。**プロトでは採らない**。本書はフック（戦場スナップショット境界）だけ用意。

## 2. C-7：要塞（イゼルローン型）と回廊の固定拠点 — #40
**狙い**：回廊に居座る固定強拠点。通過には撃破/制圧が必要、壁＋裏エッジ不在で迂回不可。フェザーン型（要塞なし通商回廊）との差も表現。

- **データ**：`CorridorData.hasFortress`(bool・コアで予約)＋ `fortressOwner`(`FactionData`)／`fortressStrength`／`fortressRange`。
- **戦略レイヤー**：要塞のある回廊は、**要塞を中立化（撃破/制圧）するまで通過不可**＝通行ゲート。ZOC（#100）と連動し、敵要塞のある回廊は補給線（L-2）も通せない。迂回は壁＋裏エッジ不在で不可。
- **戦術レイヤー（要塞戦）**：当該回廊にズームインすると、戦場の防御側端に **要塞エンティティ**を配置：
  - 既存 `FleetStrength`（巨大 `strength`）＋`FleetWeapon`（長射程・高火力）を**固定**で持つ（`FleetMovement`/`FleetAI` を付けない or 無効）。`IShipTarget` として攻撃対象になる。
  - **制圧**：要塞 strength を 0 にすると撃破/占領＝`fortressOwner` 反転 or 消滅。攻撃側が通過権を得る。
  - 既存の被弾・退却ロジックは要塞では「退却しない（不動）」に。`BeginRetreat` を抑止し、0 で撃破処理。
- **フェザーン型**：`hasFortress=false` の通商回廊。要塞戦は無く、通商破壊（L-3）と機動戦が主になる＝**回廊ごとの性格の差**。
- **【要・作者判断】**：要塞の兵力/射程、制圧＝破壊か占領か、要塞の再建可否。

## 3. C-8：自動解決（非指揮戦線） — #41
**狙い**：プレイヤーが戦術指揮していない戦線を、戦力・補給・地形で簡易解決（または膠着保持）。ズームインで手動指揮に引き継げる。

- **入力**：各陣営の戦闘力 ＝ Σ(`StrategicFleet.兵力` × 提督係数 × 補給係数(L-2) × 地形係数(要塞/ハザード))。
- **解決 `AutoResolver.Tick(corridorBattle, dt)`**：両陣営の兵力を**戦闘力比に応じて時間で削り合う**（dt 比例）。
  - 片方が 0 → **決着**（勝者が回廊/星系の支配を取得）。`BattleManager` の勝敗思想（`HasHostilePair`/`DetermineWinner` の抽象版）を流用。
  - 拮抗 → **膠着**（ゆっくり相互消耗。補給切れ＝L-2 断で一方が崩れるまで保持）。
- **手動引き継ぎ**：抽象戦線にズームイン→ 現在の兵力スナップショットから `BattleSetup`（ランタイム生成パス）で**個艦戦闘を立ち上げ**（C-3）。ズームアウト→ 兵力をスナップショットして抽象へ戻す（§1-2 の戦線スイッチと同一機構）。
- **C-4 とセット**：C-4 で並行が入ると、指揮していない戦線の処理が必須＝**C-8 はほぼ必須**。
- **【要・作者判断】**：戦闘力の式の重み（提督/補給/地形）、消耗速度、膠着の判定幅。

## 4. C-5（援軍）/ C-6（補給）の接続点（本書範囲外・参照のみ）
- **C-5 援軍 #38**：戦場の自陣側**進入口**（コア設計 §5）から、ワープ時間後に増援 `StrategicFleet` がワープイン。抽象戦線でも兵力加算で表現。
- **C-6 補給 #39**：継戦で消費、L-2 の補給線からリフィル。抽象戦線の補給係数・ライブ戦闘の消費に効く。詳細は `docs/wartime-logistics-design.md`。

## 5. 既存/コア コードとの接続
| 対象 | 後半での使い方 |
|---|---|
| `GalaxyManager` | `galaxyTime` 常時進行、抽象戦線の `AutoResolver` 駆動、戦線スイッチの司令塔。 |
| `CorridorBattleHost`（C-3） | ライブ1戦線の生成/破棄＋スナップショット境界（抽象↔ライブ）。 |
| `FleetRegistry` | プロトでは**単一のまま**（ライブ1戦線分）。本格並行で戦場スコープ化（将来の大改修）。 |
| `BattleSetup`（ランタイム生成） | 抽象→ライブ引き継ぎ時の個艦生成。 |
| `BattleManager`（勝敗） | ライブ戦線の決着＋抽象解決の勝者判定の思想を共有。 |
| `FleetStrength`/`FleetWeapon` | 要塞エンティティに流用（固定・巨大・長射程、退却抑止）。 |
| `PauseManager` | 銀河＋戦術の双方の停止/倍速。 |
| `FactionRelations` | 交戦判定・支配反転・ZOC の敵味方判定の唯一の窓口。 |

## 6. 段階手順・リスク
1. **C-4a（時間）**：`galaxyTime` 常時進行＋単一ライブ戦線が銀河と同時進行（複数戦線はまだ）。
2. **C-8（抽象解決）**：`AutoResolver`（戦力/補給/地形で消耗）＋膠着。可視化（戦略ビューに戦線状況）。
3. **C-4b（複数戦線＋スイッチ）**：交戦回廊を複数許可。ライブ1＋抽象N、ズームで戦線スイッチ（スナップショット往復）。
4. **C-7（要塞）**：要塞データ＋通行ゲート＋戦術の要塞エンティティ。
5. **C-5/C-6** を上記の上に載せる（援軍ワープイン・補給リフィル）。

- **リスク（最大）**：`FleetRegistry` 単一前提。プロトは「ライブ1戦線」で回避するが、**真の同時個艦並行へ進む判断をしたら戦場スコープ化の大改修が必要**＝早期に方針確定を。
- **リスク**：スナップショット往復（抽象↔ライブ）で兵力・状態の整合が崩れないこと（変換は一箇所に集約）。
- **リスク**：銀河時間と戦術時間の同時進行で体感速度が破綻しないか（倍速時の戦術操作性）。
- **【要・作者判断】**：並行の最小形の採否、要塞パラメータ、自動解決の式、銀河クロックの基準速度。
