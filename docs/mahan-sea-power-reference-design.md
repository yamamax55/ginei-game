# マハン『海上権力史論』参考設計（EPIC #MHAN）

> 参照元：アルフレッド・セイヤー・マハン『海上権力史論（The Influence of Sea Power upon History）』。
> 海洋を「広大な共有地（コモンズ）」と捉え、**海上交通路の支配（command of the sea）こそが国力の真の源泉**であると説いた古典的海軍戦略理論。
> 中核命題＝①**シーパワーは地理・国民性・政府の性格から導かれる国家潜在力**である ②**艦隊決戦による制海権の獲得**がすべての海洋戦略の目的である ③制海権の目的は**通商の保護**であり、通商破壊（commerce raiding）は決定打にならない ④海上交通路の**チョークポイント（要衝）**の支配が国力を増幅する。
> 本ドキュメントは、当プロジェクト（Ginei＝銀英伝風の星間国家戦略・銀河グラフ＝海図／回廊＝海上交通路）にとって**役に立つ視点**だけを抽出し、EPIC `#MHAN` として issue 化する提案。
> **著作権注意**：固有名・文章・固有の歴史記述・図表は流用せず、**戦略メカニクス／海軍戦略の構造パターンのみ**を参考にする。

## 0. なぜ「海上権力史論」が本システムに役立つか

当プロジェクトは、マハンの海図に相当する盤面（銀河グラフ）と、海戦の周辺要素を**既に大量に保有**している（[CLAUDE.md] 参照）：

| 既存（カバー範囲） | 該当モジュール |
|---|---|
| 海図＝ノード/エッジ・経路探索 | `GalaxyMap`/`Corridor`/`GalaxyPathfinder` |
| **チョークポイント概念（要衝の型）** | `Corridor.CorridorType{通商,要衝}`（型は存在・効果未配線） |
| 海上交通路の遮断＝前線 | `StrategyRules.IsFtlBlocked`（端点が敵対＝FTL不可）・亜光速侵入 |
| 補給線（補給源→所有回廊で到達・遮断で枯渇） | `SupplyRules`（L-2 #94） |
| **通商破壊（護衛 vs 襲撃）** | `CommerceRaidingRules`（L-3 #95） |
| 版図の地理的一体化（分断は国力を出し切れない） | `LogisticsRules`（GEO-3 #844） |
| 自動解決＝ランチェスター二乗則 | `AutoBattleSim`（TIME-4） |
| 支配領域（ZOC）の投射・侵入度 | `ZoneOfControl`（#81） |
| 国家潜在力（安定/結束/正統性の合成） | `FactionState`/`CampaignState` |

**しかし、これらは「個別要素の存在」であり、マハンが体系として描く以下が欠けている**：

| マハンが固有に持つ視点 | 当プロジェクトでの欠落 |
|---|---|
| **シーパワー＝地理・国民性・政府性格から導く国家潜在力** | `FactionState` は社会/政治の合成。**海洋指向（地理的アクセス・通商依存度・海軍意志）から艦隊潜在力を導く係数**が無い |
| **艦隊決戦 vs 現存艦隊（fleet-in-being）のドクトリン対立** | 戦闘解決は `AutoBattleSim`/`StrategyRules` にあるが、**「決戦を求める／温存して脅威で在り続ける」という戦略ドクトリンの選択**が無い |
| **制海権＝海上交通路の支配を勢力間で争うグラデーション** | `IsFtlBlocked` は端点所有の二値。**交通路（回廊網）の制海権が争われ、優勢度が連続値で推移する状態**が無い |
| **チョークポイントの戦略的増幅（要衝を握ると国力が増幅）** | `CorridorType.要衝` は型のみ。**要衝の支配が通商/補給/制海に与える増幅効果**が未配線 |
| **集中の原理（戦闘艦隊を分割しない）** | 編制（`OrderOfBattle`）はあるが、**分散すると各個撃破され集中すると優勢になる戦略的ペナルティ/ボーナス**が無い |
| **前進基地・補給拠点（石炭ステーション）が到達圏を投射** | `SupplyRules` は補給源からの到達。**前方拠点が艦隊の作戦行動半径を延ばす投射ノード**という概念が無い |
| **通商保護こそが艦隊の目的（commerce raiding は補助）** | `CommerceRaidingRules` は襲撃解決。**通商保護を国家目標化し制海権と結ぶ「なぜ艦隊を持つか」の動機回路**が薄い |

**結論**：海上権力史論は当プロジェクトの海図盤面に、**①シーパワー国力導出 ②決戦/現存艦隊ドクトリン ③制海権グラデーション ④チョークポイント増幅**という4本柱の欠落軸を与える。いずれも既存の `Corridor`/`StrategyRules`/`CommerceRaidingRules`/`FactionState` に**接続するだけ**で、戦争でなく**地理と通商で国力が決まる**マクロ層を完成させる。

## 1. 役に立つ視点（要約）

海上権力史論の理論を、**本システムに効く形**で1行ずつ（既存issue/モジュールへの共鳴を明記）：

1. **海は広大な共有地で、その交通路の支配が国力を決める**。→ 銀河グラフ＝海図、回廊＝海上交通路という当プロジェクトの盤面構造そのものを戦略的に意味づける。`LogisticsRules`（版図一体化 #844）の上位概念。
2. **シーパワーは6要素（地理的位置/海岸線/領土範囲/人口/国民性/政府性格）から導かれる潜在力**である。→ `FactionState`/`CampaignState` に**海洋指向係数**を足し、艦隊建造・投射の上限を地理と国民意志から導く。`ShipyardRules.ProductionFactor`（建艦力）と共鳴。
3. **海洋戦略の目的は艦隊決戦による制海権の獲得**である。劣勢側は決戦を避け「現存艦隊」として脅威で在り続ける。→ **決戦/温存ドクトリンの選択**を `AutoBattleSim`/`StrategyRules` に上乗せ。臆病な強者・しぶとい弱者の戦略を表現。
4. **制海権の目的は通商の保護であり、通商破壊は決定打にならない**。→ `CommerceRaidingRules`（#95）に**「raiding は嫌がらせ・制海権こそ本命」という非対称**を与え、通商保護を国家目標化（`VictoryCondition` 護衛と共鳴）。
5. **チョークポイント（要衝）の支配は国力を増幅する**。狭隘な海峡を握る者が海を制す。→ `Corridor.CorridorType.要衝`（型のみ存在）に**増幅効果**を配線。`ZoneOfControl`（#81）×制海権。
6. **戦闘力は集中によって最大化される（戦力分散の戒め）**。艦隊を分けると各個撃破される。→ `OrderOfBattle`（#147 編制ツリー）×戦略移動に**集中ボーナス/分散ペナルティ**。`AutoBattleSim` のランチェスター二乗則（集中の数学的根拠）と理論的に一致。
7. **前進基地・補給拠点が艦隊の到達圏を投射する**。遠い海で戦うには中継拠点が要る。→ `SupplyRules`（#94）に**前方拠点ノード**を足し、作戦行動半径を地理的に制約。
8. **（lore）海上権力こそ歴史の真のエンジン**＝大陸国家は海洋国家に敗れる。→ `DisclosureLedger`（FND-4）への lore データ入力（コード新設なし）。王道路線・フェザーン#160 と共鳴。

## 2. 取り入れるべきメカニクス（優先度つき・既存への接続）

> 大原則：**`Corridor`/`StrategyRules`/`CommerceRaidingRules`/`SupplyRules`/`AutoBattleSim`/`LogisticsRules` を作り直さない**。MHAN はそれらに**欠落軸を足し、接続する**だけ（additive）。タイクン化回避＝高位の決断（ドクトリン選択・要衝への進出）→エンジン→創発帰結（制海権の推移・各個撃破）。

### ★★★ 最優先（真の欠落・海上権力史論の signature）

#### MHAN シーパワー国力導出（地理・通商依存・海軍意志→艦隊潜在力）
- **海洋指向係数**：勢力の地理的アクセス（所有星系の回廊接続度＝海岸線の代理）・通商依存度・海軍建設意志から**シーパワー潜在力**（0..1）を合成する純関数。陸軍国家（低海洋指向）は同じ国庫でも艦隊を出し切れない。
- 接続：`FactionState`/`CampaignState` を read-only で読み、`LogisticsRules.CohesionFactor`（版図一体化）と合成。`ShipyardRules.ProductionFactor`（建艦力）へ係数として波及可能な値を返す。新 `SeaPowerRules`（純ロジック・test-first）。
- 完了条件にできる単位：6要素の重み付き合成・極端値（純陸軍/純海洋）のテスト・基準値非破壊。

#### MHAN 制海権のグラデーション（海上交通路の優勢度を勢力間で争う）
- **制海権状態**：各回廊（または回廊網）に、勢力ごとの**制海優勢度**（連続値）を持たせる。在席艦隊戦力・要衝支配・補給接続から優勢度が推移し、`IsFtlBlocked` の二値を**連続化**。優勢側は通商が流れ、劣勢側は枯れる。
- 接続：`StrategyRules`（戦闘/接触の唯一窓口）に read-only で接続する新 `SeaControlRules`/`SeaControlState`（純ロジック）。`SupplyRules`（補給遮断）・`CommerceRaidingRules`（襲撃成立条件）へ優勢度を供給。`IsFtlBlocked` は後方互換で残す。
- 完了条件にできる単位：優勢度の収束（`GovernanceRules` 流儀の目標値収束）・拮抗/一方支配のテスト・盤面接続（観測層追従）。

### ★★ 高（ドクトリンと地理の戦略的遊びを足す）

#### MHAN 艦隊決戦 vs 現存艦隊ドクトリン（決戦を求める／温存し脅威で在り続ける）
- **ドクトリン enum**`{艦隊決戦,現存艦隊}`：決戦ドクトリンは接触で交戦を強制し制海権を一気に奪う（勝てば大）。現存艦隊ドクトリンは決戦を回避し、**存在するだけで敵の自由行動を抑止**（劣勢側が時間を稼ぐ）。
- 接続：`AutoBattleSim`/`StrategyRules` に**ドクトリン修正子**として上乗せ。`AdmiralSkillRules`（条件付き修正子）の戦略版。新 `NavalDoctrineRules`（純ロジック・test-first）。
- 完了条件にできる単位：決戦/温存の交戦発生率・現存艦隊の抑止効果（敵投射半径の縮小）・基準非破壊。

#### MHAN チョークポイント増幅（要衝の支配が通商/補給/制海を増幅）
- `Corridor.CorridorType.要衝`（型のみ存在・効果未配線）に**戦略的増幅**を与える：要衝を支配すると隣接交通路の制海優勢度・通商流量・補給到達が増幅され、敵が握ると自勢力のそれが阻害される。少数の要衝が盤面を支配する。
- 接続：`SeaControlRules`（制海権）×`ZoneOfControl`（#81）×`SupplyRules`。`Corridor.type` を read-only で読むだけ（新フィールド追加なし）。新 `ChokepointRules`（純ロジック・test-first）。
- 完了条件にできる単位：要衝支配の増幅係数・要衝喪失で交通路が枯れるテスト・盤面接続（観測層追従）。

#### MHAN 集中の原理（戦闘艦隊を分割しない＝分散は各個撃破）
- **集中度**：勢力の戦闘艦隊が地理的に集中しているほど実効戦力が上がり（ランチェスター二乗則）、分散すると各個撃破されるペナルティ。`OrderOfBattle`（編制ツリー）の梯団がバラけているほど不利。
- 接続：`OrderOfBattle`（#147）×`StrategicFleet` 位置を read-only で読み、`AutoBattleSim` の実効戦力へ係数を返す純関数。新 `ConcentrationRules`（純ロジック・test-first）。`CombatModifiers`（#106 係数公式）へ将来合流可。
- 完了条件にできる単位：集中/分散の実効戦力差・二乗則との整合テスト・基準非破壊。

### ★ 中（前進基地・lore）

#### MHAN 前進基地・補給拠点（作戦行動半径の投射）
- **前方拠点**：補給源から遠い前線で戦うには中継拠点（石炭ステーションの抽象）が要る。拠点を持つと艦隊の作戦行動半径が延び、無いと遠征が枯れる。
- 接続：`SupplyRules`（#94 補給到達）に**拠点ノード**を足す薄い拡張（別レジストリ新設しない＝`SupplyRules` の到達計算に拠点を始点として加える）。新 `ForwardBaseRules`（純ロジック・test-first）。
- 完了条件にできる単位：拠点ありなしの到達圏差・拠点喪失で遠征艦隊が孤立するテスト。

#### MHAN（lore）海上権力の世界観開示データ
- 「制海権こそ国力の源泉」「大陸国家は海洋国家に敗れる」「通商の保護が艦隊の目的」「要衝を制する者が海を制す」。
- 接続：**コード新設せず** `DisclosureLedger`（FND-4）への lore データ入力。CCX-6（世界観codex退避）方針に一貫。王道路線・フェザーン#160 と共鳴。

### ❌ 不採用（重複・既存で十分）

| 不採用 | 理由 |
|---|---|
| 通商破壊そのもの（護衛 vs 襲撃の解決） | **`CommerceRaidingRules`（#95）が既にカバー**。MHAN は「raiding は決定打にならない」という非対称を制海権側に与えるだけ（接続） |
| 補給線の遮断＝前線が枯れる | **`SupplyRules`（#94）がカバー**。前進基地で薄く拡張するのみ |
| 戦闘解決（ランチェスター二乗則） | **`AutoBattleSim`（TIME-4）がカバー**。集中/ドクトリンの修正子を上乗せするだけ |
| 版図の地理的分断ペナルティ | **`LogisticsRules`（#844）がカバー**。シーパワー国力導出で接続 |
| 経路探索・海図ノード/エッジ | **`GalaxyMap`/`GalaxyPathfinder` がカバー** |
| 海戦の戦術描画（陣形・砲戦） | **会戦レイヤー（`Squadron`/`FleetWeapon` 等）が既にカバー**。MHAN は戦略マクロ層のみ |
| 海軍研究/造船技術ツリー | **`ResearchRules`（#123-127）・`ShipyardRules`（#884）がカバー**。シーパワー係数で接続 |

### 推奨着手順
`MHAN-1`（シーパワー国力導出＝最も基底の欠落・他柱の土台）→ `MHAN-2`（制海権グラデーション＝盤面の核・`IsFtlBlocked` を連続化）→ `MHAN-3`（チョークポイント増幅＝要衝の型を活かす・MHAN-2 の上に乗る）→ `MHAN-4 → MHAN-5`（ドクトリン→集中＝戦略の遊び・二乗則と一致）→ `MHAN-6`（前進基地＝遠征の地理制約）→ `MHAN-7`（lore・最後にまとめて開示データ化）。

> いずれも既存の戦略マクロ（`Corridor`/`StrategyRules`/`CommerceRaidingRules`/`SupplyRules`/`AutoBattleSim`/`LogisticsRules`）を**後退させず接続**する additive 設計。三本柱＝**シーパワー国力導出（MHAN-1）／制海権グラデーション（MHAN-2-3）／決戦・集中ドクトリン（MHAN-4-5）**。フェザーン#160（通商国家）・王道路線（武力でなく地理と通商で国力が決まる）に最も効く。
