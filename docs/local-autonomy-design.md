# 地方自治 — 総督/知事/貴族の地方権力（EPIC #1306）

> 地方（方面/星系）を統べる**地方権力者**（総督・知事・貴族）を、政体で"中の人"が替わる**統一スロット**としてモデル化する。
> **中央集権↔封建**（地方がどこまで自分で決めるか）と**任命↔選挙**（地方権力者が誰に応答するか）の2軸で、中央と地方の関係を駆動する。
> 直接の動機＝**目安箱の提案システム再設計 EPIC #1296（MEYASU）の「地方（貴族・知事）箱」の供給元**。地方箱に入った陳情を**誰が・何の基準で**裁くかは、この地方自治レイヤーが決める。
> 状態：設計メモ（実装未着手）。数値・創作裁定は【要・作者判断】。

---

## 0. なぜこの設計か — 「貴族＝藩主＝自治を任された地方権力」

MEYASU の議論で「貴族箱」を詰めるうち、**貴族は身分の名でなく"地方権力者"の一形態**だと分かった：

- 貴族 ≒ 藩主＝一定の**自治**を任され、**自分の自治領域に関する決裁**をする。
- 民主政では同じスロットが**知事**になる。
- 知事には**選挙制**（米・日）と**任命制**（露）がある＝**誰に応答するか**が政体で変わる。

つまり「国王箱／政治家箱」が**中央権力**なのに対し、「地方箱」は**地方権力スロット**であり、その中の人と判断基準は政体で替わる。これを供給するのが本 EPIC。既存の封建（#168/#169）・総督（`GovernmentRegistry` の方面スコープ）・内政三層（`Province`）を**「地方統治の制度レイヤー」として束ね**、足りない2軸（任命/選挙の応答先・中央地方の権限/財政配分）を足す。

> **重複新設しない**：`FeudalRules`（徴募/反乱/門地）・`GovernanceRules`（`Province` 安定/統合）・総督台帳・旗幟#817 は**作り直さず接続**。JICHI は統合＋応答軸＋配分＋目安箱供給だけを足す。

## 1. 2つの軸（地方統治の手触り）

### 軸1：自治度（中央集権 ↔ 封建）＝地方がどこまで自分で決めるか
- 高自治（封建）：地方権力者が**中央承認なしで**広範に決裁できる（自領の税・徴募・小競り合い）。
- 低自治（中央集権）：地方は執行機関に近く、**重要案件は中央へ上げる**。
- → MEYASU と直結：高自治なら地方箱で決着、低自治なら地方箱は実質空洞で**国王箱へ集約**される。

### 軸2：応答先（任命 ↔ 選挙）＝地方権力者が誰の顔を見るか
| 中の人 | 選任 | 判断基準 | 応答先 | 統制 |
|---|---|---|---|---|
| **世襲貴族（藩主）** | 世襲 | 自領の利益 | 自分 | 中央は罷免しにくい＝**反乱リスク** |
| **選挙知事**（米・日型） | 地方選挙 | 地方民意・再選 | 地元有権者 | 中央は罷免できない・**民意で動く** |
| **任命知事**（露型） | 中央任命 | 中央への忠誠・保身 | 中央 | 中央が**罷免可**＝従順だが地方民意に冷淡 |

→ **「地方陳情が効く政体／効かない政体」が創発**：選挙制＝地方で人気なら通る／封建＝貴族の利益に合えば通る／任命制＝地方では止まらず中央へ。

## 2. データモデル案（Core・test-first・既存に接続・並行新設しない）

### 2-1. `RegionalGovernor`（Core・`[Serializable]`）
```
scopeKey     : 統治範囲（方面/星系・Office.scope と同語彙）
faction      : 勢力
holderId     : 地方権力者（ICharacter.id・null=空席）
selection    : GovernorSelection{世襲, 選挙, 任命}        ← 政体で既定が決まる
autonomy     : float 0..1（中央集権0 ↔ 封建1）            ← 軸1
```
> 地方権力者の台帳は**新設せず** `GovernmentRegistry` の方面/星系スコープ保持者を流用。`RegionalGovernor` はそのスコープに「選任方式・自治度」を足す薄いラッパ。

### 2-2. `DevolutionRules`（static・軸1＝権限委譲）
> 既存 `AutonomyRules`（#544 指揮ドクトリン）と名前衝突を避け `Devolution` を使う。
- `CanDecideLocally(governor, domain, scope)` … 自治度で「中央承認なしに決裁できる案件か」。
- `RequiresCentralApproval(...)` … 低自治では地方案件も中央へ。
- `EffectiveLocalAuthority(governor)` … 自治度×選任方式から実効の地方権限（実効値パターン）。

### 2-3. `GubernatorialRules`（static・軸2＝選任と応答）
- `DefaultSelection(政体)` … 封建=世襲／民主=選挙／中央集権=任命。
- `DecisionCriterion(governor)` … 自領利益／地方民意／中央忠誠（MEYASU 地方箱の判断基準を返す）。
- `Accountability(governor)` … 応答先（自分/地元/中央）。罷免可否・再選圧。
- 知事選は `LeadershipElectionRules`（#165 総裁選）を流用＝新選挙エンジンを作らない。

### 2-4. 中央↔地方の財政配分（軸の帰結・`FiscalRules` 接続）
- `RemittanceRate(governor)` … 地方産出のうち中央へ上納する率（中央集権=高い吸い上げ／封建=地方留保）。
- 地方留保は地方の安定（`GovernanceRules`）・徴募（`FeudalRules.LevyContribution`）へ。低い上納＝中央財政が痩せる＝集権化の動機。

### 2-5. 中央統制 vs 地方自立（離反・既存に接続）
- 高自治＋自利＋不満→**不服従/反乱**＝`FeudalRules.VassalRebellionRisk`・旗幟#817（中央への忠誠/調略）へ委譲（新ロジックを作らない）。
- 任命知事は罷免で抑えられるが、罷免の乱発は地方の支持を削る（`CaptivityRules` 流儀の支持ペナルティ思想）。

## 3. MEYASU（目安箱）への供給＝この EPIC の出口

地方箱に振られた陳情を、JICHI が次の3点で解決する：

1. **中の人**＝`RegionalGovernor.holderId`（世襲貴族/選挙知事/任命知事）。
2. **判断基準**＝`GubernatorialRules.DecisionCriterion`（自領利益/地方民意/中央忠誠）。
3. **そもそも地方で決まるか**＝`DevolutionRules.CanDecideLocally`（低自治なら国王箱へ集約）。

→ MEYASU は「地方箱」という入口だけを持ち、**中身は JICHI が供給**（責務分離）。

## 5. 完了条件（EPIC）
- 地方（方面/星系）に**統一スロットの地方権力者**が立ち、政体で世襲貴族/選挙知事/任命知事に替わる。
- **自治度（集権↔封建）**で地方の決裁権限の委譲範囲が決まり、低自治なら地方案件が**国王箱へ集約**される。
- **任命↔選挙**で応答先（中央/地元）が変わり、「地方陳情が効く政体／効かない政体」が創発する。
- 中央↔地方の**財政配分**・**離反/反乱**が既存（`FiscalRules`/`FeudalRules`/旗幟#817）へ接続して動く。
- MEYASU の地方箱へ**中の人・判断基準・地方決裁可否**を供給する（責務分離）。
- いずれも封建#168・総督・内政三層を**後退させず接続**する additive 設計。

## 6. 【要・作者判断】
- **自治度のレンジと既定**：政体ごとの初期自治度（封建=高／中央集権=低）。プレイ中に集権化/分権化できるか（明治の廃藩置県／連邦化）。
- **罷免の代償**：任命知事を切る際の地方支持ペナルティ。乱発のリスク。
- **上納率の具体値**：中央財政と地方安定のトレードオフのバランス。
- **知事選の粒度**：`LeadershipElectionRules` をどこまで流用するか（簡易 roll か候補つきか）。
