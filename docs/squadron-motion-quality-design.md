# 配下艦運動品質（Squadron Motion Quality）── #69/#80 の次段

> EPIC（GitHub）の設計メモ。**配下艦（`Squadron.memberShips` / `EscortShip`）は陣形を見せるための追従表示＝「見た目」の領域**（戦闘の最小単位は旗艦側 `FleetStrength`/`FleetWeapon`）。本EPICは戦闘ロジックではなく**運動の品質（見た目のリアルさ）**を一段上げる。設計原則どおり「壊れて見えない」最低ラインを越えたら深追いしない。

## 現状（実装済み＝正）
`#69`（横滑り・振り回し・すり抜け・ワープの解消）はクローズ済みで、`Squadron.cs` に以下が入っている：
- **速度上限つき追従**：`catchUpRatio`（旗艦 `FleetMovement.maxSpeed × catchUpRatio`）で SmoothDamp の maxSpeed と 1フレーム移動量を二重クランプ＝ワープ防止。
- **進行方向への回頭**：`escortRotationSpeed`/`headingMoveThreshold`/`alignToFlagshipSpeed`＝移動中は進行方向、定位置では旗艦の向き（射界）へ戻す。
- **簡易セパレーション**：`minSeparation`/`separationStrength`/`separationUpdateInterval`＝部隊内 O(n²) の押し離し（間引き＋初回位相分散）。

`#80`（艦種差別化）もクローズ済みで、戦艦/巡航艦/駆逐艦の耐久・火力・速度・スケール・色味が入っている。

## 未解決（本EPICが拾う）
1. **スロット割当が添字順**（#69 の積み残し）：`UpdateShipPositions` は `cachedSlots[i]` を**添字順**で参照する。`EnsureSlots` は隻数・陣形が変わると**スロットを総再計算**するため、`RemoveMember`（戦死で添字シフト）や陣形変更で**席が再採番**され、配下艦が遠いスロットへ割り当て直されて**席替えの交差・横断**が起きる。セパレーションは事後の対症療法で、交差そのものは防げていない。
2. **スロット配置が艦種非依存**（#80 の明示的先送り）：`AssignShipClasses` のコメント「スロット配置は艦種非依存（最小実装＝混成のみ。前衛/遊撃の配置最適化は将来）」。戦艦は添字前半・駆逐艦は後半に置くだけで、実際のスロット位置（前面/外周/側面）と無関係＝艦種差が陣形配置に出ない。
3. **慣性が無い**：発進→最大速が瞬時、停止も急。SmoothDamp の追従だけで、加減速のランプや回頭のバンク（傾き）が無い。
4. **航行/戦闘で密度が変わらない**：交戦中は `combatSmoothTime` でゆっくり追従するだけ。`spacing` は不変＝移動隊形と戦闘隊形の密度差が無い。
5. **セパレーションが N²**：`ApplySeparation` は部隊内 O(n²)（`escortCount`=80→約3160ペア）×全部隊。間引き済みだが終盤の多部隊会戦で重い（スケーラビリティ規律の対象）。

## 設計方針（全子issue共通）
- **`FleetMovement.cs` は変更しない**。改善は `Squadron.cs` 内で完結（必要なら `EscortShip.cs`）。
- **実効値パターン**：`spacing`/速度等の基準フィールドは上書きせず、ローカル実効値で計算。
- **純ロジックは Core へ切り出して test-first**：スロット割当（位置集合→スロット集合の最近傍マッチング）・艦種重みは `Ginei.Core` の純関数にし EditMode/TestHarness で担保。`Squadron`（MonoBehaviour）はそれを呼ぶだけ。
- **スケーラビリティ規律**：派生（スロット）は変化時のみ再計算してキャッシュ（既存 `EnsureSlots` の規律を維持）。N² の相手数を増やさない・毎フレーム全再計算しない・見ていない部隊は粗く。
- **後方互換**：調整値は public＋`[Header]`/`[Tooltip]`、既定値で従来の見た目を維持。`Time.deltaTime` 基準で timeScale 追従。

### EMOV-1 スロット割当の安定化（#69残件）
- **核**：各配下艦を「現在位置から最も近い空きスロットへ」貪欲割当する**安定マッピング**（`memberShips[i] → slotIndex`）を導入。スロット集合が変わった時（`EnsureSlots` のキャッシュ無効化時）だけ再割当し、対応表を保持する。
- **Core 純ロジック**：`EscortSlotAssignmentRules.Assign(現在位置[], スロット[]) → 割当インデックス[]`（貪欲最近傍・決定論）を切り出して test-first。
- **完了条件**：陣形変更・戦死時に席替えの交差が目視で消える。セパレーションに頼らず交差を**起こさせない**。

### EMOV-2 艦種を考慮したスロット配置（#80残件）
- 艦種ごとのスロット嗜好（戦艦=前面/外周の楯、駆逐艦=側面/後方の遊撃、巡航艦=中核）を重みで定義し、EMOV-1 の割当に艦種重みを合成。
- **完了条件**：硬い前衛・速い遊撃が陣形配置でも見える。既存シナリオ後方互換。

### EMOV-3 加減速ランプと回頭バンク
- 追従速度の立ち上がり/減衰にランプ（旗艦 `FleetMovement` の accel/decel 思想を配下艦に適用）。回頭時にわずかなバンク演出（傾き・任意）。#69 のワープ防止クランプと両立。

### EMOV-4 航行⇄戦闘隊形の密度切替
- 移動中は隊形をやや広げ（航行＝隊列を伸ばす）、交戦/定位置では締める（戦闘＝密集）。`spacing` を実効値で動的化（基準非破壊）。再計算はキャッシュ規律順守。

### EMOV-5 セパレーションの負荷対策
- 空間ハッシュ/グリッドで近傍のみ比較、または画面外/遠方/非交戦部隊のセパレーションを粗く（シミュLOD）。挙動は据え置きで負荷だけ削る。
