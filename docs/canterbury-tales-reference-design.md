# カンタベリー物語 参考設計（EPIC #TALE）

> 参照元：『カンタベリー物語』（ジェフリー・チョーサー・14世紀）。聖地カンタベリーへ向かう**29人の巡礼者**が旅籠に乗り合わせ、退屈しのぎに順番に話を語るオムニバス＝**群像劇（ensemble drama）の原点**。
> 騎士・修道士・粉ひき・「バースの女房」——**中世のあらゆる階層**が揃い、語られる話そのものが**各人のキャラクターと階層間のピリついた関係**を浮き彫りにする。
> 本ドキュメントは、当プロジェクト（Ginei＝銀英伝風の群像戦略劇）にとって**役に立つ視点**だけを抽出し、EPIC `#TALE` として issue 化する提案。
> 著作権注意：固有名・文章・キャラクターは流用せず、**群像劇の構造パターンのみ**を参考にする。

---

## 0. なぜ「カンタベリー物語」が本システムに役立つか

当プロジェクトは**ネームド人物層を大量に保有**している（[CLAUDE.md] 参照）：

| 既存（人物・群像） | カバー範囲 |
|---|---|
| `Person`/`ICharacter`・`AdmiralData`（異名/構造化命名#523） | 能力・役割・呼称を持つネームド |
| 列伝/殿堂（#784/#785） | 人物の経歴・記録 |
| 内部勢力#113・閥（学閥/文官閥・`CareerPipelineRules`） | 派閥・クリーク |
| 階級#110・身分（軍人/文民 `PersonRules`・聖職 `ReligionRules`） | 社会階層 |
| 旗幟#817・カリスマの日常化#812 | 忠誠・寝返り・組織存続 |
| `EventEngine`#116・`DisclosureLedger`（FND-4） | イベント・開示 |

**しかし、これらは「能力値・役職・経歴」という静的なデータ**であり、群像劇が固有に描く以下が**欠けている**：

| カンタベリー物語が固有に持つ視点 | 当プロジェクトでの欠落 |
|---|---|
| **巡礼の旅籠＝異階層の人物が一時的に集う社交場** | 戦闘・戦略・役職はあるが、人物が**機能の外で交流する非戦闘の場**が無い |
| **語る話が語り手を浮き彫りにする（人物挿話）** | 列伝は経歴の記録。**人物の性格/階級/思想から挿話が生成され人物像を立てる**回路が無い |
| **粉屋 vs 騎士＝話による代理闘争（階層間の摩擦）** | 派閥#113はあるが、**階級摩擦が社交の応酬として表面化**しない |
| **三身分の風刺＝理想と堕落の人物ギャラリー** | 能力値はあるが、**社会類型を理想型/腐敗型で描くフレーバー**が無い |
| **司会者（ホスト）＝場を招集し話を判定** | 場を束ね**評判を授ける主催者**の役が無い |
| **「バースの女房」型の自己呈示＝声/語り口** | 人物に**語り口（voice）**属性が無い |

**結論**：カンタベリー物語は、当プロジェクトの**名簿（stat-block の羅列）を“生きた群像劇”に変える**。具体的には**①社交場 ②人物挿話 ③階級摩擦 ④身分の風刺**の4軸を足し、それらが**既存の政治メカニクス（内部勢力#113・旗幟#817・忠誠・支持）へフィードバック**する——「人物が機能の外で人間として振る舞い、その振る舞いが政治に効く」。**銀英伝＝群像劇である本作の核心**を強化する。

## 1. 役に立つ視点（要約）

カンタベリー物語の群像劇構造を、**本システムに効く形**で1行ずつ：

1. **群像劇は「機能の外」で生まれる**。戦場でも役職でもない**社交の場**（祝勝会・評定・長征・巡礼）で人物が人間として現れる。→ 名簿を生かす非戦闘 venue。
2. **語る話が語り手を暴く**。挿話のジャンル（騎士のロマンス／粉屋の笑話／免罪符売りの偽善）が**階級と人格を雄弁に語る**。→ 列伝#784 を**生成エンジン**へ。
3. **階層の摩擦は話で戦われる**。粉屋が騎士の高雅を下品な話で茶化し、荘官が粉屋に報復する。剣でなく**社交の応酬が代理闘争**になる。→ 内部勢力#113・旗幟#817 に社交の歯を入れる。
4. **三身分の風刺＝理想と堕落**。気高い騎士・敬虔な教区司祭（理想）↔ 世俗の修道士・好色な粉屋・偽善の免罪符売り（堕落）。→ 腐敗 vs 有徳（既存の `Regime.腐敗`/`DynastyRules`）を**人物フレーバー**で可視化。
5. **多角的視点が世界への没入を生む**。同じ世界を29の目線で味わう＝**オムニバスが世界に厚みを与える**。→ 殿堂#785・`DisclosureLedger` への lore 蓄積。

## 2. 取り入れるべきメカニクス（優先度つき・既存への接続）

> 大原則：**列伝#784/殿堂#785・内部勢力#113・EventEngine#116・DisclosureLedger を作り直さない**。TALE はそれらに**社交の軸を足し、接続する**だけ（additive）。タイクン化回避＝会話の逐次マイクロ管理はしない。**高位の状態（階級/思想/関係/腐敗）から挿話と摩擦が創発**する。

### ★★★ 最優先（群像劇の核・真の欠落）

#### TALE 社交場（フレーム）＝非戦闘の人物交流 venue
- 異なる勢力/階級の人物が**一時的に集う場**（祝勝会・軍議・長征の船中・捕虜交換・宮廷）。場の中で**関係値が動く**（親愛/反目）。巡礼＝「目的地へ向かう道中、たまたま乗り合わせた者たちの場」。
- 接続：新 `GatheringRules`（純ロジック・test-first）。`Person`/列伝#784・内部勢力#113 の関係を更新。UI は将来。

#### TALE 人物挿話の生成（語る話が語り手を暴く）
- 人物の**階級/役割/思想/能力/archetype**から、決定論的に**短い挿話**を生成。ジャンル（ロマンス/笑話/教訓/風刺/英雄譚）が**人物像を立てる**。生成挿話は列伝#784 に蓄積。
- 接続：列伝#784 を**生成エンジン**化＋`EventEngine`#116（挿話イベント）。flavor だがフックは決定論。

### ★★ 高（群像が政治へ効く）

#### TALE 階級摩擦の社交的表出（粉屋 vs 騎士＝代理闘争）
- 階級#110/閥#113 の**対立が社交の応酬**として表面化（茶化し・当てこすり・報復の挿話）。結果は**関係値・派閥結束#113・支持・旗幟#817**に波及＝寝返り/離反の伏線。
- 接続：階級#110・内部勢力#113・旗幟#817・`RedistributionRules.ClassTension`。**社交が政治の火種**に。

#### TALE 三身分の風刺＝人物ギャラリー（理想 vs 堕落）
- 社会類型（**戦う者/祈る者/働く者**＝軍人/聖職/平民）を**理想型↔腐敗型**のフレーバーで描く「総序（General Prologue）」。腐敗した修道士・偽善の免罪符売り vs 敬虔な司祭。
- 接続：`PersonRules`（軍人/文民）・`ReligionRules`（聖職）・`Regime.腐敗`/`DynastyRules`（腐敗 vs 有徳）。人物カードのフレーバー軸。

### ★ 中（声・主催・lore）

#### TALE 司会者・主催（ホスト/判定者）
- 社交場を**招集し挿話を判定**、勝者に**評判**を授ける（旅籠の主人＝コンテストの審判）。
- 接続：元首/主人公#735・評判（ALM-5 王道/覇道と連動可）。

#### TALE 人物の声・自己呈示（バースの女房型）
- 人物に**語り口（voice）**属性（高雅/粗野/雄弁/皮肉…）。自己呈示が**評判/カリスマ#812**に効く。
- 接続：`AdmiralData`（異名/呼称#523 の隣に voice）・カリスマ#812・列伝。

#### TALE（lore）挿話→列伝→世界観の厚み
- 生成挿話を**殿堂#785・`DisclosureLedger`** に蓄積＝オムニバスが世界に多角的な厚みを与える。
- 接続：**コード新設せず** 殿堂#785・`DisclosureLedger`（FND-4）への蓄積。CCX-6（世界観codex退避）に一貫。

### ❌ 不採用（重複・タイクン化）

| 不採用 | 理由 |
|---|---|
| 会話の逐次選択肢を手動で回す対話アドベンチャー | **タイクン化/別ジャンル化**。挿話は高位状態から**創発**させる |
| 列伝/殿堂そのものの再実装 | **#784/#785 が既にカバー**。TALE は生成と蓄積で接続 |
| イベントエンジンの再実装 | **#116 がカバー**。挿話は `EventEngine` に乗せる |

`TALE-1 → TALE-2`（社交場＋挿話生成＝群像劇エンジンの核）→ `TALE-3`（階級摩擦→政治フィードバック＝最も「効く」）→ `TALE-4`（身分の風刺フレーバー）→ 残り（TALE-5/6/7）。
