# 創発キャンペーン 設計（EPIC #117）— モード併存・政体創発・内政統合

> イデオロギーを**プレイヤーが選ばず、地理・歴史・地政学から創発**させる新キャンペーンを、既存の戦闘/戦略/内政の上に載せる統合設計。
> 関連：EPIC #117（創発）／G-1 #118・G-2 #119・I-1 #120・I-2 #121／内政 EPIC #109（P-1〜P-6 #110-115）／イベントエンジン #116／Phase C（#33・C-1 #34…）。
> 基盤設計：`docs/phase-c-core-design.md` / `phase-c-late-design.md` / `wartime-logistics-design.md` / `planetary-governance` 系（#109）。

## 0. 方針：**モード併存（テスト容易性のため）**
- 既存の **シナリオ会戦モード（Title→Battle→Result）はそのまま残す**（個々の戦術・要塞・勝利条件などを単体テストする土台）。
- 新規 **キャンペーンモード**を別系統として**追加**（Galaxy シーン／創発系統）。両者は**戦術エンジンを共有**（C-3 のランタイム生成パス）。
- 切替：`GameSettings.gameMode`（`enum GameMode { シナリオ, キャンペーン }`）。タイトルでモード選択。既定はシナリオ（既存挙動を壊さない）。

```
Title ──┬─ [シナリオ] → 既存：陣営/会戦選択 → Battle → Result   （現状維持）
        └─ [キャンペーン] → Galaxy（1惑星/フォグ） → 探索/回廊戦闘/内政/政体創発 …
                                   └ 戦闘はC-3のランタイム生成で Battle 相当を起動（共有）
```

## 1. アーキテクチャの肝：`FactionData`＝**政体アーキタイプ（テンプレート）**
> 「可変イデオロギー」と「既存プリセット勢力」を**両立**させる鍵。既存アセットを壊さない。
- `FactionData`（色/思想/`ranks`/`nonHostileFactions`/階層構成）は**アーキタイプ（典型）ライブラリ**として扱う：原始首長制／王党派／民主派／共産／軍閥…。
- キャンペーンの各**政体（Polity＝実行時の政治主体）**は、現在の創発状態 `IdeologyState`（I-1）を持ち、それに最も合う `FactionData` アーキタイプを**参照**する（色・階級構成・階級ラダーをそこから引く）。
- 政体が変質（I-2）したら、**参照するアーキタイプを差し替える**だけ（`FactionData` アセット自体は不変＝非破壊）。
- シナリオモードは従来どおり `FactionData` を直接の勢力として使う。→ **同じアセットが両モードで使える。**

> これにより `FactionData.ideology`（固定文字列）は「アーキタイプの説明」のまま、キャンペーンの動的政体は別オブジェクト `IdeologyState` が持つ（実効値パターンの思想＝基準は非破壊）。

## 2. キャンペーンの1周（ゲームループ）
1. **1惑星・原始首長制**でスタート（G-2）。銀河は手続き生成済み（G-1）だが大半は**フォグ**。
2. **探索**：艦隊が回廊を進むと隣接星系/回廊が開示（G-2）。**チョークポイント**（G-1で保証）が戦略の要に。
3. **接触・拡張**：他政体と接触。回廊で衝突→**回廊戦闘（Phase C C-3）**＝戦術エンジン共有。
4. **内政（#109）**：支配星系を統治。**階級（#110）は現在の政体の表現**。安定度/政策/補給で産出・徴募が変わる。
5. **政体の創発・変質（I-1/I-2）**：地理（孤立/交易/チョーク支配）＋歴史（戦争/飢饉/革命イベント #116）＋地政学（隣接/依存）で政体が変わる。プレイヤーは**誘導**するが選ばない。
6. 2〜5 を回しながら銀河の覇権を争う。

## 3. 銀河生成（G-1 #118）
- `GalaxyGenerator` が `GalaxyMapData`（C-1）を手続き生成：ノード（星系）＋エッジ（回廊）。回廊以外は航行不能。
- **チョークポイント保証**：グラフの**橋（bridge）/関節点（articulation point）**を最低1つ含めるよう生成→検証→無ければ再生成/補正。イゼルローン型の攻防地形を担保。
- 回廊型（チョークポイント型＝要塞 #76-78 候補／フェザーン型＝通商回廊）・星系類型（工業/農業/鉱業/居住＝L-1）・初期資源を配置。**シード固定で再現可能**。

## 4. 探索・フォグ（G-2 #119）
- 可視性：未発見／発見済み（位置のみ）／可視（情報あり）／視界（リアルタイム）。
- 初期は自惑星＋隣接の一部のみ可視。探索（時間制ワープ＝C-1）で段階開示。未踏は隠す（未発見宙域＝世界観バイブル §4）。
- 可視性は `SaveData`（#108 規約）に保持。

## 5. 政体の創発モデル（I-1 #120）
- **`IdeologyState`**：固定文字列でなく、**要因から導かれる派生値＋現在の政体ラベル**（原始首長制／王党派＝封建王政／民主派＝共和／共産／軍閥）。
- 創発の要因（→ public 重み）：
  - **地理**：孤立/中心、資源豊富/貧、交易路上/僻地、**防衛容易（チョークポイント支配）**。
  - **歴史**：戦争/占領/解放/飢饉/革命（イベント #116）。
  - **地政学**：強敵隣接、交易依存、孤立、覇権圏。
- **要因→政体マッピング（案・要調整 §要・作者判断）**：

| 状況 | 創発しやすい政体 |
|---|---|
| チョークポイント覇権＋拡張・中央集権 | 王党派（帝政） |
| 交易ハブ・開放経済・分権 | 民主派（共和） |
| 抑圧・困窮・階級対立 | 共産主義 |
| 辺境・無秩序・自衛・中央崩壊 | 軍閥 |
| 出発点（小規模・未分化） | 原始首長制 |

- 政体に最も合う `FactionData` アーキタイプを参照（§1）。**階級構成（#110）は政体に従う**（首長制＝単純→王党派化で貴族/奴隷発生→民主派化で平民へ均す）。

## 6. 政体の変化（I-2 #121）
- I-1 の要因が閾値超過、または特定イベント（#116）で**遷移**：困窮×階級対立→共産革命／チョーク覇権→帝政化／交易繁栄→共和化／中央崩壊→軍閥割拠。
- **急変（革命）と漸進（変質）**。変化時に**階級（#109/#110）・政策（#112）・派閥支持（#113）が再編**。
- プレイヤーは政策・戦争・交易で要因を動かせる（誘導）が、政体は直接選べない。`SaveData` に政体履歴。

## 7. 内政との一体化（#109 / #110-115）
- **階級制度＝政体の表現**（#110）。安定度（#111）・統治政策（#112）・派閥（#113）・占領/解放（#114）・イベント（#115）はキャンペーンの統治レイヤーそのもの。
- 解放のジレンマ（民主派が王党派世界を解放→奴隷解放・貴族廃止→民心↑だが将校/生産特権喪失）が、政体創発とも連動（解放＝相手政体を自政体アーキタイプへ転換）。

## 8. 既存コードとの接続・併存
| 対象 | 併存での扱い |
|---|---|
| `GameSettings` | `gameMode`（シナリオ/キャンペーン）＋キャンペーン状態（銀河/政体/可視性）。 |
| `TitleManager` | モード選択を追加。シナリオは現行フロー、キャンペーンは Galaxy へ。 |
| `FactionData` | **アーキタイプ・ライブラリ**（不変）。両モードで共用。首長制アーキタイプを追加（FactionDataCreator）。 |
| `IdeologyState`/`Polity`（新規） | キャンペーンの動的政体。アーキタイプを参照、変質で差し替え。 |
| `BattleSetup`（ランタイム生成＝C-3） | キャンペーンの回廊戦闘を `StrategicFleet` 状態から起動（シナリオと戦術エンジン共有）。 |
| `FactionRelations` | 敵対判定の唯一の窓口（両モード共用）。政体間の関係も `nonHostileFactions`/政体ロジックで。 |
| `SaveData`（#108） | キャンペーンの銀河・政体・可視性・履歴を保持（schemaVersion で後方互換）。 |
| イベントエンジン #116 | 革命/簒奪/独立などの政体イベントの基盤。 |

## 9. 段階ビルド手順（各段でコンパイル＆**シナリオモードの無事**を確認）
1. **モード分岐の器**：`GameSettings.gameMode` ＋ タイトルのモード選択（キャンペーンは空のGalaxyへ）。シナリオは不変。
2. **G-1 #118**：銀河生成（チョークポイント保証）→ Galaxy に静的表示。
3. **G-2 #119**：1惑星スタート＋フォグ＋探索開示。
4. **Phase C コア（C-1〜C-3）**：戦略移動＋回廊戦闘（既存設計）をキャンペーンに接続。
5. **内政最小（#110 P-1＋#111 P-2）**：階級・安定度。
6. **I-1 #120**：政体の創発（要因→政体、アーキタイプ参照、原始首長制スタート）。
7. **#116 イベントエンジン → I-2 #121**：政体変質（革命/変質）。
8. 以降：政策#112／占領解放#114／派閥#113／兵站L／要塞 等を順次。

## 10. リスク・未決事項
- **二モードの分岐が散らからないこと**：戦術エンジン（生成本体）は共通化し、入口だけ分ける（`BattleSetup` の二系統化の方針＝core 設計と同じ）。
- **政体アーキタイプの粒度**：首長制→各政体→さらに細分（穏健/強硬共産 #17 等）をアーキタイプで持つか、状態の付加軸で持つか。
- **`FleetRegistry` 単一性**（Phase C の既知課題）：キャンペーンの複数戦線は「ライブ1＋抽象N」で回避（`phase-c-late-design.md`）。
- **創発の調整**：要因→政体の重み・閾値はバランス調整が要る（地理が宿命になりすぎ/しなさすぎ）。
- **【要・作者判断】**：要因→政体マッピングの確定、政体ツリー、原始首長制の中身、キャンペーンの勝利条件（覇権/存続）、最終的にシナリオモードを残すか統合するか。
