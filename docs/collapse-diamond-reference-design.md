# 文明崩壊 参考設計（EPIC #CLPS）

> 参照元：ジャレド・ダイアモンド『文明崩壊（Collapse: How Societies Choose to Fail or Succeed）』。
> イースター島・マヤ・ノルウェー領グリーンランド・アナサジ等の**環境収奪による社会崩壊**と、徳川日本・ティコピア等の**持続に成功した社会**を比較し、「**社会はなぜ自ら破滅を選ぶのか**」を解剖したノンフィクション。
> 本ドキュメントは、当プロジェクト（Ginei＝銀英伝風の星間国家戦略＋巨大な社会・政治シミュ純ロジック層）にとって**役に立つ視点**だけを抽出し、EPIC `#CLPS` として issue 化する提案。
> 著作権注意：固有名・文章・固有設定（書中の具体地名/人名/逸話）は流用せず、**崩壊メカニクス／環境-社会フィードバックの構造パターンのみ**を参考にする。

## 0. なぜ「文明崩壊」が本システムに役立つか

当プロジェクトは社会・政治の**マクロ純ロジックを大量に保有**している（[CLAUDE.md]「テスト基盤」節 参照）。崩壊の「人間側」の因子はほぼ揃っている：

| 既存（カバー範囲） | 担当モジュール |
|---|---|
| 内政・安定度・占領統合・反乱圧力 | `GovernanceRules`/`Province`（#109） |
| 天命喪失・腐敗・易姓革命（王朝サイクル） | `DynastyRules`/`Regime`（#867） |
| 国家状態の合成・崩壊判定（`IsCollapsing`） | `FactionStateRules`/`FactionState` |
| 版図一体化度（散在は国力を出し切れない） | `LogisticsRules`（#844） |
| 希望と末人（フロストパンク） | `HopeRules`/`Community`（#852） |
| 合意・統治不能（権力は借り物） | `ConsentRules`/`Polity`（#836） |
| 財政破綻・債務スパイラル | `FiscalRules`/`FiscalState`（#161） |
| 階級対立・再分配・格差→反乱 | `RedistributionRules`/`CapitalRules`（#163/#917） |
| 資源生産・補給・通商破壊（兵站） | `ResourceProductionRules`/`SupplyRules`（#92） |
| 人口動態（ボーナス/オーナス） | `DemographicsRules`/`Population`（#153） |
| 気候/災害・危機イベント | `EventEngine`/`GameEventDef`（#116） |
| 研究・技術（文化の対応力） | `ResearchRules`（#123） |
| 開示エンジン（lore データ入力） | `DisclosureLedger`（FND-4 #495） |

**しかし、これらは全て「人間社会の内部状態」のマクロ均衡**であり、文明崩壊が固有に描く以下が**欠けている**：

| 文明崩壊が固有に持つ視点 | 当プロジェクトでの欠落 |
|---|---|
| **環境ストック＝再生可能資源の枯渇**（森林/土壌/漁場の有限性と再生速度） | `ResourceProductionRules` は安定度比例の**フローのみ**。**有限ストック・再生率・収奪が再生を超えると不可逆減少**する環境資本が無い |
| **環境収容力（carrying capacity）と過剰人口** | `DemographicsRules` は人口三コホートの加齢。**環境が養える上限・上限超過＝マルサス的崩壊**の連結が無い |
| **生態系崩壊→社会崩壊のティッピングポイント** | `GovernanceRules`/`FactionStateRules` は人間内因のみ。**環境劣化が安定度・人口・希望を押し下げる外因経路**が無い |
| **エリートの自己隔離（短期利益で長期破滅を選ぶ意思決定失敗）** | 階級対立`RedistributionRules`はあるが、**エリートが被害から隔離され合理的に破滅を選ぶ**（モアイ競争/共有地の悲劇）回路が無い |
| **持続可能管理＝意図的な自己制限（成功する社会）** | 収奪/抑圧のレバーはあるが、**自制（伐採制限/減産による持続）という"勝つための我慢"**の選択肢が無い |
| **5つの崩壊因子の複合**（環境破壊/気候変動/敵対者/交易相手の喪失/社会の対応失敗） | 各因子は個別に存在するが、**崩壊を「複合要因の同時成立」として診断する統合フレーム**が無い |

**結論**：文明崩壊は当プロジェクトの「人間内因の崩壊」モデルに、**①有限環境ストックと収容力 ②環境→社会のティッピング経路 ③エリート隔離による意思決定失敗 ④自制という勝ち筋**という、**外部環境という新しい崩壊の軸**を与える。既存の `FactionStateRules.IsCollapsing`（人間内因）に**環境因子を合流させる**ことで、崩壊が一層多層的・創発的になる。タイクン化を避け、**高位の政策（伐採率/移民/自制）→環境エンジン→創発的崩壊 or 持続**の形に収める。

## 1. 役に立つ視点（要約）

文明崩壊の論を、**本システムに効く形**で1行ずつ：

1. **環境は有限ストックで、収奪が再生を超えると不可逆に崩れる**。森林・土壌・漁場は再生可能だが速度に上限がある。→ `ResourceProductionRules`（フロー）の下に**環境資本ストック**を敷く＝兵站の枯渇#94を「滅びの時計」から「自業自得の時計」へ深める。
2. **環境が養える人口には上限（収容力）があり、超えるとマルサス的に崩れる**。→ `DemographicsRules`#153 と `GovernanceRules`#109 を環境ストックで連結。人口オーナスの「環境版」。
3. **生態系崩壊は社会崩壊のティッピングポイントになる**。閾値を割ると安定度・希望・人口が連鎖的に落ちる。→ `FactionStateRules.IsCollapsing` に**外因経路**を足す（`DynastyRules` の腐敗と並ぶ第二の崩壊エンジン）。
4. **エリートは被害から自己隔離し、短期利益のため長期破滅を選ぶ**（共有地の悲劇/地位競争）。→ `RedistributionRules`#163・`CapitalRules`#917・`PowerRules`#164 に**「破滅へのインセンティブ非対称」**を足す＝なぜ合理的個人が崩壊を選ぶかの回路。
5. **崩壊は「複合要因の同時成立」で起きる**（環境/気候/敵対/交易喪失/対応失敗の5因子）。→ 既存の各因子を読む**統合診断フレーム**＝崩壊リスクを可視化（観測層）。トルストイ的「単因論の否定」と共鳴。
6. **自制（伐採制限・減産・移民）こそが持続の勝ち筋**＝徳川日本/ティコピア型。我慢が報われる戦略を入れる。→ 高位の**環境政策レバー**＝収奪0↔保全1の `inclusiveness`（#817）に似た一軸を環境に。
7. **交易相手の喪失が孤立社会を道連れにする**（グリーンランド型）。→ `LogisticsRules`#844（版図一体化）×`SupplyRules`#94 に**「孤立×環境劣化の二重苦」**を接続（コード新設せず）。

## 2. 取り入れるべきメカニクス（優先度つき・既存への接続）

> 大原則：**`GovernanceRules`/`DemographicsRules`/`FactionStateRules`/`ResourceProductionRules`/`DynastyRules` を作り直さない**。CLPS はそれらの下に**環境ストック層を1枚敷き、接続する**だけ（additive）。タイクン化回避＝建設/伐採のマイクロ操作は作らず、**高位政策→環境エンジン→創発帰結**。

### ★★★ 最優先（真の欠落・文明崩壊の signature）

#### CLPS 環境資本ストックと再生（森林/土壌/漁場の有限性）
- **環境ストックモデル**：各惑星`Province`に**再生可能資源ストック**（健全度0..1）と**再生率**。**抽出（収奪）が再生を超えるとストックが不可逆に減少**（劣化が進むほど再生率も落ちる＝劣化のスパイラル）。
- 接続：新 `EnvironmentRules`/`EnvironmentStock`（純ロジック・test-first）。`ResourceProductionRules.Produce`（安定度比例フロー）の**上流にストック制約**を掛ける＝産出はストック健全度に比例（枯れた星系は産めない）。既存のフロー兵站#92を後退させず外側から制約。

#### CLPS 環境収容力と過剰人口（マルサス的崩壊）
- **収容力**：環境ストックが養える人口上限。`Population.Total` が収容力を超えると**過剰人口ペナルティ**（食料不足→安定度低下→死亡率上昇）。収容力割れまで人口が縮小して均衡（オーバーシュート&コラプス）。
- 接続：`DemographicsRules`#153（人口三コホート）に**収容力比**を渡す＝`OutputFactor` の環境版。`GovernanceRules`#109 の安定度目標へ**過剰人口modifier**を1項追加（既存の収束式に乗せるだけ）。

#### CLPS 環境ティッピングと崩壊の合流（外因の崩壊エンジン）
- **ティッピングポイント**：環境ストックが閾値を割ると、安定度・希望・人口が**連鎖的・非線形に**落ちる（不可逆な体制崩壊）。
- 接続：`FactionStateRules.IsCollapsing`（現状＝天命喪失/統治不能/組織崩壊/末人＝人間内因）に**「環境崩壊」を第5の崩壊条件として合流**。`DynastyRules`#867（腐敗の崩壊）と並ぶ**第二の崩壊エンジン**＝崩壊が単因でなく複合になる。

### ★★ 高（意思決定の歪み・勝ち筋）

#### CLPS エリートの自己隔離と破滅的意思決定（共有地の悲劇）
- **隔離係数**：支配層（`PowerRules`#164/富裕層`CapitalRules`#917）が環境被害から隔離されているほど、**収奪を続ける合理性**が高まる（短期利益はエリートへ・長期被害は全体へ）。隔離が高い社会は自制レバーを引かず崩壊へ向かう。
- 接続：`RedistributionRules`#163（階級別負担）×`CapitalRules`#917（r>g集中）に**「環境費用の非対称な負担」**を足す＝なぜ合理的エリートが破滅を選ぶか。`EventEngine`#116 で「警告を無視するエリート」イベントの火種。

#### CLPS 自制という勝ち筋（持続可能管理・徳川/ティコピア型）
- **環境政策レバー**：高位の一軸**収奪0↔保全1**（伐採制限/減産/移民）。保全を選ぶと短期産出は落ちるが**ストックが再生し長期に持続**＝我慢が報われる戦略。収奪は即効だが崩壊へ（`FactionState.inclusiveness`#817 の環境版＝同型パターン）。
- 接続：`FactionState` に環境政策フィールドを1つ足し、`EnvironmentRules.Tick` が抽出率を決める。**新レバーは1本だけ**（マイクロ操作を増やさない）。`GovernanceRules`/`DynastyRules` の有徳と相互作用（徳が高い体制ほど自制できる）。

### ★ 中（複合診断・孤立・lore）

#### CLPS 崩壊5因子の統合診断（複合要因の可視化）
- ダイアモンドの5枠（環境破壊/気候変動/敵対者/交易相手の喪失/社会の対応失敗）を**既存モジュールから読む統合スコア**＝崩壊リスクを多軸で診断（環境=`EnvironmentRules`/気候=`EventEngine`/敵対=`DiplomacyRules`#189/交易=`LogisticsRules`#844/対応失敗=自制レバー未行使）。
- 接続：**コード新設は薄く**＝`CollapseDiagnosisRules`（既存値を集計する純関数のみ・新state持たず）。観測層`CoreStateInspector`で5因子バーを表示＝「なぜ滅びるか」が見える。

#### CLPS 孤立×環境劣化の二重苦（交易相手の喪失）
- 版図が分断・孤立した勢力（`LogisticsRules`#844 の一体化度が低い）は、環境劣化時に**交易による救済が効かず道連れに**なる（グリーンランド型）。
- 接続：**コード新設せず** `LogisticsRules.CohesionFactor`×`SupplyRules`#94 と `EnvironmentRules` を `CampaignRules.Tick` で掛け合わせるだけ＝孤立した枯れた星系は最も脆い。

#### CLPS（lore）世界観の開示データ
- 「社会は自ら崩壊を選ぶ」「短期合理が長期破滅を生む」「自制した者だけが続く」「環境は文明の隠れた基盤」。
- 接続：**コード新設せず** `DisclosureLedger`（FND-4）への**lore データ入力**。秘史/エンディング分岐（環境崩壊エンド/持続文明エンド）へ。

### ❌ 不採用（重複・既存で十分・タイクン化）

| 不採用 | 理由 |
|---|---|
| 個別の伐採/耕作/漁獲マイクロ操作 | **タイクン化回避**。環境は高位政策レバー1本＋エンジン駆動で創発させる |
| 多段の食料生産チェーン/BOM | **`ResourceProductionRules`#92・SCM#982 がカバー**。CLPS はストック制約を外から掛けるだけ |
| 気候変動システムの新規実装 | **`EventEngine`#116 の気候/災害イベントがカバー**。5因子診断で読むのみ |
| 敵対者（外敵侵攻）の新規システム | **`DiplomacyRules`#189・戦略レイヤーがカバー**。診断で読むのみ |
| 人口動態の作り直し | **`DemographicsRules`#153 がカバー**。収容力比を渡して接続するだけ |
| 王朝崩壊そのものの再実装 | **`DynastyRules`#867・`FactionStateRules` がカバー**。環境を第二の崩壊エンジンとして合流するのみ |

### 推奨着手順
`CLPS-1`（環境ストック＝最も固有で欠落の大きい signature・全ての土台）→ `CLPS-2`（収容力＝人口連結）→ `CLPS-3`（崩壊合流＝既存の崩壊判定へ外因経路）→ `CLPS-4`（自制レバー＝勝ち筋を入れて初めて意味を持つ）→ `CLPS-5`（エリート隔離＝なぜ自制しないか）→ `CLPS-6`（5因子診断＝複合の可視化）→ `CLPS-7`（lore＋孤立接続＝コード新設なし）。

> いずれも既存の社会・政治・兵站ロジックを**後退させず接続**する additive 設計。`Province`#109（惑星内政）と `FactionStateRules`（勢力崩壊）の間に**環境という外因層**を1枚敷き、崩壊を「人間内因の単因」から「環境×社会の複合」へ深める。**自制という我慢の勝ち筋**を入れることで、収奪一辺倒でない戦略の幅を生む。
